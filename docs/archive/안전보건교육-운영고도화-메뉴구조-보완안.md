# 안전보건교육 운영 고도화 — 메뉴 구조 보완안

- 작성일: 2026-07-18
- 상태: **후속 보완사항**
- 적용 범위: 현재 RFP 최소 시연 범위에는 미적용
- 목적: 엑셀로 관리하던 현업 근로자·관리감독자의 교육 업무를 역할별로 분리해 운영하기 위한 확장안

## 1. 제안 메뉴 구조

```text
안전보건교육 관리
├─ 현업 근로자
│  ├─ 근로자 명단 관리
│  ├─ 정기교육
│  ├─ 채용 시 교육
│  └─ 이수현황
└─ 관리감독자
   ├─ 교육 관리
   └─ 이수현황
```

## 2. UX 검토 결론

사용자가 실제 업무에서 먼저 구분하는 대상이 `현업 근로자 / 관리감독자`라면 이해하기 쉬운 구조다. 엑셀 시트가 대상과 교육 종류별로 나뉘어 있던 업무를 시스템 메뉴로 옮기는 방향도 자연스럽다.

다만 현재 제안요청서 최소 시연 범위에 그대로 추가하면 메뉴가 다시 복잡해지므로, **현재 단일 안전보건교육 화면을 유지하고 향후 운영 고도화 단계에서 적용**한다.

## 3. 적용 시 필수 원칙

### 3.1 메뉴 깊이를 늘리지 않는다

현재 구조에 그대로 붙이면 다음처럼 4단계가 될 수 있다.

`안전보건관리체계 → 안전보건교육 → 현업 근로자 → 정기교육`

사이드바를 4단계로 확장하지 말고, 다음 방식으로 구성한다.

- 사이드바: `안전보건교육` 단일 메뉴 유지
- 화면 상단 1차 전환: `현업 근로자 / 관리감독자`
- 화면 내부 탭: 명단·교육 종류·이수현황

### 3.2 명단 데이터의 중복 원장을 만들지 않는다

- 조직·사용자 정보에 존재하는 인원은 공통 인력정보를 참조한다.
- 조직정보에 없는 현업 근로자만 별도 교육대상자로 등록한다.
- 대량 인원은 엑셀 일괄등록을 지원하고, 화면에서 한 명씩 다시 입력하게 만들지 않는다.
- 화면 명칭은 `근로자 명단 관리`보다 `교육대상자 명단`을 우선 검토한다.

### 3.3 이수현황은 화면만 분리하고 데이터는 통합한다

현업 근로자와 관리감독자의 이수현황 메뉴가 각각 있더라도 별도 데이터로 저장하지 않는다.

- 공통 교육이력 데이터 사용
- 메뉴 진입 시 대상 유형 필터만 다르게 적용
- 동일한 이수현황 표·상세 화면을 재사용
- 교육 결과 확정 시 실제 출석자에게만 시간·최근 이수일·증빙을 반영

### 3.4 교육 종류별 메뉴는 운영량에 따라 결정한다

현업 근로자의 정기교육과 채용 시 교육을 담당자가 실제로 별도 시트·절차로 관리한다면 메뉴 분리는 적절하다. 운영 건수가 적다면 `교육 관리` 한 화면에서 교육 유형 필터로 구분하는 편이 더 단순하다.

관리감독자의 `교육 관리`도 현업 근로자 메뉴와 용어 수준을 맞춰 다음 중 하나로 통일한다.

- 안 A: 현업 근로자와 관리감독자 모두 `교육 관리 / 이수현황`
- 안 B: 실제 업무량이 많은 현업 근로자만 `정기교육 / 채용 시 교육`로 세분화

## 4. 권장 최종 표현

```text
안전보건교육
├─ 현업 근로자
│  ├─ 교육대상자 명단
│  ├─ 정기교육
│  ├─ 채용 시 교육
│  └─ 이수현황
└─ 관리감독자
   ├─ 교육 관리
   └─ 이수현황
```

단, 실제 UI에서는 위 구조를 4단계 사이드바로 모두 노출하지 않고 `대상 유형 전환 + 내부 탭`으로 표현한다.

## 5. 현재 프로토타입과의 관계

현재 시연용 프로토타입은 다음 최소 기능을 유지한다.

`교육 등록 → 출석 확인 → 결과 증빙 → 결과 확정 → 개인 이수현황 반영`

이 문서의 메뉴 구조는 다음 조건이 확인된 뒤 확장한다.

1. 현업 근로자와 관리감독자의 실제 담당자·권한이 다른지
2. 근로자 명단의 원천 데이터가 조직정보인지 별도 엑셀인지
3. 정기교육과 채용 시 교육을 별도 메뉴로 나눌 만큼 운영 건수가 많은지
4. 발주처가 추가 화면을 구축 범위로 승인하는지

## 7. 현재 시연용 구현 확정사항

2026-07-19 재검증에서도 현재 화면은 다음 원본 구조를 그대로 유지한다.

- 사이드바: `안전보건교육` 단일 메뉴
- 내부 탭: `교육 일정 / 이수 현황 / 과정 관리`
- 핵심 흐름: `교육 등록 → 실제 출석 확인 → 결과 증빙 확인 → 결과 확정 → 출석자 이수시간 반영`

발표 화면에 추가한 기능은 메뉴나 업무 단계를 늘린 것이 아니라, 핵심 처리 흐름을 `등록 → 출석 → 확정 → 이수` 순서로 직접 열어 주는 4단계 실행형 가이드다. 등록·출석·확정 모달 안에는 현재 단계의 확인 포인트를 인라인으로 표시하고, 확정 후에는 이수현황 탭으로 이동해 반영 결과를 확인한다.

`과정 관리`와 `반기 교육 점검표`는 원본 3탭의 운영 기능으로 그대로 유지한다. 다만 발표 가이드에는 포함하지 않고 필요할 때 담당자가 해당 탭에서 직접 확인하도록 분리해, 핵심 흐름을 짧고 명확하게 보여 준다.

현업 근로자·관리감독자별 확장 메뉴는 이 문서의 후속 보완안으로만 유지하며 현재 시연 UI에는 적용하지 않는다.
