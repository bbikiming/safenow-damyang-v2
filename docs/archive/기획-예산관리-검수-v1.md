## 3. 화면 정의서 전달성 인벤토리

> 목적: 추후 화면 정의서를 **기획 관점만으로** 작성할 때 개발자에게 전달해야 할 요소의 전수 목록.
> 아래 요소는 전부 프로토타입에서 실물 확인 가능 — 정의서는 이 목록 대비 누락 없이 기술하면 된다.

### 3.1 BGT01-V 예산 총괄표 (`bgt-main.html`)

**진입**: GNB 예산관리 > SNB 예산 총괄표. 이행관리 화면의 [예산 총괄표 바로가기]로도 진입. URL `?sheet=<id>` 지정 시 상세 직행.

**목록 뷰**
- KPI 카드 3: 총 편성액 / 총 집행액 / 집행률(%) — 하단 테이블(현재 필터 적용분) 자동 집계. 집행률 아이콘은 70% 미만 시 warning 색.
- 필터 3: 연도(2025·2026, 기본 2026) / 대상구분(전체·기관·시설) / 상태(전체·작성중·상신·승인·반려)
- 액션: [+ 점검표 생성] (모달 오픈)
- 테이블 7열: 대상(구분 칩+이름) · 항목 수 · 편성액 · 집행액 · 집행률(바+%) · 상태 칩 · 관리(조회 / 삭제)
  - 행 클릭 = 상세 진입. 삭제 버튼은 **작성중 행에만** 노출, 클릭 시 확인 대화상자.
  - tfoot 합계 행(편성·집행·집행률).
- 상태 칩 색: 작성중=회색(wt) / 상신=파랑(st-doing) / 승인=초록(st-done) / 반려=빨강(st-todo)
- 빈 상태: "조건에 맞는 점검표가 없습니다. 연도·대상구분·상태 필터를 바꾸거나 [+ 점검표 생성]으로 새로 만드세요."

**점검표 생성 모달 (단일 모달·단일 폼)**
- 필드 4: 연도 select / 대상구분 라디오(기관·시설, 기본 기관) / 대상 select(구분에 따라 목록 교체) / 예방 항목 체크 트리(`.org-inline` 인라인 스크롤 패널)
- 대상 목록: 기관=부서 10종(재난안전과…기획예산실), 시설=6종(담양군민체육관…담양군립요양병원) — 프로토타입 하드코딩
- 트리 규칙: 대분류 체크 시 하위 전체 선택/해제. **시트 행은 리프(말단) 항목만** 생성(대분류는 그룹핑 단위).
- 유효성 3: 대상 미선택 / 항목 0개 / 동일 연도+대상 중복 → 각각 토스트 차단
- 성공: 모달 닫힘 → 생성된 점검표 상세로 자동 이동 + 토스트

**상세 뷰**
- 헤더: [← 목록] / "{연도}년 {대상} 예산 편성·집행 점검표" / 상태 칩. 반려 상태면 반려 사유 경고 박스.
- 상태별 편집 안내 문구 1줄(테이블 상단).
- 행 테이블 5열: 항목(트리 경로 '시설 > 안전난간·추락방지') · 편성액 input(천원) · 집행액 input(천원) · 집행률(바, 자동) · 비고 input
- **편집 규칙 매트릭스** (UI disabled + 데이터 레이어 이중 강제):

  | 상태 | 편성액 | 집행액·비고 | 삭제 | 상신 버튼 |
  |---|---|---|---|---|
  | 작성중 | ○ | ○ | ○ | ○ |
  | 상신 | × | × | × | × (승인/반려 시뮬레이션 버튼) |
  | 승인 | × | ○ (집행 상시 등록) | × | × |
  | 반려 | ○ | ○ | × | ○ (재상신) |

- 저장 방식: input change 즉시 자동 저장 + '저장됨' 토스트 + 해당 행 집행률·tfoot 합계 즉시 재계산
- 전자결재 카드 (상태별 3형태):
  - 작성중·반려: [승인 요청 — 온나라 결재 상신] → 온나라 팝업(문서번호 자동 채번, 결재선 팀장→과장→부군수) → [확인] 시 상신 전이
  - 상신: 안내 문구 + [승인 처리 (시뮬레이션)] / [반려 처리 (시뮬레이션)] — 반려는 버튼 아래 **인라인** 사유 입력(필수, 빈 값 차단) 후 확정
  - 승인: "✓ 승인 완료 — 집행액·비고는 계속 입력할 수 있습니다."
- 결재 이력 카드: 최신순 타임라인 (생성/상신/반려(사유)/승인 이벤트 + 일시)

### 3.2 BGT02-S 예산 기준 설정 (`bgt-settings.html`)

**섹션 A — 편성·관리 원칙 및 준수사항 (R1)**
- 현행 문서 하이라이트 카드: 제목·시행일·등록(수정)일·첨부 칩·주요 내용(개행 유지)
- [원칙 등록·개정] 모달: 제목*(필수)·시행일*(필수, date)·주요 내용(textarea)·첨부(upload-drop 클릭 토스트 + `DYV2.fileHint()` 규격 문구)
- 개정 규칙: 신규 저장 = 새 문서가 현행으로 승격, 기존 현행은 개정 이력 테이블로 내려감
- 개정 이력 테이블 4열: 제목·시행일·등록(수정)일·관리(수정/삭제)
- 빈 상태 2종: 문서 0건 / 이력 0건 각각 안내 문구

**섹션 B — 예방관련 예산 항목 (R3)**
- 안내 노트: depth 무제한·사용 중 삭제 불가 설명
- 트리: 레벨당 20px 들여쓰기 + 세로 라인, 노드별 [+ 하위]/[수정]/[삭제], 점검표 사용 항목에 '사용 중' 뱃지
- 추가·수정 = 행 아래 **인라인 입력**(Enter 확정·Esc 취소, 모달 아님), [+ 대분류 추가]는 카드 헤더
- 삭제 차단 2종(토스트): "하위 항목이 있어 삭제할 수 없습니다…" / "점검표에서 사용 중인 항목은 삭제할 수 없습니다."

### 3.3 상태 전이 (정의서 공통 다이어그램용)

```
작성중 ── 상신(온나라) ──> 상신 ──> 승인 (편성 잠금·집행 상시)
  ↑                        │
  └────── 반려(사유 필수) ──┘   * 반려 → 편성 수정 후 재상신 가능
```

### 3.4 데이터·표기 규칙

- 금액 단위: 천원, 3자리 콤마 (`480,000천원`)
- 집행률 = round(집행액 ÷ 편성액 × 100), 편성 0이면 0%. 바 폭은 100% 캡, 숫자는 실제값(초과 집행 표기 가능). 70% 기준 색 분기(green/warning)
- 총괄 집계 = 점검표 rows 합산의 합 (별도 입력 없음). KPI는 **현재 필터 적용분** 기준
- 점검표 유일성: 연도+대상 1건
- ID 체계: 점검표 `bs-<연도>-<n>`, 항목 `it-<n>`, 원칙 `pol-<n>`

### 3.5 메시지 카탈로그 (토스트·차단·빈 상태 전수)

생성: `대상을 선택하세요.` / `예방 항목을 1개 이상 선택하세요.` / `이미 {연도}년 {대상} 점검표가 있습니다.` / `점검표가 생성되었습니다.`
편집: `저장됨` / `현재 상태에서는 편성액을 수정할 수 없습니다.` / `상신 중에는 편집할 수 없습니다.`
삭제: `작성중 상태의 점검표만 삭제할 수 있습니다.` / `점검표가 삭제되었습니다.`
결재: `온나라로 상신되었습니다.` / `승인 처리되었습니다.` / `반려 처리되었습니다.` / `반려 사유를 입력하세요.`
원칙: `제목을 입력하세요.` / `시행일을 입력하세요.` / `원칙 문서가 개정 등록되었습니다.` / `원칙 문서가 수정되었습니다.` / `원칙 문서가 삭제되었습니다. (프로토타입)`
항목: `항목명을 입력하세요.` / `저장되었습니다.` / `항목이 삭제되었습니다.` / 삭제 차단 2종(§3.2)

## 4. 정의서 작성 시 명시가 필요한 사항 (프로토타입 ↔ 실 구축 차이)

정의서를 기획 관점으로 쓸 때 아래 5건은 "프로토타입 동작"과 "실 구축 요건"을 구분해 기술해야 개발자 오해가 없다.

1. **결재 결과 수신**: 프로토타입은 [승인/반려 처리 (시뮬레이션)] 버튼. 실 구축은 온나라 결재 결과 연계 수신으로 대체 (연계는 온나라 1개만 — e호조 등 기타 연계 없음이 확정 방침).
2. **대상 목록 출처**: 기관·시설 목록이 하드코딩. 실 구축은 기본정보 > 관리대상 현황 연동으로 기술.
3. **연도 범위**: 2025·2026 고정. 실 구축은 연도 자동 생성 규칙(당해연도±n) 명시 필요.
4. **첨부파일**: 클릭 시 토스트만(실 업로드 없음). 제약 문구는 `DYV2.FILE_LIMITS` 단일 출처 그대로 기술.
5. **점검표 삭제 확인**: 네이티브 confirm 사용. 정의서에는 "확인 대화상자"로 기술(구현 방식 무관).
