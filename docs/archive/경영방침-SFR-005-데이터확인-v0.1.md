## 1. 코드값 (Enum) — 내부 코드는 개발자, 표시 라벨은 기획

> 프로토타입은 **표시 라벨**만 사용한다. DB 저장 코드(영문 상수)는 개발자가 정하되, **라벨↔코드 매핑을 1:1 고정**해 주세요.

### 1-1. 점검 결과 유형 (점검표 마스터) 🟡
| 표시 라벨 | 의미 | 비고 |
|---|---|---|
| `O / X` | 적합/부적합 2값 판정 | 런타임 점검표(F5)는 현재 `O · X · 해당없음` 3값 |
| `텍스트` | 자유 서술 입력 | **확인 필요**: 런타임에서 텍스트형 항목은 O/X 버튼 대신 텍스트 입력으로 렌더할지 (현재 F5는 전 항목 O/X+비고 공통) |

→ **개발 확인**: 결과유형을 항목 속성으로 저장하고 런타임 렌더를 분기할지 여부. (현재 마스터 모달엔 유형 선택, 런타임 F5는 미분기)

### 1-2. 점검 결과 판정값 (런타임) 🟢
| 라벨 | 처리 |
|---|---|
| `O` | 적합 |
| `X` | 부적합 → **확정 시 개선조치 자동 생성** |
| `해당없음` | 집계 제외 |

### 1-3. 전자문서 상태 (e-Doc 상태머신) 🟢
`작성중` → `등록완료` → `확정`(온나라 결재 상신). 코드: `edoc.js ST_CHIP` 참조.

### 1-4. 개선조치 상태 (X 항목 자동 유입) 🟢
`접수` → `계획` → `진행` → `완료확인` → `종결`. 코드: `edoc.js IMP_FLOW`.

### 1-5. 고려사항 구분 🟢
`설정 시` · `점검 시` · `공통` (3값).

### 1-6. 문서 출처 (첨부) 🔴
원본 시스템(하늘팀장님) v0.2에는 `시스템 원본 / 온나라 사본 / 외부 PDF / HWP 기존` 4분류 존재. **v2 적용 여부·코드 확정 필요.**

## 2. 마스터 데이터 구조 — 필드·키는 개발자, 항목 값은 기획 제공

### 2-1. 안전보건 목표·KPI 🟡
프로토타입 목표는 자유 텍스트 3줄. 실제는 **항목화**가 필요한지 확인.
| 기획 제공 필드(개념) | 예시 | 개발 결정 |
|---|---|---|
| KPI명 | 위험성평가 이행률 | 컬럼·타입 |
| 목표값·단위 | 100 / % | 숫자·단위 분리 여부 |
| 집계 주기 | 실시간·월·분기·연 | enum |
| 산출식·데이터 소스 | 완료/계획×100 (RSK) | **모듈 연계 키** |

→ **개발 확인**: KPI 실측 집계를 어느 모듈 데이터에서 가져올지(RSK·INS·EDU·OPN·IMP). 현황 이행률(`complianceRate`)과의 관계.

### 2-2. 근거법령 ↔ 세부내용 🟡
| 필드(개념) | 예시 |
|---|---|
| 법령명 | 산업안전보건법 |
| 조항 | 제14조 |
| 세부내용(방침 적용 요지) | 안전보건경영방침 설정·게시·주지 |

→ **개발 확인**: **전사 공통 법령 마스터와 통합할지, 경영방침 전용으로 둘지.** (원본 시스템엔 `법령 마스터(CMP03-S)` 별도 존재 — v2 통합 정책 미정)

### 2-3. 점검표 항목 마스터 🟡
| 필드(개념) | 예시 |
|---|---|
| 영역 | 목표·KPI / 조직·인력 / 위험성평가 / 도급·협력 / 의견청취 |
| 점검 항목 | KPI 목표값 대비 달성 여부 |
| 결과유형 | O/X · 텍스트 (1-1) |
| 관련 근거 | 중처법 시행령 §4-1 (직접 입력) |

현재 런타임 항목 = `edoc-templates.js > CHECKLIST_PRESETS.policy` (7건). 마스터 모달(`policyChecklist`)과 **단일 출처화** 필요.

## 5. 기획 vs 개발 경계 (요약)

| 기획이 정한다 | 개발자가 정한다 |
|---|---|
| 점검 항목·영역·근거 문구 | 테이블·필드·키·인덱스 |
| 결과유형·상태의 표시 라벨 | 상태 코드(영문 상수)·상태머신 구현 |
| 점검 주기·결재선·공표 정책(개념) | 온나라/알림 API·스케줄러 |
| KPI 정의(개념) | KPI 집계 쿼리·데이터 소스 |
| 마스터 통합 범위 결정 | 마스터 저장 구조 |

## 6. 수용 기준 (기획 관점)

1. 경영방침·목표 작성 → 저장 → 온나라 상신 → 공표 흐름이 끊김 없이 동작.
2. 점검표 항목을 **O/X·텍스트 + 관련근거**로 등록·수정·삭제·조회 가능.
3. 반기 점검에서 **X 판정 항목이 개선조치로 자동 등록**되고 확정 시 온나라로 상신.
4. 근거법령·고려사항을 등록·수정·삭제·조회 가능하고, 점검표·방침에서 인용 가능.
5. 증빙자료 다중 첨부·조회 가능.
