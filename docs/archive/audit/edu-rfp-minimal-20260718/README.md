# 안전보건교육 RFP 최소 시연 검토

- 검토일: 2026-07-18
- 대상: `edu.html` 안전보건교육 이수 관리
- 결론: **조건부 적합** — 현재 저장소의 RFP AI 정리본과 팀장 초기 설계를 기준으로, 직접 요구사항을 한 화면의 현실적인 업무 흐름으로 축소했다.

## 1. 판단 근거

현재 저장소에서 확인 가능한 직접 요구는 `SFR-008`의 다음 두 항목이다.

1. 안전보건관계법령에 따른 교육 이수 관리
2. 인력에 대한 증빙자료 관리

팀장 초기 설계의 확정 흐름은 다음과 같다.

`교육 등록 → 대상자 안내 → 담당자 출석 확인 → 결과 증빙 첨부 → 결과 확정·온나라 요청 → 실제 출석자 이수시간 반영`

전자서명은 제안요청서 직접 요구가 아니므로 제외한다. 신청자 전체를 자동 이수 처리하지 않고, 담당자가 확인한 실제 출석자만 결과 확정 시 반영한다.

> 한계: 저장소에는 원문 제안요청서가 아니라 AI 정리본만 있다. 계약 범위 최종 판정은 원문 RFP와 발주처 확인이 필요하다.

## 2. 최소화 결정

유지한 기능:

- 교육 일정 등록과 대상자 선택
- 담당자 출석 확인
- 계획·교안 및 결과 증빙 연결
- 결과 확정 후 출석자 이수시간 반영
- 개인별 이수현황, 부서 필터, 미이수자 알림
- 초기 설계의 3탭: 교육 일정 / 이수 현황 / 과정 관리
- 공통 단일 모달, 공통 첨부파일 제한 안내

시연 범위에서 제외한 기능:

- 교육 종류별 7개 하위 메뉴
- 별도 근로자 명단 원장
- 교육 신청·승인·신청자 전자서명
- D-day 중심의 복잡한 대상자 현황판
- 회차 자동 생성·보강교육 전용 화면
- 별도 통계 메뉴와 중복되는 상세 현황

## 3. 간부 시연 순서

| 단계 | 시연 내용 | 상태 |
|---|---|---|
| 1 | `3분 핵심 시연` 버튼으로 최소 범위 설명 | 정상 |
| 2 | 교육 일정·장소·강사·대상자·교안 등록 | 정상 |
| 3 | 교육 후 담당자가 실제 출석자만 선택 | 정상 |
| 4 | 교육일지·출석부 등 결과 증빙을 필수 첨부하고 확정 | 정상 |
| 5 | 확정된 출석자의 이수시간·최근 이수일 확인 | 정상 |

## 4. 현실성 및 구현 검증

- 대상자를 고르지 않으면 교육 등록이 되지 않는다.
- 실제 출석자를 고르지 않으면 실시완료로 전환되지 않는다.
- 결과 증빙이 없으면 결과 확정이 되지 않는다.
- 실제 출석자만 이수시간이 가산된다.
- 확정 결과에서 증빙 파일명과 사람별 반영 시간을 확인할 수 있다.
- 모달은 한 번에 하나만 열리고, 튜토리얼은 비모달 안내 패널로 동작한다.
- 시연 초기화로 데모 데이터를 원상복구한다.
- 탭, 버튼, 필터, 등록·출석·확정 모달의 주요 동작을 브라우저에서 검증했다.
- 중복 ID, 이름 없는 버튼, 가로 넘침, 콘솔 오류가 없음을 확인했다.
- 온나라 연계는 시연용 요청 팝업이며, 실제 전자결재 API 연동 완료를 의미하지 않는다.
- 파일은 프로토타입 특성상 실제 바이너리가 아니라 파일명만 저장한다.
- 법정 관리 기준 시간은 시연 설정값으로 표시하고 발주처 협의 전 확정값처럼 표현하지 않는다.
