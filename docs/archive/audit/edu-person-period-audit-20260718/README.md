# 교육대상자 이수 기간·시간 관리 UX 검토

- 검토일: 2026-07-18
- 사용자 목표: 엑셀에서 사람별로 수기 관리하던 교육 기간과 이수시간을 한 화면에서 빠르게 확인·관리
- 결론: **시간 관리는 부분적으로 편해졌지만, 기간 관리까지 맡기기에는 현재 화면이 부족하다.**

## 단계별 점검

1. 교육 일정 등록 — 정상
   - 한 교육에 여러 대상자를 묶어 등록하고 출석을 한 번에 처리할 수 있어 사람별 수기 입력은 크게 줄어든다.
   - 증빙 확정 후 실제 출석자에게만 시간이 반영되는 흐름은 현실적이다.

2. 개인별 이수 현황 — 보완 필요
   - 현재는 `이수시간 / 관리기준`, 최근 이수일, 상태를 한눈에 볼 수 있다.
   - 그러나 관리기간, 이수기한, 남은 시간, 기한 임박·초과 여부가 없어 주무관이 다시 날짜를 계산해야 한다.
   - 서로 다른 교육과정 시간을 한 사람의 총시간으로 합치면 과정별 의무 충족 여부를 잘못 판단할 수 있다.

3. 과정 관리 — 보완 필요
   - 과정별 주기와 기준시간은 확인할 수 있다.
   - 이 기준이 개인별 현황의 특정 관리기간과 연결되어 있지 않아, 누가 어느 기간에 어떤 과정을 충족해야 하는지 추적할 수 없다.

## 최소 보완안

새 메뉴를 만들지 않고 `이수 현황` 탭만 다음처럼 보완한다.

- 관리 단위: 사람 1명 × 교육과정 1개 × 관리기간 1개
- 표 컬럼: 성명 / 부서 / 대상 교육 / 관리기간 / 이수시간·기준 / 남은 시간 / 이수기한 / 상태
- 상태: 충족 / 시간 부족 / 기한 임박 / 기한 초과
- 필터: 부서 / 대상 교육 / 상태 / 30일 이내 마감
- 행 상세: 확정된 교육 이력, 출석 여부, 반영 시간, 연결 증빙
- 일괄 처리: 현재 필터 결과를 선택해 미이수자 알림

권장 예시:

`김도시 | 관리감독자 정기교육 | 2026.01.01~12.31 | 11/16h | 5h 남음 | 2026.12.31 | 시간 부족`

## 접근성 및 검증 한계

- 진행률은 색상뿐 아니라 시간 텍스트와 상태 칩을 함께 사용해 기본적인 판독성은 좋다.
- 컬럼을 늘릴 때 가로 스크롤만 추가하면 읽기 어려워질 수 있으므로 중요도 순서와 반응형 축약이 필요하다.
- 스크린샷으로는 대량 인원 성능, 엑셀 가져오기, 키보드·스크린리더 사용성, 실제 법정 기준 계산, 운영 데이터 영속성을 검증할 수 없다.
- 과정별 주기와 시간 값은 발주처 및 관련 기준 확인 후 확정해야 한다.
