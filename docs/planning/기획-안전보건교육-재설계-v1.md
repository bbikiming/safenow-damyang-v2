# 기획 — 안전보건교육 재설계 v1

2026-07-16. `기획-안전보건교육-결정사항-v1.md`의 A/B 항목 확정값 반영.
법정 시간: 산업안전보건법 시행규칙 별표4 (2023-09-27 개정).

## 0. 확정된 결정사항 요약
- 정기교육 주기: **채용일 기준 개인별 반기(6개월) 사이클** (담양군 운영 방침). 예: 3월 채용 → 9월까지, 2월 채용 → 8월까지.
- 고용형태·계약기간 반영 (계약직 다수, 모든 부서에 존재 가능 — 재난안전과 포함).
- 근로자 명단: 프로토타입 수준 목업 (일반적인 지자체 구조).
- 관리감독자 메뉴: 정기교육 / 기타 교육 2분류.
- 기타 교육 분류: 특별교육 / 작업내용 변경 시 / MSDS / 직무교육 / 자체·기타 — 5종 확정.
- 무재해 감면 미적용, 온라인교육 없음(전부 대면).
- 집합교육: 정원·마감일 없음. 서명파일은 **신청 시** 업로드. 불참자 처리 없음.
- 독촉: 단순 알림 발송(해당 부서 담당자 대상), 사유 회신 없음. 이력 기록.

## 1. 메뉴 구조 (안전보건교육 그룹)

| 메뉴 | 파일 | 내용 |

| 정기교육 (현업근로자) | edu-reg.html | 집합교육 · 자체교육 탭 |
| 채용시교육 (현업근로자) | edu-hire.html | 채용 시 교육 등록·이수 처리 |
| 기타 교육 (현업근로자) | edu-etc.html | 분류 셀렉트 5종 |
| 관리감독자 정기교육 | edu-sup.html | 연 16h |
| 관리감독자 기타 교육 | edu-sup-etc.html | 분류 셀렉트 (채용시·작업변경·특별 포함) |
| 이수현황 | edu-status.html | 부서/대상자별 필요·인정·미달·완료 + 독촉 |
| 근로자 명단 관리 | edu-workers.html | 인사연동 + 부서별 직접 등록·엑셀 업로드 |

기존 edu.html은 리다이렉트 스텁. 전역 네임스페이스 `DYEDU`(js/edu-data.js) + 화면별 js.

## 2. 법정 필요시간 규칙 (edu-data.js에 상수·계산식 분리)

### 2.1 정기교육 — 개인별 채용일 기준 반기 사이클
- 사이클: 채용일부터 6개월 단위 반복. 현재 사이클 = 오늘이 속한 구간. 사이클 종료월까지 필요시간 충족해야 함.
- 필요시간/사이클: 비사무직(현업) **12h** · 사무직/판매 **6h**.
- 관리감독자: 채용(지정)일 기준 **12개월 사이클 16h**.
- 채용 첫 사이클: 채용시교육 필수 + 정기교육도 동일 카운트(담양군 방침 — 채용일 기준이므로 첫 사이클부터 발생).
- 계산식은 `DYEDU.requiredHours(worker, date)` / `DYEDU.cycleOf(worker, date)` 로 분리 — 법 해석 변경 시 이 함수만 교체.

### 2.2 채용시교육 — 고용형태·계약기간별
- 일용·계약 1주 이하: 1h / 계약 1주 초과~1개월: 4h / 그 밖: 8h.
- 업무 투입 전 실시 원칙 — 채용일 대비 이수일 경과 시 '지연' 경고 표시.

### 2.3 인정시간
- 완료된 교육의 시간 단순 합산(감면·방법 제한 없음). 교육 유형별로 해당 카운터에만 합산(정기→정기, 채용시→채용시).
- 기타 교육은 이수현황에 참고 표시(법정 정기시간에 미합산).

## 3. 근로자 명단 관리 (edu-workers)
- 인사정보시스템 연동분(공무원·공무직): `DYV2.ORG` members 파생, "🔗 인사정보시스템 연동" 라벨, 읽기 전용.
- 계약직: **부서 담당자**가 직접 등록 또는 엑셀 업로드(목업 — 업로드 시 샘플 N명 추가). 모든 부서에 존재 가능. 재난안전과는 조회만(등록 불가 전제이나 프로토타입은 관점 구분 없이 노출).
- 속성: 이름, 부서(deptId), 구분(사무직/비사무직/판매/관리감독자), 고용형태(공무원/공무직/기간제/일용), 채용일, 계약기간(기간제), 재직상태.
- 필터: 부서 · 구분 · 고용형태. 퇴직자는 목록 필터로 숨김, 이력 보존.

## 4. 정기교육 — 현업근로자 (edu-reg)

### 4.1 집합교육 탭
1. 재난안전과 담당자가 교육 등록: 일정·강사·장소·내용 (+첨부).
2. 부서에서 [신청]: 부서 선택 → 근로자 명단에서 대상자 체크 선택 → **서명파일 업로드(필수)** → 신청 완료.
3. 교육 종료 시 [교육 종료 처리]: 신청 대상자 전원에게 교육시간 카운트 + 이력.
- 정원·마감일 없음. 불참 처리 없음.

### 4.2 자체교육 탭
1. 부서 담당자가 등록: 부서·일정·강사·장소·내용.
2. 교육대상자 선택 → [진행 처리] = **즉시 교육완료** + 대상자 시간 카운트 + 이력.

## 5. 채용시교육 (edu-hire)
- 목록: 근로자·부서·채용일·필요시간(고용형태별)·이수일·상태(투입 전 이수/지연/미이수).
- 등록: 대상 근로자 선택 → 교육일·내용·증빙 첨부 → 완료 처리 시 카운트.

## 6. 기타 교육 (edu-etc / edu-sup-etc)
- 자체교육과 동일 폼 + **교육 분류 셀렉트**: 특별교육 / 작업내용 변경 시 / MSDS / 직무교육 / 자체·기타.
- 일정·강사·내용·대상자·증빙 서류(서명 등) 업로드.

## 7. 관리감독자 (edu-sup / edu-sup-etc)
- 대상 = 명단에서 구분이 '관리감독자'인 인원.
- 정기교육: 12개월 사이클 16h, 등록·진행은 집합/자체 동일 패턴.
- 기타 교육: 6장 분류 셀렉트에 채용시·작업변경·특별 포함.

## 8. 이수현황 (edu-status)
- 뷰 전환: 부서별 요약(대상 인원·완료 인원·완료율) ↔ 대상자별 상세.
- 대상자별 컬럼: 이름 | 부서 | 구분 | 고용형태 | 채용일 | 현재 사이클(기간·종료 D-day) | 필요시간 | 인정시간 | 미달시간 | 완료여부 | 채용시교육 상태.
- 미이수자 **개별 체크 선택 또는 [일괄 독촉]** → 해당 부서 담당자에게 알림 목업 + history 기록(REMIND). 사유 회신 없음.
- 독촉 이력 열람 가능.

## 8.5 v1.1 개선 (2026-07-16 화면 피드백)
1. SNB 3뎁스: 안전보건교육 > **현업근로자**(정기/채용시/기타) · **관리감독자**(정기/기타) + 이수현황·근로자 명단은 2뎁스 직속.
2. 집합교육 상세·신청현황은 모달 → **상세화면**(신청현황이 길어짐 전제).
3. 집합 정기교육은 **매월 진행** — 시드 월별 반영.
4. 채용시교육: 근로자 직접등록/엑셀업로드 시 **자동 생성**(채용 전이라 인사연동 대상 아님). 미이수/이수 탭, 체크박스 일괄 이수처리, 부서·고용형태 필터(가로) + 필터 결과 전체선택.
5. 이수현황에서 채용시교육 컬럼 제거(연동 안 함). 시드에 이수 완료자 비중 확대.
6. 이수현황 기본 탭 = 부서별 요약.
7. 근로자 명단: 자물쇠 이모지 삭제 → 출처 칩 `[인사연동] 홍길동`. 계약기간은 고용형태에 병합 `기간제[12개월]`.
8. 세로 배치 필터 전부 가로 배치로.

## 8.6 v1.2 개선 (2026-07-20 화면 피드백)
1. **교육 회차(일자 탭)** — 등록 폼의 `일자 + 시작 시각 + 교육 시간(h)` 직접 입력을
   `일자 + 시작~종료 시각`의 회차 탭으로 대체(집합·자체 공용, **최대 5일**).
   - 탭 라벨에 `N회차 MM/DD · Nh` 를 적어 탭을 열지 않아도 전체 일정이 읽히게 한다.
   - 일자 추가 시 **직전 회차 다음 날·같은 시간대**를 기본값으로 채운다(연속 일정이 가장 흔함).
2. **교육 시간 자동 산정** — 회차별 `종료−시작`(0.1h 단위)의 합계. 사용자가 직접 입력하지 않는다.
   계산은 `DYEDU.sessionHours(s)` / `sumSessionHours(list)` 단일 출처. 종료 ≤ 시작이면 그 회차는 0h + 경고.
   - 저장 시 대표값(`date`·`time`·`endTime`)은 1회차, `hours`는 합계로 파생 — 기존 화면·집계와 호환.
   - **수정 시 이수기록 동기화** — 시간이 바뀌면 `DYEDU.syncCourseRecordHours()`로 이미 쌓인
     records 의 hours 를 함께 갱신한다(카드의 교육시간과 이수현황 인정시간이 어긋나면 안 됨).
3. **이수현황 상세 필터** — 구분(현업/관리감독자/사무/판매) · 고용형태(공무원/공무직/기간제/일용) ·
   **채용연도** · **명단 출처**를 모집단 필터로 통일하고, **부서별 요약에도 같은 필터를 적용**한다.
   - 요약의 완료율은 선택된 모집단으로 다시 집계한다(`DYEDU.deptSummary(date, filterFn)`) —
     "관리감독자만 본 부서별 완료율"이 성립해야 한다.
   - 두 뷰 모두 카드 우측에 집계 모집단 배지(`.edu-pop-note`)를 표시해 필터를 건 채 전체 수치로 오독하는 것을 막는다.
   - 딥링크 `?cat=SUPERVISOR` · `?emp=CIVIL` 추가(기존 `?dept`·`?status`·`?short` 유지).

## 9. 데이터 모델 (DYEDU)
```
workers   [{ id, name, deptId, category:'OFFICE'|'FIELD'|'SALES'|'SUPERVISOR',
             empType:'CIVIL'|'PUBLIC'|'CONTRACT'|'DAILY', hireDate, contractMonths?,
             active, source:'HR'|'MANUAL'|'EXCEL' }]
courses   [{ id, kind:'REG_GROUP'|'REG_SELF'|'HIRE'|'ETC'|'SUP_REG'|'SUP_ETC',
             etcType?, deptId?, date, time, endTime, hours, instructor, place, desc, files,
             sessions:[{ date, start, end }],          // 회차(일자 탭) — 최대 5
             status:'OPEN'|'DONE', createdBy, history:[] }]
enrolls   [{ courseId, deptId, workerIds:[], signFile, at }]
records   [{ workerId, courseId, kind, hours, date }]   // 완료 시 생성 — 카운트 원본
reminders [{ at, by, deptId, workerIds:[], memo }]
```
- 시드: 일반 지자체 구조 목업 — 부서별 현업근로자·계약직 혼합 40~60명, 관리감독자 6~8명, 진행·완료 교육, 미달자·지연자 포함(독촉 시연 가능).
- 첨부 안내는 `DYV2.fileHint()`, 모달은 `DYV2.openModal` 단일 모달 + 인라인 패널(CLAUDE.md 규칙).
- 내 할일(my-work) 연동: 부서 담당자 관점 — 집합교육 신청, 자체교육 진행, 독촉받은 미이수 건 노출(카테고리 '교육' 추가).
