## 1. 총평

| 축 | 건강도 | 한 줄 진단 |
|---|---|---|
| 토큰 체계 | ⚠️ 이원화 | style.css는 토큰 70종으로 정비돼 있으나, **v2.css가 토큰을 쓰면서도 색 27종·폰트 260회·반경 72회를 원시값으로 재발명** |
| 컴포넌트 일관성 | ⚠️ 신·구 분기 | 공용 헬퍼(모달·업로드·조직도)는 대체로 준수. 단 **evl 계열 자작 모달(적층 포함)과 menu.js body-append 오버레이가 CLAUDE.md MUST 위반** |
| 모바일 반응형 | 🔶 부분 대응 | 셸(햄버거+드로어)은 완비. **두 CSS의 브레이크포인트 체계가 상이**하고, rsk 계열 커스텀 클래스는 미디어쿼리 전무 |

근본 원인은 3가지로 수렴한다.

1. **표면 토큰 부재** — 흰색(`#fff` 74회)·`#fafafa`(16회)를 담을 `--surface`/`--surface-alt` 토큰이 없어 매번 원시값이 재기입됨.
2. **레이어(z-index)·브레이크포인트 표준 부재** — 파일마다 임의 지정(9998/9999/99999 경쟁, style 767·1023 vs v2 560·860).
3. **신규 화면의 독자 계열 신설** — evl(자작 모달·픽커·버튼), sh-*(배지·테이블·탭·빈상태)가 기존 표준과 병렬 체계를 만들며 이원화 심화.

### 준수 확인(조치 불요) 항목
- 조직도 단일 출처: `PICK_TREE`(evl-eval)·`EDOC.ORG_TREE` 모두 `DYV2.ORG` 파생. 자체 조직 데이터 없음.

## 4. 리스크 및 주의사항

1. **조직도 uid 불변** — 권한 시드가 참조하므로 어떤 단계에서도 `DYV2.ORG` uid를 건드리지 않는다(CLAUDE.md §3).
2. **evl-eval 모달 재편(2-1)이 최대 리스크** — 평가 매트릭스↔평가자 선택 플로우가 UX 레퍼런스 정합으로 최근 재구현된 영역과 인접. 재편 시 기존 플로우(자동 선정 안내·임시저장 분리)를 회귀 검증할 것.
3. **토큰 수렴은 색이 미세하게 변한다** — HC-01 치환은 "픽셀 동일"이 아니라 "토큰 정합"이 목표. 근사색(예: `#1a7a32`→`--main #2D9270`)의 시각 변화는 의도된 것임을 검수 기준에 명시.
4. **별칭 전략** — select·배지 등 사용처가 많은 클래스는 즉시 삭제 대신 별칭 유지 후 점진 치환(한 커밋에 전 화면 회귀 부담 방지).
5. **onepager-docflow·screen-definitions는 예외 문서** — 레이아웃 시스템 밖 독립 문서 성격이므로 토큰 치환 대상에서 제외(하드코딩 카운트에서 분리 집계 완료).
6. 다온플레이스 포크에 동일 이슈가 승계돼 있을 가능성 — 본 플랜 완료 후 포크 반영 여부는 별도 결정.
