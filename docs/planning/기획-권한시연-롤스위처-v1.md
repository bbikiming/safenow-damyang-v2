# 기획 — 권한 계층 시연 롤 스위처 + 계층별 대시보드 (v1, 2026-07-21)

시연에서 **"같은 시스템이 직위에 따라 다르게 보인다"** 를 즉시 보여주기 위한 장치.
우측 상단 사용자 칩을 클릭해 권한(페르소나)을 전환하면, 전 화면의 헤더·GNB 와
대시보드(통합 현황)가 선택한 직위의 관점으로 다시 렌더된다.

## 1. 책임체계 → 권한 계층 매핑

중대재해처벌법 관리체계의 3계층을 그대로 시연 권한으로 쓴다.

| 계층 id | 라벨 | 대상 직위 | 법적 근거 | 계층 색(tone) |
|---|---|---|---|---|
| `head` | 총괄 책임자 | 군수 | 중처법 §4 안전보건 확보의무 총괄 (경영책임자, §2) | purple |
| `super` | 관리감독자 | 실과장·사업소장·읍면장 | 산안법 §16 소속 부서 관리·감독 | info |
| `staff` | 업무담당자 | 실무 수행자 | 부서 안전보건 업무 수행·기록 | success |

### 시연 페르소나 5인 (uid·deptId 는 `DYV2.ORG` 와 동일 — 변경 금지)

| 페르소나 | 계층 | 인물 | 비고 |
|---|---|---|---|
| `mayor` | head | 김담양 군수 | 경영책임자 |
| `safety` | super | 홍길동 재난안전과장 | 실과장 예시 |
| `fac` | super | 임시설 공공시설사업소장 | 사업소장 예시 |
| `town` | super | 노읍장 담양읍장 | 읍면장 예시 |
| `staff` | staff | 박안전 안전관리 주무관 (재난안전과 중대재해팀) | **기본값** — 기존 헤더 인물 |

## 2. 전환 UX

- 진입점: 헤더 우측 **사용자 칩**(이름 + 계층 배지) 클릭 → 권한 전환 드롭다운.
- 드롭다운은 계층 3그룹으로 페르소나를 나열(현재 항목은 `현재` 마크 + 좌측 브랜드 바).
- 선택 → `localStorage(dy-role-sim-v1)` 저장 → 화면 새로고침 → 도착 토스트
  ("총괄 책임자 — 김담양 군수 관점으로 전환되었습니다").
- 새 권한에서 숨겨지는 GNB 그룹의 화면이었다면 대시보드(`index.html`)로 이동.
- 대시보드 최상단 **권한 안내 바**(계층 배지 + 인물 + 법적 근거 + [권한 전환] 버튼)로
  현재 관점을 항상 표시 — [권한 전환]은 헤더 드롭다운을 그대로 연다.

## 3. 화면 차등 매트릭스

| 화면 요소 | head 군수 | super 관리감독자 | staff 업무담당자 |
|---|---|---|---|
| GNB | 업무문서·시스템 관리 숨김 | 시스템 관리 숨김 | 전체 노출 |
| 대시보드 히어로 | 군 전체 이행률(브랜드 그라디언트) + 중대재해·기한초과·결재대기·사업장 | 소관 부서 이행률(info 어센트 카드) + 부서 스코프 지표 | 내 할일 statbox 4종 |
| 대시보드 본문 | 책임체계 지정 현황 체인 · 부서별 이행 현황 · 반기 의무이행 점검(시행령 §4) · 결재 대기 · 예산 집행 · 부서 보고 피드 | 부서 진행 업무 · 위험성평가/개선조치 · 담당자 제출 확인 대기 · 부서원 교육 이수 | 시기도래 알림(목록/달력) · 대메뉴별 이행률 · 예방활동/알림 피드 |
| 관리감독자 부서 스코프 | — | 페르소나 `deptId` 기준 시드 (재난안전과·공공시설사업소·담양읍 각각 상이) | — |

## 4. 구현 위치

| 파일 | 내용 |
|---|---|
| `js/layout.js` | `DYROLE`(계층·페르소나·저장·드롭다운) — 헤더가 layout.js 소관이므로 여기 상주. GNB 필터(`hideNav`) 포함 |
| `js/dashboard.js` | `DYDSH` — `DYROLE.current()` 기준 3계층 대시보드 렌더 (`index.html`의 `#dsh-root` 마운트) |
| `css/style.css` | 롤 드롭다운·사용자 칩 계층 배지 (`.dy-role-*`, `.dy-user-tier`) |
| `css/v2.css` | 대시보드 히어로·책임체계 체인·권한 안내 바 (`.dsh-hero`, `.dsh-chain`, `.dsh-rolebar` 등) |

표준 준수: 모달 아님(헤더 드롭다운 — 알림 드롭다운과 동일 계열, 동시 1개만 열림) ·
색·폰트·z-index 전부 토큰 · 상태 배지는 `chip-status` + `DYV2.toneOf` · 계층 색은
`--status-{purple|info|success}-*` 재사용(신규 팔레트 없음).

## 5. 한계 (프로토타입 범위)

- 권한은 **표시 차등**만 시연한다 — 데이터 접근 통제·서버 권한 검증은 범위 밖.
- 대시보드 외 화면(목록·상세)은 계층과 무관하게 동일하다. 필요 시 `DYROLE.current()`
  를 각 화면 모듈에서 참조해 확장한다.
- 부서 스코프 수치는 시연 시드(`js/dashboard.js` `SUPER_SEED`·`DEPT_RATES`)다.
