# 기획 — 안전보건교육 온나라 결재 상신 (결재 문서 미리보기 3종) v1.1

2026-07-24. 안전보건교육(`edu` GNB 그룹)에 **온나라 전자결재 상신** 기능을 붙이고,
상신 전에 **어떤 서식으로 기안문이 생성되는지**를 화면에서 눌러 확인하게 한다.

> **이 문서의 목적** — 결재 서식(문서 양식) 컨펌용이다. 실제 온나라 연계(호출기안·회신)는
> 프로토타입 시뮬레이션이며, 확정 대상은 **문서에 무엇이 어떤 순서로 담기는가**다.

> **v1.1 (2026-07-24)** — 상신 트리거를 **3종(총괄 / 교육별 / 개인별)**으로 분리하고,
> 목록·상세에 **온나라 결재 상태(미상신·결재중·결재완료·반려)**를 칩으로 노출한다. §7 참조.
> (§3의 3종 서식은 그중 **총괄** 트리거의 탭 구성으로 편입된다.)

---

## 1. 배경 · 문제

- 현재 안전보건교육은 등록·신청·이수 관리까지만 시스템 안에서 돌고, **결재는 시스템 밖**(한글 문서 작성 → 온나라 첨부)에서 이뤄진다.
- 담당자는 매번 교육 목록과 이수 현황을 손으로 옮겨 적어 기안문을 만든다 — 옮겨 적는 과정에서 수치가 어긋나고, 부서마다 서식이 다르다.
- 따라서 **시스템이 보유한 데이터(DYEDU)로 규격 기안문을 자동 생성**하고, 담당자는 서식·범위만 고른 뒤 상신하면 되게 한다.

## 2. 확정 사항

| 항목 | 결정 |
|---|---|
| 진입점 | 전 `edu-*` 화면 **우측 최상단**(페이지 제목 행 오른쪽) 고정 버튼 `[온나라 결재 상신]` |
| 상호작용 | 버튼 → **결재 문서 미리보기 팝업** (인력평가 `evl-eval` 의 결재 문서 미리보기와 같은 패턴) |
| 서식 | 한 팝업 안에서 **3종을 탭으로 전환** — ① 교육 종류별 목록 · ② 개인별 이수 현황 · ③ 통합 |
| 조회 조건 | 팝업 상단에서 **대상 연도 · 대상 부서**를 바꾸면 문서가 즉시 다시 생성된다 |
| 상신 | 팝업 하단 `[온나라로 결재 상신]` → 확인 → 문서번호 부여 안내(연계 시뮬레이션) |
| 결재선 | **고정하지 않는다** — 기본값 `기안 → 팀장(검토) → 과장(결재)`, 조직도에서 변경·추가·삭제 (§5) |
| 데이터 | 전부 `DYEDU` 파생 — 결재용 별도 시드를 만들지 않는다 |

### 2.1 화면별 기본 서식

버튼은 모든 화면에서 같지만, **그 화면에서 가장 쓸 법한 서식이 먼저 열린다**(탭으로 언제든 전환).

| 화면 | 기본 탭 |
|---|---|
| 정기교육 · 채용시교육 · 기타 교육 · 관리감독자 정기/기타 · 정기교육 상세 | ① 교육 종류별 목록 |
| 이수현황 · 근로자 명단 관리 | ② 개인별 이수 현황 |

## 3. 문서 서식 3종

세 서식은 **머리(기관 표시 + 결재란 + 제목 + 문서 개요표)와 꼬리(붙임 · 연계 안내)를 공유**하고, 본문 구성만 다르다.

### 공통 머리

- 기관 표시 `담양군청 / {기안자 소속}`, 우측 **결재란** — 칸 수·직위·이름은 §5 의 결재선을 그대로 따른다
- 제목 `{연도}년 …` · 부제 `대상기간 · 기준일 · 대상 범위`
- 문서 개요표 — 문서번호(상신 시 온나라 부여) · 기안일자 · 기안자 · **결재선** · 대상 범위 · 근거 법령
  - 근거 법령: 산업안전보건법 §29, 같은 법 시행규칙 §26 [별표 4] / 중대재해처벌법 §4①1호, 시행령 §4

### ① 교육 종류별 결재 목록 문서 — `{연도}년 안전보건교육 실시 결과 보고`

**무엇을 언제 몇 시간 실시했는가**를 결재받는 문서.

| 절 | 내용 |
|---|---|
| Ⅰ. 종합 요약 | 실시 교육(건) · 총 교육시간(h) · 이수 연인원(명) · 참여 부서(개)<br>※ 부서를 지정한 문서에서는 마지막 칸이 **부서 대상 인원**으로 바뀐다(‘참여 부서 1개’는 정보가 없으므로) |
| Ⅱ. 교육 종류별 실시 집계 | 교육 종류 / 실시 건수 / 교육시간 합계 / 이수 연인원 / 종료·전체 + **합계 행** |
| Ⅲ. 교육 종류별 상세 목록 | 종류별 소제목(건수·시간·연인원) 아래 표 — 연번 / 교육 일시 / 교육명(내용) / 시간 / 강사 / 장소 / 대상 부서 / 이수 / 상태 |
| 붙임 | 교육계획서 · 참석자 서명부 · 교육 사진 |

- 교육 종류 순서: 정기(집합) → 정기(자체) → 채용시 → 기타 → 관리감독자 정기 → 관리감독자 기타.
- 해당 연도에 실적이 없는 종류는 표에 나타나지 않는다(빈 절을 만들지 않는다).

### ② 개인별 교육 이수 현황 결재 문서 — `{연도}년 안전보건교육 개인별 이수 현황 보고`

**누가 몇 시간 이수했고 누가 미달인가**를 결재받는 문서.

| 절 | 내용 |
|---|---|
| Ⅰ. 종합 요약 | 교육 대상자 · 이수 완료 · 미이수 · 이수 완료율 + 필요시간 산정 근거 문장 |
| Ⅱ. 부서별 이수 현황 | 부서 / 대상 / 완료 / 미이수 / 완료율 |
| Ⅲ. 개인별 이수 현황 | 연번 / 부서 / 성명 / 구분 / 고용형태 / 채용일 / **교육 사이클(기간)** / 필요 / 인정 / 미달 / 이수 여부<br>미이수 행은 옅은 경고색으로 구분해 결재권자가 훑어볼 수 있게 한다 |
| Ⅳ. 미이수자 조치 계획 | 미이수 인원·부족 시간 합계, **사이클 종료 D-30 이내 인원**, 차기 집합교육 안내 |
| 붙임 | 개인별 이수 내역 · 미이수자 명단 |

- 필요·인정·미달·사이클은 전부 `DYEDU.statusRow()` 한 곳에서 나오므로 **이수현황 화면의 수치와 항상 일치**한다.

### ③ 통합 결재 문서 — `{연도}년 안전보건교육 실시 결과 및 이수 현황 통합 보고`

①과 ②를 **한 건의 기안문**으로 묶는다. 실시 실적과 이수 결과를 한 번에 결재받을 때 쓴다.

Ⅰ. 종합 요약(지표 8칸: 실시 교육·총 교육시간·이수 연인원·교육 대상자·이수 완료·미이수·완료율·부족 시간 합계)
→ Ⅱ. 교육 종류별 실시 집계 → Ⅲ. 교육 실시 목록(종류 열 포함 단일 표) → Ⅳ. 부서별 이수 현황
→ Ⅴ. 개인별 이수 현황 → Ⅵ. 미이수자 조치 계획.

## 4. 조회 조건 — 대상 연도 · 대상 부서

- **대상 연도** — 데이터에 실재하는 교육 연도 + 기준일 연도. 문서의 대상기간(`{연도}.01.01~12.31`)을 결정한다.
- **대상 부서** — 전체 부서 또는 특정 부서 1개. 지정하면
  - 교육은 그 부서가 **신청(집합)했거나 그 부서가 등록(자체·채용시)한 건**만 남고,
  - 이수 인원·대상 부서 열도 그 부서만 집계·표기하며,
  - 대상자·부서별 요약도 그 부서만 남는다.
- 조건을 바꾸면 문서가 즉시 다시 그려진다 — **바뀐 조건이 문서 부제·개요표에 그대로 찍히므로** 무엇을 결재받는지 오독할 수 없다.

## 5. 결재선 — 법령이 정하지 않으므로 지정한다

### 왜 고정하지 않는가

| 구분 | 정하는 것 | 근거 |
|---|---|---|
| 안전보건 법령 | **의무 주체**(사업주 = 군수)와 교육 실시·기록·점검 의무 | 산업안전보건법 §29 등 / 중대재해처벌법 시행령 — 교육 실시 여부 반기 1회 이상 점검·조치 |
| 행정 문서 규정 | 결재의 원칙 — 원칙적으로 기관장 결재, 위임 사항은 **전결**, 부재 시 **대결** | 「행정업무의 운영 및 혁신에 관한 규정」(대통령령) |
| 지자체 자치법규 | **문서 종류별 전결권자** — "안전보건교육 실시 결과 보고 = 과장 전결" 식 | 담양군 사무 위임·전결 규칙 |

즉 **누가 결재하는지는 안전보건 법령이 아니라 담양군 위임전결규칙이 정한다.** 시스템이 결재선을 못박으면
규칙 개정·조직 개편 때마다 코드를 고쳐야 하고, 부서·문서 종류에 따라 전결 단계가 다른 현실을 담지 못한다.
그래서 **기본값만 제안하고 지정·변경할 수 있게** 한다. (실제 온나라 연계 시에도 결재선은 온나라 쪽 값이 최종이다.)

### 동작

- 위치 — 팝업 안 `결재선` 카드(조회 조건 아래, 서식 탭 위). 바꾸면 **문서의 결재란·개요표가 즉시 갱신**된다.
- 기본값 — **기안자 소속 부서의 팀장(검토) → 부서장(결재)**. 조직도(`DYV2.ORG`)에서 직위명(`…팀장`, `…과장/소장/실장/국장/읍장/면장`)으로 찾아 파생하므로 조직 개편이 그대로 반영된다.
- 1단계 `기안`은 **현재 로그인 사용자로 고정**(작성자 = 기안자).
- 2단계부터 `[조직도]` 버튼 → **공용 인라인 조직도**(`ORGPICK` `member` 모드)에서 선택. 새 드롭다운을 만들지 않고 전 화면이 쓰는 같은 트리를 그대로 쓴다. 별도 모달을 띄우지 않으므로 단일 모달 규칙에 어긋나지 않는다.
- `[단계 추가]` — 최종 결재자 앞에 검토 단계를 끼운다(기안 외 최대 3단계). `[삭제]` — 해당 단계 제거(최종 결재 1단계는 유지). `[기본값]` — 팀장 → 과장으로 복원.
- **단계 이름은 자동** — 마지막이 `결재`, 그 앞은 모두 `검토`(행정 결재 관행: 검토 N + 최종 결재 1).
- 결재자를 고르지 않은 단계가 있으면 **상신이 차단**되고 안내 토스트가 뜬다.
- 지정한 결재선은 세션에 남아 **다른 교육 화면으로 이동해도 유지**된다.

## 6. 상신 · 이력

1. `[온나라로 결재 상신]` → 문서명·대상 범위·결재선을 확인 문구로 다시 보여준다.
2. 확인 시 문서번호(`재난안전과-NNNN`)를 부여하고 상신 안내 팝업 + 토스트를 띄운다.
3. 상신 이력은 세션에 남아 **팝업 상단 ‘최근 상신’ 줄**과 **문서 개요표의 문서번호 칸**에 반영된다.
4. 실제 연계 시에는 온나라가 결재 완료 시점에 **문서등록번호·결재일**을 회신하고, 그 값이 교육 이력과 연결된다(현 단계는 시뮬레이션).

## 7. v1.1 — 상신 트리거 3종 + 결재 상태

§3의 서식은 "무엇을 담느냐"였다. v1.1은 **"어디서 무엇을 대상으로 상신하느냐"**를 3종으로 나눈다.

### 7.1 트리거 3종

| # | 트리거 | 진입 위치 | 대상 | 문서 |
|---|---|---|---|---|
| Type 1 | **총괄 상신** | 목록 화면 **우측 최상단** `[총괄 결재 상신]` | 대상 연도·부서 범위의 **모든 총괄 데이터** | §3의 3종 탭(종류별·개인별·통합) |
| Type 2 | **교육별 상신** | 각 교육 **카드/상세의 결재 상태 칩** · 상세 상단 `[이 교육 결재 상신]` | **교육 1건**(그 N월 집합/자체/기타/관리감독자 교육) | 그 교육의 **실시 결과 보고**(개요·회차·참석/이수·서명·증빙) |
| Type 3 | **개인별 상신** | 이수현황 **대상자별 상세 표의 각 인원 행** | **대상자 1명** | 그 사람의 **교육 이수 확인서**(이수 내역·인정시간·판정) |

- 교육별(Type 2)은 **모든 교육 종류**에 적용한다 — 상세 페이지가 있는 정기 집합교육은 상단 버튼으로, 그 외(정기 자체·기타·관리감독자)는 **카드의 상태 칩/상신 버튼**으로 같은 문서를 띄운다.
- **채용시교육**은 화면이 근로자 행 단위라 교육 카드가 없었으므로 **`교육 건` 탭**(미이수 / 이수 / 교육 건)을 신설해 실시한 채용시 교육을 **건 단위 카드**로 노출하고 같은 교육별 상신 컨트롤을 건다. 대상자별 이수 취소는 종전대로 `이수` 탭에서 처리한다.
  - 일괄 이수처리는 **교육 건 생성 방식**(근로자별 개별 건 / 한 건으로 묶기)을 고를 수 있다. 묶기는 이수일이 하나여야 하므로 일괄 이수일과 함께 쓰이고, **필요시간(1h·4h·8h)이 다르면 법정 교육시간이 다른 별개 과정이라 건이 나뉜다**. 묶인 건은 한 장의 결재 문서로 부서별 이수 현황을 담는다.
  - 묶인 건에서 개인 이수를 취소해도 **그 사람 기록만** 회수되고 건과 다른 대상자는 유지된다(`DYEDU.removeWorkerFromCourse`).
- 교육별 문서 범위는 **해당 교육 1건**이다("N월 전체"가 아니라 그 상세의 교육 1건).
- 개인별(Type 3) 문서의 인정시간은 현재 사이클 기준이고, 이수 내역은 대상자의 전체 이력을 담는다.

### 7.2 결재 상태 (목록 노출)

- 상태 4단계: **미상신 → 결재중 → 결재완료 / 반려**. `DYV2.STATUS_TONE` 매핑(미상신 neutral · 결재중 info · 결재완료 success · 반려 danger).
- 노출 위치: 교육 카드·상세(교육별 상태) · 이수현황 대상자 행(개인별 상태) · 총괄 팝업 상단(최근 총괄 상신 상태).
- 칩 동작: **미상신**이면 칩 자리가 `[결재 상신]`(교육별)·`[개인 상신]`(개인별) 액션 버튼이고, **상신 이후**엔 상태 칩이며 클릭 시 **결재 상태 팝업**이 열린다.
- 결재 상태 팝업: 문서번호·상신일·결재선·상태 이력 + **시연 회신 버튼**(`[결재 완료 처리]`·`[반려 처리]`) + `[문서 다시 보기]`·`[재상신]`. 실제 연계 시 완료·반려는 온나라가 회신하며, 이 버튼은 프로토타입 시연용이다.
- 상신·상태 변경 시 목록이 자동 갱신된다(화면 모듈이 `EDUAPV.registerRefresh(render)` 등록).

### 7.3 결재선 (§5 그대로)

세 트리거 모두 §5의 결재선(기본 팀장→과장, 조직도로 편집)을 공유한다. 문서 종류가 달라도 결재란·개요표는 지정한 결재선을 그대로 따른다.

### 확장 여지 (미구현)

- 결재선 **기본값을 담양군 위임전결규칙에서 확정** — 현재 기본값(팀장 → 과장)은 관행 기준 제안값이다. 규칙 원문 확인 후 문서 종류별 전결 단계를 고정 기본값으로 반영할 것.
- 상신 이력 **목록 화면**(문서번호·상신일·상태를 한 곳에서 조회) — 현재는 대상별 상태 팝업에서만 이력 확인.
- 기타 교육 분류(특별·MSDS 등)별 별도 서식 — 현재는 총괄 ①의 종류 구분으로 흡수.
