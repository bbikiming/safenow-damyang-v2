## 1. 문서 개요

### 1.1 목적
담양군 중대재해 통합관리 시스템 v2의 **의견청취 화면(SCR-SBM-003)** 을 발주처/기획이 제안한 **1 메인메뉴 + 3탭 구조**로 개편하기 위한 기획안이다. 본 문서는 (a) 현재 단일 화면 구현 현황, (b) 목표 3탭 구조, (c) 요구사항·법령 매핑, (d) 변경 범위·난이도·예상 공수(갭 분석)를 정리하여, 개발 착수 전 발주처·기획 검토와 의사결정의 근거를 제공한다.

본 문서는 기획 관점으로 작성되며, 코드 표현·변수명은 지양하고 표·텍스트 와이어프레임 중심으로 기술한다.

### 1.2 대상 화면
- 화면 ID: **SCR-SBM-003 의견청취**
- 진입: `menu.html?m=opinion` (단일 진입점, 화면 내부에서 탭 전환)
- 현재: 통계박스 + 2개 섹션(의견 접수·처리 / 위원회·협의체)으로 구성된 **단일 화면**

### 1.3 근거
- **요구사항 원문(첨부 이미지)**: 종사자 의견청취 점검표 관리, 의견청취(제안 등) 접수, 조치결과 관리(추진계획/추진경과/추진결과)
- **법령**
  - 산업안전보건법 제24조 — 산업안전보건위원회(안건 부서이행 현황)
  - 산업안전보건법 제64조 — 도급 시 안전·보건 협의체 / 종사자 의견청취
  - 중대재해처벌법 시행령 제4조 제7호 — 종사자 의견 청취 및 개선 이행점검(반기 1회)
  - 중대재해처벌법 시행령 제4조 제9호 — 도급·용역·위탁 시 안전·보건 확보 기준·절차
- **유사 서비스 벤치마크**: 안전신문고 처리절차, EHS 인시던트 SaaS(상태배지·대시보드), 모바일 안전점검 앱(오프라인 동기화), 위원회 회의록 표준 양식 (8장 참조)
- **재사용 패턴**: 경영방침(SCR-SBM-001) psub-tabs 3탭 패턴 — 검증된 셸·서브탭·라우터 구조

## 2. 현황 요약

현재 의견청취는 **단일 화면**(통계박스 4개 + 섹션 2개)으로 구성되어 있다. 접수·처리 워크플로우와 위원회/협의체 회의록은 이미 동작하나, **점검표(체크리스트) CRUD**와 **조치결과 3단 구조(추진계획/추진경과/추진결과)** 가 없다.

### 2.1 현재 화면 구성
- **통계박스 4개**: 접수대기 / 점검중 / 개선조치중 / 완료 (상태별 실시간 카운트)
- **섹션 1 — 의견 접수·처리**: 리스트(의견·경로·접수일·담당·처리단계·액션) + 진행 스테퍼 + [+ 의견 등록]
- **섹션 2 — 위원회·협의체**: 2개 고정 행(산업안전보건위원회/분기, 협의체/월간) + 회의록 작성 + [결과 공유]

### 2.2 이미 구현된 것 / 없는 것

| 영역 | 세부 | 현재 상태 | 비고 |
|---|---|---|---|
| 통계박스 | 상태별 실시간 카운트(접수대기/점검중/개선조치중/완료) | **있음** | 대시보드 요약의 기반으로 재활용 가능 |
| 의견 접수 | 온라인·QR·오프라인·위원회 경로 등록 폼 | **있음** | 단, 내용·사진·접수일은 저장 안 됨(부분) |
| 의견 처리 | 담당 배정 + 조치방법 3분기(현장점검/개선조치/반려) | **있음** | 처리 의견 텍스트 미저장(부분) |
| 상태 플로우 | 접수대기→점검중→개선조치중→완료(+반려) 스테퍼 | **있음** | 점검중→다음 단계 전이 액션 부재(부분) |
| 개선조치 연계 | 처리 시 개선조치 자동 생성·연동칩 표시 | **있음** | 개선조치 메뉴에 자동 집계됨 |
| 위원회 회의록 | 회의록 폼(일시·장소·참석자·안건·의결사항·첨부) | **있음** | 표준 e-Doc 라이프사이클 사용 |
| 협의체 회의록 | 회의록 폼(위원회와 동일 폼) | **있음** | 월간인데 분기 키 사용 — 의미 불일치 |
| 결과 공유 | 게시+알림(메일) + 실적 토스트 | **부분(시뮬레이션)** | 실제 문서 인스턴스 생성 없음 |
| **조치결과 3단** | 추진계획 / 추진경과 / 추진결과 구조 | **없음** | 요구사항 필수 — 신규 |
| **점검표 CRUD** | 점검결과지 생성·등록·수정·삭제·조회 | **없음** | 요구사항 필수 — 신규 |
| **위원 그리드** | 근로자대표·사용자대표 등 조직도/현황 | **없음** | 인사/조직 연동 — 신규 |
| **협의체 운영관리** | 위원 구성·주기·용역/구매설치/위탁/기타 항목 | **없음** | 신규(단, 도급관리 연계 권고) |
| 점검중 후속 전이 | 점검중→개선조치중/완료 명시 액션 | **없음** | 부분 결함 |

## 3. 목표 구조

### 3.1 기본 방침
- **메뉴 뎁스를 무겁게 쪼개지 않는다**: 상위 메뉴는 "안전보건관리체계 > 의견청취" 1개를 유지하고, 화면 내부에서 **3개 탭**으로 콤팩트하게 전환한다.
- **상단 공통 대시보드 요약**: 모든 탭 위에 미니 통계 카드(올해 접수 건수 / 처리 완료율 / 미조치 건수)를 고정 배치하여 통계·리포트를 대체한다.
- **경영방침(policy) psub-tabs 패턴 재사용**: 검증된 셸·서브탭·라우터 골격을 그대로 복제한다.

### 3.2 경영방침 패턴 재사용 — 메뉴 구조 변경 부담이 낮은 이유
경영방침(SCR-SBM-001)은 이미 **공통 셸 + 서브탭 바(3탭) + 인메모리 라우터(목록↔상세↔폼·탭 전환)** 구조로 동작한다. 이 골격은 다음과 같이 의견청취에 그대로 재사용된다.

- **메뉴 골격 코드 무변경**: 타이틀·서브타이틀·요약바는 메뉴 메타데이터 한 곳에서 자동 생성되므로, 의견청취는 진입점·레이아웃 변경 없이 내부 뷰만 추가하면 된다.
- **"메뉴 구조 변경"이 아니라 "1메뉴 내부 뷰 분기"**: 3탭은 사이드바·URL을 늘리는 것이 아니라 화면 내부 서브탭이다. IA/메뉴 트리 변경 부담이 없다.
- **검증된 라우팅 패턴 복제**: 탭 전환 시 목록 리셋, 상세 진입 시 탭 보존 등 흐름이 경영방침에서 이미 동작 확인됨.
- **결론**: 메뉴 구조·라우팅·셸은 사실상 0 변경(재사용), 신규 작업은 의견청취 도메인 데이터·뷰 본문에 한정 → **메뉴 구조 변경 부담 = 낮음(Low)**.

### 3.3 목표 화면 흐름도 (텍스트 다이어그램)

```

* 탭 전환 시 목록 뷰로 리셋 / 상세 진입 시 현재 탭 보존
```

## 5. 탭별 상세 기획

### 5.0 공통 상단 — 대시보드 요약

**[구성 요소]** 미니 통계 카드 3~5종 (가로 정렬, 모든 탭 위 고정)
- 올해 의견청취 접수 건수 (절대수 + 전년 대비 추세)
- 처리 완료율 (% — 완료/전체)
- 미조치 건수 (지연 포함, 빨강 강조 분리)
- (확장) 이행점검 완료율 / 다가오는 위원회·협의체 일정

**[주요 기능]** 현재의 상태별 통계박스를 "연/완료율/미조치" 중심의 요약 카드로 재구성. 통계·리포트 화면을 별도로 두지 않고 이 요약 카드 + 탭별 현황으로 대체.

**[현재 대비 변경]** 기존 통계박스 4개(상태별)는 탭① 내부 현황으로 이동하고, 상단에는 경영 지표성 카드를 신설.

**[화면 와이어]**
```
┌─ 의견청취 ───────────────────────────────────────────┐
│ [올해 접수 27건] [처리 완료율 78%] [미조치 4건 ●]      │
├──────────────────────────────────────────────────────┤
│ ① 의견청취·건의함  ② 위원회 운영  ③ 협의체·점검표      │
└──────────────────────────────────────────────────────┘
```

### 5.1 탭① 의견청취 및 건의함 (접수/조치)

**[구성 요소]** 리스트형 게시판 + 상태 배지 + 등록 폼 + 상세(조치결과 3단)

**[주요 기능]**
- **상태배지 플로우 강조**: `접수 대기 → 검토 중 → 개선조치 연계 → 처리 완료` (+ 반려). 배지는 색 + 다음 책임자/기한을 함께 표기.
- **온·오프라인 병행 등록**: 제목 / 제안 내용 / 위치 / 현장 사진 첨부. 온라인·QR(근로자 직접) + 오프라인(담당자 대리 입력) 경로 선택.
- **조치결과 3단 구조(추진계획 / 추진경과 / 추진결과)**: 상세 화면에서 단계별로 입력·누적. 추진경과는 타임라인 형식으로 기록.
- **개선조치 연계**: 상세에서 [개선조치 관리] 버튼 → 개선조치 메뉴로 토스(데이터 연동 유지).

**[현재 대비 변경]**
- 등록 폼의 **내용·위치·사진을 실제 저장**하도록 수정(현재 제목·경로만 저장).

## 7. 데이터 모델 영향 (기획 수준)

현재 의견청취 데이터는 "의견 접수 건"(제목·경로·접수일·상태·담당·연계) 중심의 단순 구조다. 3탭 개편 시 다음 개념이 확장 필요하다(개념 중심, 코드 식별자 나열 안 함).

- **의견 건**: 내용 본문, 발생 위치, 현장 사진 첨부, 처리 이력(누가·언제·무엇을)을 보존하도록 확장.
- **조치결과(3단)**: 추진계획(목표·담당·기한), 추진경과(시점별 진행 기록 = 타임라인), 추진결과(완료 내용·증빙)로 분리.
- **위원회 위원**: 구분(근로자대표/사용자대표), 부서, 직책, 성명 — 인사/조직 연동 여부에 따라 마스터 출처 결정.
- **회의·출결**: 회차, 정기/수시, 일시·장소, 참석/결석, 안건, 의결사항, 부서이행 현황.
- **협의체 운영**: 위원 구성, 운영 주기, 운영 결과.
- **점검표/점검결과지**: 점검 구분(용역/구매설치/위탁사업/기타), 점검 문항, 문항별 결과(확인/미흡/해당없음), 첨부, 점검일, 점검자 — 생성·수정·삭제·조회 라이프사이클.
- **연계 키**: 의견·의결사항·점검 미흡 → 개선조치 백본으로 수렴(중처법 PDCA 기록 요건 충족).

## 9. 단계별 로드맵

### Phase 1 — MVP (재구성 우선)
- 경영방침 패턴 복제로 3탭 셸·서브탭·라우터 구성
- 상단 대시보드 요약 카드 (접수/완료율/미조치)
- 탭① 의견 게시판 + 상태배지 + 온/오프라인 등록(내용·위치·사진 저장) + 개선조치 연계(기존 재사용)
- 탭① **조치결과 3단(추진계획/경과/결과)** 신규
- 탭② 회의록 목록·아카이브(기존 회의록 폼 재사용)
- 산안법 §64 법령 사전 등록

### Phase 2 — 확장
- 탭② **위원 조직도 그리드** + 회의 출결·정족수 + 안건 부서이행 현황
- 탭③ **협의체 운영관리** + **점검표 체크리스트 + 점검결과지 CRUD**(용역/구매설치/위탁/기타)
- 회의록 의결사항·점검 미흡 → 개선조치 연계
- 도급관리 모듈 연계(점검표 재사용)
- 결과 공유 실적 실제 문서화, 회차 자동 증분 로직
