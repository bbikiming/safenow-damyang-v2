## 2. 검수 중 발견·조치 이슈

1. **(결함 → 수정 완료)** 상신 확인 팝업을 [확인]이 아닌 ESC·배경 클릭·닫기 버튼으로 닫으면 화면이 갱신되지 않아 상신 재클릭 시 결재 이력이 중복 누적 — 상신 가능 상태 제한 + 팝업 전 화면 선갱신으로 수정, 재검증 통과.
2. (에이전트 자가 수정) 대상 '사용 중' 뱃지 판정이 잘못된 식별자로 호출되던 버그 — 구현 단계에서 발견·수정됨, 검수에서 정상 확인.
3. (병렬 작업 과도기) 리팩터링 중간 시점의 일시적 스크립트 오류 — 완료 시점 코드에는 부재, 검수에서 정상 확인.

또한 시뮬레이션 단순화 1건을 기록해 둔다: **반려 후 재상신하면 이전 회차의 중간 결재(팀장·과장)를 인정**하고 최종 승인 수신 시 부군수 행만 추가된다. 프로토타입 시연에는 문제없으나, 정의서에는 "재상신 시 결재선 운영(재기안 여부)은 온나라 정책을 준용"으로 기술할 것.

## 3. 화면 정의서 전달성 — v1 인벤토리 대비 변경분 (델타)

v1 리포트 §3 인벤토리에서 아래만 교체·추가하면 정의서 작성 기준이 최신이 된다.

**용어·상태**
- 상태 명칭: 상신→**결재중**, 승인→**승인완료** (작성중·반려 유지). 상태 색: 작성중=회색 / 결재중=파랑 / 승인완료=초록 / 반려=빨강 — 경영방침 결재 칩과 동일 체계.
- 상태 전이: 작성중 → (상신) → 결재중 → (온나라 수신) → 승인완료 | 반려 → (재상신) → 결재중. **승인·반려는 시스템 조작이 아니라 온나라 수신값.**

**BGT01-V 상세 — 전자결재 카드 (전면 교체)**
- 공통 행: '온나라 결재' 라벨 + 상태 칩 + 문서번호(상신 후) + [이력] 버튼
- 작성중·반려: [온나라 결재 상신] → 확인 팝업(문서번호 자동 발급·결재선 팀장→과장→부군수 안내) → 결재중 전이. 재상신 시 문서번호 유지.
- 결재중: [결재 상태 조회] → 조회 모달(결재 진행 표 + 모의 수신 — 중간 승인/최종 승인/반려. 반려는 사유 필수 인라인 입력). 중간 승인 수신 시 모달 유지·표만 갱신, 최종·반려 수신 시 모달 닫고 화면 전이.
- 승인완료: 안내문(집행액·비고 상시 입력).
- [이력] 모달: '온나라 결재 이력 — {문서번호}', 4열(일시/처리/처리자/비고), 실제 결재 진행 기록을 동적 표시. 미상신 시 "결재 이력이 없습니다 — 상신 후 이력이 기록됩니다."
- 하단 카드 명칭: '결재 이력' → **'처리 이력'** (문서 생명주기 타임라인 — 온나라 결재 이력과 역할 구분).

**BGT01-V 생성 모달**
- 대상 목록 출처가 '점검 대상 관리' 마스터로 변경. 대상 0건 시: "예산 기준 설정 > 점검 대상 관리에서 대상을 먼저 등록하세요."
- KPI 카드 아이콘 없음(텍스트만).

**BGT02-S 섹션 C '점검 대상 관리' (신규)**
- 기관 블록 + 시설 블록. 행 = 이름 + 출처 칩(FMS/수기) + (점검표 사용 중 뱃지) + [삭제]
- [+ 등록] 인라인 입력(Enter 확정·Esc 취소). 유효성: 공백/중복명 차단.
- 시설 블록: [FMS 연계 동기화] + "마지막 동기화 {일시}". 동기화 = 신규만 추가·기존 유지, 결과 토스트 "FMS 시설 {N}건 동기화 — 신규 {M}건".
- 안내 노트: "…시설은 FMS(시설관리시스템) 연계로 가져오거나 직접 등록할 수 있으며, 점검표에서 사용 중인 대상은 삭제할 수 없습니다."

**메시지 카탈로그 추가분**
`작성중·반려 상태에서만 상신할 수 있습니다.` / `결재중 상태에서만 수신할 수 있습니다.` / `중간 결재가 이미 완료되었습니다. 최종 승인을 수신하세요.` / `{단계} 수신됨` / `최종 승인이 수신되었습니다.` / `반려가 수신되었습니다.` / `결재중에는 편집할 수 없습니다.` / `대상명을 입력하세요.` / `이미 등록된 대상입니다.` / `점검표에서 사용 중인 대상은 삭제할 수 없습니다.` / `대상이 등록되었습니다.` / `대상이 삭제되었습니다.`

**전면 삭제된 요소** (정의서에 남기면 안 됨): 상세의 [승인 처리/반려 처리 (시뮬레이션)] 버튼, KPI·첨부·팝업의 그림 이모지 전부, 기관·시설 하드코딩 목록.

## 4. 기획 순수성 검수 (AC9)

- **화면 노출 문구 전수 검사 통과** — 두 화면(목록·상세·생성 모달·조회 모달·이력 모달·설정 3섹션)의 표시 텍스트에서 구현 용어(스토리지·API·함수·클래스·파일명) 0건. 경계 표기는 "(연계 시뮬레이션)"·"(프로토타입)" 2종만 사용.
- **기획 완결성** — 모든 버튼이 "동작 → 결과 → 차단 메시지"의 기획 언어로 화면 안에서 설명됨. 예: "승인·반려 결과는 온나라에서 수신하며, 시스템에서 직접 처리하지 않습니다."(결재중 카드), "모의 수신 (온나라 연계 응답)"(조회 모달 — 시뮬레이션 경계를 화면이 스스로 설명).
- **문서 분리** — 화면 정의서는 본 리포트 §3(기획 언어)만 참조해 작성한다. §1의 근거 열·코드 스캔 등 구현 추적 정보는 정의서에 싣지 않는다.

## 5. 프로토타입 ↔ 실 구축 차이 (정의서 명시 대상 — v1 §4 갱신)

1. 결재 결과 수신: 프로토타입은 조회 모달의 '모의 수신' 버튼. 실 구축은 온나라 연계 자동 수신(또는 주기 조회)으로 대체.
2. FMS 동기화: 프로토타입은 수동 버튼 즉시 반영. 실 구축은 일 1회 배치(연계 관리 화면 기준) + 수동 동기화 병행.
3. 연도 범위(2025·2026 고정), 첨부 업로드(안내만), 점검표 삭제 확인 대화상자 — v1과 동일.
4. ~~대상 목록 하드코딩~~ — 본 개편으로 해소(시스템 등록 + FMS 연계).

## 3-2. 정의서 델타 (§3에 추가 반영할 것)

- 섹션 C 명칭·구조 교체: 블록 헤더 "조직도 연계"/"대상 관리(FMS) 연계", 자체 등록·동기화 요소 전면 삭제
- 선택 팝업 2종: '기관 선택 — 조직도'(부서명+구성원 수) / '시설물 선택 — 대상 관리 연계'(시설명+종별+안전점검 등급). 공통: 검색(부분일치), 이미 등록 항목 비활성 '등록됨', [선택 추가 (N건)] 실시간 카운터, 0건 차단, 추가 토스트 "{N}건이 추가되었습니다."
- 안내 노트: "점검 대상은 조직도(기관)와 대상 관리(시설물)에서 선택해 추가합니다. 시설물 현황은 대상 관리 메뉴에서 FMS 연계로 자동 현행화되며, 점검표에서 사용 중인 대상은 삭제할 수 없습니다."
- 삭제된 메시지: 수기 등록·중복 등록·FMS 동기화 관련 문구 전부 (v2 §3 메시지 카탈로그에서 제거)
- 추가 메시지: `추가할 대상을 선택하세요.` / `{N}건이 추가되었습니다.`

## 3-3. 미확정·연계 전제 (정의서 명시 대상)

1. **기관 대상 범위 — 고객사 확인 필요.** 프로토타입은 타 메뉴 공용 조직도의 부서 단위 기준. 확정 시 범위(전 부서/일부/사업소 포함 여부) 반영.
2. 시설 원본 = **대상 관리 메뉴(별도 구축 예정)**: FMS 시설물을 엑셀 데이터로 불러와 업데이트, 안전점검 등급·시설물안전법 1·2·3종 분류 자동 연계, FMS 변동 시 자동 현행화. 예산 화면은 그 결과를 조회·선택만 한다. 프로토타입의 시설 목록 9건은 이 연계 데이터의 표본이다.
