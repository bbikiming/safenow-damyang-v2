# 기획 — 위험성평가 재설계 v1

2026-07-14 담양군 인터뷰 반영. 기존 KOSHA식(공정→4×4 추정) 구조를 폐지하고
**용역업체 주도 정기평가 + 부서 단위 조치 + 보고서(hwpx) 기반** 흐름으로 재설계.

## 1. 메뉴 구조 (위험성평가 그룹)

| 메뉴 | 파일 | 비고 |
|---|---|---|
| 정기 위험성평가 | rsk-list.html / rsk-detail.html | 전면 개편 |
| 수시 위험성평가 | rsk-occ.html | 신규 |
| 내 할일 (부서) | rsk-my.html | 신규 — 부서 담당자 관점 |
| 개선조치 | rsk-imp.html | 유지 — 조치 데이터 **원본** |
| (숨김) 작업공정 관리 · 위험성 추정 | rsk-proc / rsk-exec | layout.js 메뉴에서 제거, 파일 보존 |

관점: 역할 전환 없음. 기존 관례대로 한 화면에서 모든 액션 노출.
부서 담당자 관점 액션만 '내 할일'로 분리 배치.

## 2. 정기 위험성평가

### 2.1 목록 (rsk-list)
- **연도 셀렉트박스** (정기평가는 1년 단위). 선택 연도의 평가 표시.
- [+정기평가 생성] → 대상지 선정 마법사 (단일 모달, STEP 교체 방식).

### 2.2 대상지 선정 (생성 마법사)
재난안전과 담당자 + 용역업체 논의 후 확정한다는 전제.
- STEP 1: 부서 체크박스 선택 — `DYV2.ORG` 파생 트리 재사용.
- STEP 2: 부서별 점검일자 — 일괄 입력 + 건별 조정.
- 생성 시: 해당 연도 정기평가 생성 + 선정 부서에 알림(점검예정일) + 이력 기록.

### 2.2.1 점검설문지 첨부 — 생성 이후 목록에서 (5차 피드백, 2026-07-20)
설문지 단계를 **마법사에서 제외**한다. 설문지 확정이 늦어지면 부서 선정·점검일자 통보까지 함께 막히므로,
평가를 먼저 만들어 일정을 통보하고 설문지는 나중에 붙인다.
- **공통본** — 요약 카드 `공통 설문지` [＋첨부]/[변경]/[삭제]. 부서별본이 없는 모든 부서에 적용되며,
  요약 카드에 적용 진행률(`부서 적용 N/M`)을 함께 표시한다.
- **부서별본** — 부서별 조치 표의 `설문지` 열에서 [＋설문지 첨부]/[부서별]/[교체]/[×].
  부서별본이 있으면 그 부서만 공통본 대신 사용한다(표시 우선순위: 부서별본 > 공통본 > 미첨부).
- 첨부 모달의 드롭존은 `DYV2.uploadDrop(label, onAct, {hint:true})`만 사용한다(CLAUDE.md §2).
- 완료(COMPLETED)된 평가는 첨부·삭제 버튼을 숨긴다(이력 왜곡 방지).
- 데이터 창구는 `DYRSK.setSurveyAll(aid, name)` · `setDeptSurvey(aid, deptId, name)` · `surveyProgress(aid)`.
  두 함수 모두 `type:'FILE'` 이력을 남긴다.

### 2.3 상세 (rsk-detail)
단계 프로그레스 **없음**. 상단은 연도·전체 완료율 요약만.

부서별 테이블: 부서 | 점검일 | 설문지 | 개선건수 **N/M** | 조치기한 | 상태(**조치전/조치완료** 2단계) | 관리

- **보고서 업로드**: 용역업체 위험성평가 보고서(hwpx) 첨부.
- **데이터 입력**: 부서별로 유해위험요인(요인명·분류·원인) + 개선조치사항 행 단위 입력.
- **검토완료 → 전달**: 부서 단위 일괄 조치기한 입력 + 건별 조정
  → `DYRSK.improvements`에 건별 개선조치 생성(assessment_id·dept_id 링크) + 부서 알림.
- **재촉**: 기한초과 건 [재촉] → 부서 알림. 부서 담당자는 내 할일에서
  **사유 입력 + 처리기한 수정** 후 응답. 전 과정 history 기록.
- **이력 탭**: 알림·전달·재촉·사유·기한변경·상태변경 타임라인.
- 전 부서 조치완료 시 평가 상태 자동 '완료'.

### 2.4 조치 진행
원본은 개선조치(rsk-imp) 메뉴. 정기 상세는 assessment_id로 불러와 N/M 집계 표시.

## 3. 수시 위험성평가 (rsk-occ, 신규)
- **연도 셀렉트박스** + 사유 필터(재해사고 발생 / 설비·물질 변경 / 기타).
- [+등록]: 부서 · 사유 · 발생일 · 내용 · 첨부파일 → 상태 '등록됨' + 재난안전과 알림.
- 재난안전과 [검토완료] → 상태 '검토완료'. 이력 기록.

## 4. 내 할일 — 부서 담당자 (rsk-my, 신규)
상단에 부서 셀렉트(프로토타입용 관점 전환). 알림 드롭다운 항목은 이 페이지로 링크.
- **점검 예정 카드**: 평가명 · 점검일 · [점검설문지 다운로드].
- **조치할 사항 카드**: 전달받은 개선조치 목록(N/M) — 요인명 · 개선조치사항 · 기한 · [완료 처리].
- **재촉 응답**: 재촉받은 건 강조 표시 → 인라인 폼으로 **사유 입력 + 처리기한 수정** → 이력 기록.
- 수시평가 등록 진입 버튼(해당 부서로 프리필).

## 5. 데이터 모델 (DYRSK 개편 요지)

```
assessment  { id, year, type:'REGULAR', status, files:{ surveyAll, report },
              depts:[{ deptId, inspectDate, surveyFile, status:'BEFORE'|'DONE',
                       deliveredAt, dueDate }],
              history:[{type,at,by,memo}] }
improvement { ..기존 스키마.. + assessment_id, dept_id,
              hazard:{name, category, cause}, action, due, status,
              history:[{type:'NOTIFY'|'REMIND'|'REASON'|'DUE_CHANGE'|'STATUS', at, by, memo}] }
occasional  { id, year, deptId, reason:'ACCIDENT'|'EQUIP_CHANGE'|'OTHER',
              date, desc, files, status:'REGISTERED'|'REVIEWED', history:[...] }
```

- 부서는 `DYV2.ORG` deptId 참조 (자체 조직 데이터 금지).
- 모달은 전부 `DYV2.openModal` 단일 모달 + 인라인 패널(CLAUDE.md 규칙).
- 첨부파일 안내는 `DYV2.fileHint()` 단일 출처.

## 6. v1.1 변경 (2026-07-14 2차 피드백)

### 6.1 정기 위험성평가 목록·상세 통합
정기평가는 연 1건 → 목록→상세 이동 없이 **rsk-list에서 연도 셀렉트 + 해당 연도 상세를 바로 표시**.
rsk-detail의 부서 테이블·보고서·이력 UI를 rsk-list로 통합, 해당 연도 평가 없으면 [+정기평가 생성] 빈 상태.
rsk-detail.html은 리다이렉트 스텁 처리.

### 6.2 내 할일 전사화 (rsk-my → 내 할일 전역 메뉴)
위험성평가 전용이 아닌 **전 업무 통합 내 할일**로 승격. 레퍼런스: v1 프로토타입 myw01-v.
- 상단 KPI 칩: 지연 / 오늘 마감 / 이번주 / 결재 대기 / 전체 (+동기화 시각 표시)
- 필터: 상태(전체/지연/진행/예정) · 카테고리(결재/위험/개선/점검/의견/이행/도급/평가) · 정렬(마감 임박순 등)
- 그룹핑: 🔴 지연 / 🟡 이번주 / 🟢 예정 / 📌 기타 — 항목 = 카테고리 배지 + 제목 + D-day + 부서 + 액션 링크
- 위험성평가 항목 편입: 점검예정(설문지 다운로드)=카테고리 '위험', 전달받은 조치(N/M)·재촉 응답(사유+기한수정)=카테고리 '개선'
- 타 도메인(결재·점검·의견·이행·도급)은 목업 시드로 채움
- 메뉴 위치: 위험성평가 그룹에서 빼서 전역(상위) 메뉴로

### 6.3 용역업체 보고서 반영 흐름 (수동 입력 → 파싱·검수)
초기 정기평가에는 부서별 조치 내용이 **없음**. 수동 '데이터 입력' 모달 폐기.
1. 대상지 선정 → 용역업체+부서 점검 진행 (부서별 조치내용 비어 있음)
2. 점검 완료 후 보고서(hwpx) 업로드 → **자동 파싱(목업)**으로 부서별 유해위험요인·개선조치사항 추출
3. 검수 화면: 추출 항목별로 "이 항목·이 내용이 맞는지" 확인 — 항목 확인/수정/삭제/추가
4. 검토완료 → 조치기한 설정(일괄 + 건별 조정) → 부서별 자동 전달(improvements 생성 + 알림 + 이력)
5. 이후 부서 조치내용 확인·N/M 집계 (기존과 동일)

### 6.4 대상 부서 선정 — 인사정보시스템 연동 (3차 피드백, 4차 정정)
**대상자 = 대상 부서** (직원 개인 선택 아님 — 직원 단위 선택으로 구현된 4-STEP·targets는 되돌림).
마법사는 **2-STEP** (부서 선정 → 점검일자). ※ 구 3-STEP 의 설문지 단계는 §2.2.1 로 이관(2026-07-20).
- STEP1 부서 선정: **인사정보시스템(조직정보) 연동 목업**으로 부서 목록 로드 — "🔗 인사정보시스템 연동" 라벨 표시. 데이터는 `DYV2.ORG` 파생(자체 조직 데이터 금지).
- 부서 체크박스 **기본값 전체선택**, 전체선택/해제 토글 제공.

## 7. 결정 이력
- 기존 공정·4×4 화면: 메뉴에서 숨기고 파일 보존.
- 개선조치 원본은 rsk-imp, 정기평가는 불러오기(집계).
- 조치기한: 일괄 + 건별 조정.
- 수시평가: 등록 후 재난안전과 검토.
- 부서 담당자 화면: 별도 '내 할일' 메뉴 (알림은 링크만).
- 정기평가 단계 표시 제거, 부서 상태는 조치전/조치완료 + N/M.
