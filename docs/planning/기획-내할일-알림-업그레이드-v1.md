# 기획 — 내 할일·알림 UX 업그레이드 (v1, 2026-07-21)

원칙: **"지금 할 수 있는 건 내 할일 카드에서 끝내고, 화면이 필요한 건 처리 위치로 보낸다."**
발행된 업무는 내 할일에 모이고, 항목마다 처리 유형이 정해져 있어 사용자가 다음 행동을 고민하지 않게 한다.

## 1. 내 할일 (my-work.html · js/my-work.js MYWORK)

### 화면 구조
- 상단 KPI 칩 5개는 **클릭 = 필터** (지연/오늘 마감/이번주/결재 대기/전체 해제). 활성 칩은 포커스 링 강조.
- 뷰 탭 2개 (표준 `.tabs`):
  - **마감 할일** — 기한 그룹(지연/이번주/예정/기타/**완료**). 완료 그룹 신설로 DONE 항목 렌더 크래시(기존 버그) 해결.
  - **발행된 업무** — 업무 목록(`DY_DOCS_V2`, `js/sets-data-v2.js`)에서 현재 부서로 발행된 담당 문서. 처리유형 칩 필터(전체/바로 첨부/양식 작성/메뉴 진행) + 상태 셀렉트(기본 미완료만).

### 처리 유형 (atype) — 항목별 액션 단일화
| 유형 | 액션 | 동작 |
|---|---|---|
| `attach` 파일 첨부형 | [파일 첨부] (primary) | 카드 안 인라인 패널(`DYV2.uploadDrop`+`fileHint`) → 모의 파일 선택(필 목록·제거 가능) → [첨부 완료] → 완료 그룹 이동 / 발행 업무는 `DY_DOCS_V2.status='완료'` 반영 |
| `menu` 메뉴 진행형 | [~하러 가기 →] (outline) | 해당 화면 딥링크. 메타 줄에 **처리 위치 칩**("이행관리에서 진행") 상시 표기 |
| `download` | [설문지 다운] | 즉시 다운로드 (프로토타입 토스트) |
| `inline` | 재촉 응답 / 완료 처리 | 기존 인라인 폼·단일 모달 유지 (위험성평가 개선조치) |

- 발행 업무 처리유형 매핑: 첨부파일→attach / 전자문서→업무 목록에서 작성(딥링크) / 프로그램→`MENU_HREF_V2[menuKey]` 딥링크(라벨은 `DY_MENUS_V2`).
- 그룹 헤더 이모지(🔴🟡🟢📌)·🔔 제거 → 토큰 색점·SVG 아이콘(주석 참조 Lucide 계열 stroke 2).
- 딥링크 유지·확장: `?dept=` `?cat=` + `?due=over|today|week`(대시보드 statbox 연동) + `?view=pub`.

## 2. 알림 (js/layout.js — 헤더 드롭다운)

- 항목을 `NTF_ITEMS` 데이터 배열로 단일화 — 배지 수·"안 읽음 N건"·목록이 전부 파생.
- **읽음 처리**: 항목 클릭/Enter 시 읽음 저장(sessionStorage `dy-ntf-read-v1`) 후 이동, [모두 읽음] 실동작(배지 0·버튼 비활성).
- **분류 칩 필터**: 전체/결재/지정/이행/점검/위험 — 목록 인플레이스 필터.
- **내 할일 연결**: `work` 링크가 있는 알림은 "내 할일에서 처리 →" 바로가기 노출(해당 필터가 걸린 my-work 딥링크).
- 푸터 2링크: [내 할일 열기] | [알림 전체 보기].
- 항목이 `<a>`→`role="link"` div 로 바뀌어 커서·포커스 링을 CSS(`css/style.css` 알림 개편 블록)에서 명시.

## 3. 프로토타입 한계
- 파일 첨부는 모의 흐름(파일명 시드) — 실제 input 배선 없음. 지원 형식·용량 안내는 `DYV2.fileHint()` 단일 출처.
- 발행 업무 상태 변경은 메모리 반영(새로고침 시 초기화). `DY_DOCS_V2`의 부서명 중 조직도(`DYV2.ORG`)에 없는 부서(도시과·행정과·안전건설과)는 내 할일 부서 후보에 나타나지 않는다.
- 알림 읽음 상태는 세션 단위(sessionStorage) — 시연 리셋은 새 탭/브라우저 재시작.
