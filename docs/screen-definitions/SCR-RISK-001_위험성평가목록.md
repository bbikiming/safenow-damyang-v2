- 완료된 평가는 보고서로 열람하며, 진행 중 평가는 상세로 이동해 공정별 진행을 확인한다.

## 4. 화면 구성별 정리

### 4-1. 상단 도구모음
- 구성: 관리대상 선택기, 소속 업종 표기, [정기 평가]·[수시 평가].
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [정기 평가] | 현재 공정 전체 대상 평가 생성 | 상세로 이동. 공정 없으면 "작업공정을 먼저 등록하세요" |
| [수시 평가] | 변경 사유·변경 공정 선택 요청 | 선택 후 생성, 상세로 이동 |

### 4-2. 필터
- 구성: 연도·유형(정기/수시)·상태(진행중/완료) 선택.
- 상태: 조건에 맞는 항목이 없으면 빈 안내.

### 4-3. 연도별 목록
- 구성: 연도 그룹 아래 각 평가의 유형 배지·평가명(수시는 변경 사유 병기)·상태·공정 진행 집계(완료/진행/전)·완료일.
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [상세] | 진행 상세로 이동 | rsk-detail.html |
| [보고서] | 완료 보고서로 이동 | 완료 단계 후속 안내 |

### 4-4. 수시 평가 생성
- 구성: 변경 사유 입력, 변경 공정 다중 선택.
- 상태: 변경 공정이 없으면 생성 불가 안내. 전체 재평가가 필요하면 정기 평가로 안내.

## 6. 정책

- **정기/수시 구분:** 정기는 현재 공정 전체를 대상으로, 수시는 변경된 공정만을 대상으로 한다. 연도별 이력을 보존한다. (근거: 산업안전보건법 제36조, 위험성평가 고시)
- **선행 조건:** 작업공정이 등록되어 있어야 평가를 생성할 수 있다.
- **수시 자동 연계:** 작업공정 관리에서 설비·작업방법 변경 반영 시 생성된 수시 평가가 이 목록에 나타난다.
