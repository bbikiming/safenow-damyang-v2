## 4. 화면 구성별 정리

### 4-1. 평가 목록 표
- 구성: 정기·수시·최초 평가를 한 표로 표시. 컬럼 — 평가명(유형 구분 + 출처·발생사유 표시), 관리대상, 진행 상태(사전준비/진행중/완료/반려), 기간(기한 초과 강조), 담당부서, 담당자, 결재 상태(미요청/결재 진행중/완료/반려).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 평가 행 클릭 | 해당 평가의 회차 진행(상세) 화면으로 이동 | risk-regular.html |
| 방금 제출된 평가 행 클릭 | 체크리스트 평가 화면으로 이동 | risk-checklist.html |

- 상태: 기한 초과 평가는 강조 표시. 행 종류에 따라 이동 목적지가 달라짐(§6·§7).

### 4-2. 평가 등록 진입
| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [정기평가 생성] | 정기 위험성평가 생성 화면으로 이동 | risk-create.html |
| [수시평가 등록] | 수시 위험성평가 등록 폼으로 이동 | risk-form.html |

- 상태: 같은 연도에 정기평가가 이미 있어도 중복 생성 차단이 없음 — 중복 방지 정책 미정(§6·§7).

### 4-3. 연도 정기평가 진입 배너
- 구성: 올해 정기 위험성평가의 유해·위험요인별 진행 현황(완료·진행률) 요약.
| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [차수 열기] | 올해 정기평가 회차 진행 화면으로 이동 | risk-regular.html |

### 4-4. 검색 · 필터 · 페이징
- 구성: 부서·관리대상·유형·상태·기간 필터, 평가명 검색, 페이지 이동.
| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 검색 / 필터 / 페이지 이동 | 현재 화면에 배치되어 있으나 동작 규칙 미확정 | 검색 대상·필터 조합·정렬·페이지 기준 §6·§7 |

- 상태: 검색 결과 0건·빈 목록·로딩·오류 상태의 표시는 미정(§6·§7).

### 4-5. 방금 제출 평가 강조
| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 체크리스트 평가 제출 후 복귀 | 제출 평가를 목록 상단에 강조 표시하고 해당 행으로 이동 | "평가가 정상 제출되어 목록에 추가되었습니다(결재 진행 중)" 안내 |

## 6. 정책

- **검색·필터·정렬·페이징** — 검색 대상 필드(평가명만/담당자·대상 포함), 필터 조합 방식, 컬럼 정렬, 페이지당 행 수·기본 정렬·정렬 방향의 동작 규칙 확정 필요(§7).
- **행 클릭 상세 진입 일관성** — 일반 평가 행과 방금 제출 평가 행의 이동 목적지가 다름. 평가 유형·방법별 상세 진입 규칙을 일관되게 확정 필요(§7).
- **진행 상태 ↔ 결재 상태 매핑** — 목록의 진행 상태와 결재 상태의 관계, 표준 상태값 정의 필요(§7).
- **정기평가 중복 생성 방지** — 동일 연도 정기평가 중복 생성 차단·경고 정책 확정 필요(§7).
- **빈/0건/오류 상태** — 전체 빈 목록·검색 결과 0건·데이터 로드 오류의 표시 문구·UI 확정 필요(§7).
- **제출 데이터 영속·결재 연동** — 제출된 평가의 저장·결재 워크플로 연동, 제출 후 상태 갱신 규칙 확정 필요(§7).
