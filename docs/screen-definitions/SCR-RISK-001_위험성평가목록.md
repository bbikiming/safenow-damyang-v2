**목적·설명** — 정기·수시·최초 위험성평가를 한 표로 모아 조회하고, 각 평가의 진행 상태·결재 상태·기한·담당부서·담당자를 한눈에 파악하는 화면이다. 여기서 ① 새 정기평가 생성과 수시평가 등록으로 진입하고, ② 올해 정기평가의 회차(차수) 진행 화면으로 들어가며, ③ 개별 평가 행을 눌러 해당 평가의 상세(회차 진행)로 이동한다. 평가명 셀은 유형 칩(정기/수시/최초)과 함께 출처(정기 자동·최초 자동·e호조 자동·수동)·수시 라벨·발생사유(사고·중대변경·민원·기타)를 함께 표시해 평가가 어떤 경로로 만들어졌는지 구분한다. 체크리스트 평가를 제출하고 이 목록으로 돌아오면 방금 제출한 평가가 목록 맨 위에 강조되며, 본 화면은 위험성평가 순차 이행 제품 투어의 한 단계(정기평가 생성 버튼 안내)로도 사용된다. 연관: SCR-RISK-002(정기평가 생성), SCR-RISK-003(수시평가 등록 폼), SCR-RISK-004(정기평가 회차 진행), SCR-RISK-005(체크리스트 평가).

> 화면명("위험성평가 목록")과 실제 노출 제목("평가 목록")이 다르다 — 정식 표기 통일은 §7(RISK-09).

## 4. 화면 구성별 정리

### 4-1. 페이지 헤더(제목 + 등록 버튼)
- 구성: 좌측에 제목 "평가 목록"과 부제 "정기·수시·최초 위험성평가 등록 및 진행 관리", 우측 한 줄에 [정기평가 생성]·[수시평가 등록] 두 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [정기평가 생성] | 정기 위험성평가 생성 화면으로 이동 | risk-create.html (SCR-RISK-002) |
| [수시평가 등록] | 수시 위험성평가 등록 폼으로 이동 | risk-form.html (SCR-RISK-003) |

- 상태: 두 버튼은 항상 노출(역할별·연도별 노출 차이 없음). 같은 연도에 정기평가가 이미 있어도 중복 생성 차단·경고가 없다 — 중복 방지 정책 미정(§7 RISK-06). 제품 투어 진행 시 [정기평가 생성] 버튼이 강조 대상이 된다.

### 4-2. 연도 정기평가 진입 배너
- 구성: 표 위에 정보색 배너 한 줄. 좌측에 "정기" 칩 + 안내 문구("올해 정기 위험성평가 — 유해·위험요인별 평가 진행 현황: 평가 건수, 검토완료 건수, 진행률"), 우측에 [차수 열기]. 배너 전체가 링크로 동작한다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 배너 클릭 / [차수 열기] | 올해 정기평가 회차 진행 화면으로 이동 | risk-regular.html (SCR-RISK-004) |

- 상태: 배너 안의 건수·검토완료·진행률 문구는 현재 고정 문구로 표시된다(목록 데이터와 연동되지 않음). 실제 진행 현황과의 연동 여부는 §7(RISK-07).

### 4-3. 필터 · 검색 행
- 구성: 한 줄에 필터 셀렉트 5종 + 평가명 검색바 + [검색] + [초기화].
  - **부서**: 전체 / 건설과 / 공공시설사업소 / 환경과 / 문화체육과 / 물순환사업소
  - **관리대상**: 전체 / 군청 청사 / 군민체육관 / 담양읍 행정복지센터 / 물순환사업소 본관 / 환경자원시설 / 죽녹원
  - **유형**: 전체 / 정기 / 최초 / 수시
  - **상태**: 전체 / 사전준비 / 진행중 / 완료 (※ 표 데이터에는 "반려" 행이 있으나 상태 필터에는 "반려" 옵션이 없다)
  - **기간**: 전체 / 진행중 / 올해 / 지난 6개월
  - **검색바**: 입력란 안내문 "평가명 검색 (0.3초 자동 검색)" + [검색] 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 필터 5종 선택 변경 | (현재 미연결) 화면에 배치만 되어 있고 목록을 걸러내는 동작은 아직 연결되지 않음 | — |
| 검색어 입력 / [검색] | (현재 미연결) 안내문은 0.3초 자동 검색을 명시하나 실제 검색 동작은 아직 연결되지 않음 | — |
| [초기화] | (현재 미구현) 필터를 실제로 비우지 않고 "준비 중" 안내만 표시 | "준비 중" 토스트 |

- 상태: 필터·검색·초기화 모두 현재 동작이 연결되지 않은 상태다. 검색 대상 필드, 필터 조합 방식, 0.3초 자동 검색 적용 여부, 상태 필터에 "반려" 추가 여부는 §7(RISK-02, RISK-03)에서 확정한다. 검색 결과 0건·빈 목록·로딩·오류 전용 표시는 정의되어 있지 않다(§7 RISK-08).

### 4-4. 평가 목록 표
- 구성: 정기·수시·최초 평가를 한 표로 표시. 컬럼 7개 — **평가명 / 관리대상 / 상태 / 기간 / 담당부서 / 담당자 / 결재**.
  - **평가명 셀**: 좌측 유형 칩(정기 / 수시 / 최초) + 본문(평가명) + 메타 태그. 메타 태그는 ① 출처 태그(정기 자동 · 최초 자동 · e호조 자동 · 수동 — "e호조 자동"은 외부 e호조 시스템 연계로 생성된 평가, 나머지 자동은 시스템 자동 생성, "수동"은 사용자가 직접 등록), ② 수시 라벨(수시 / 최초), ③ 발생사유 태그(사고 · 중대변경 · 민원 · 기타 — 수시평가가 만들어진 계기).
  - **관리대상 셀**: 대상명 + 대상 유형 태그(시설 / 업무 / 사업).
  - **상태 컬럼**: 사전준비 / 진행중 / 완료 / 반려(4종).
  - **기간 컬럼**: 마감일 날짜 표시. 기한을 넘긴 평가는 경고 아이콘과 함께 강조 표시.
  - **담당부서 / 담당자 컬럼**: 텍스트.
  - **결재 컬럼**: 미요청 / 결재 진행중 / 완료 / 반려(4종) — 진행 상태와 별개의 결재 상태값을 표시.
  - 기한 초과 평가는 행 배경 전체를 경고색으로 강조한다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 일반 평가 행 클릭(정기·수시·최초 모두 동일) | 해당 평가의 회차 진행(상세) 화면으로 이동하며 평가명·상태·유형·담당부서·담당자를 함께 전달 | risk-regular.html (SCR-RISK-004) |
| 방금 제출된 평가 행 클릭 | 체크리스트 평가 화면으로 이동 | risk-checklist.html (SCR-RISK-005) |
| 행 내부 링크·버튼 클릭 | 행 이동을 일으키지 않고 해당 요소의 동작만 수행 | — |

- 상태: 기한 초과 평가는 기간 셀과 행 전체를 강조한다. 현재 일반 행은 유형(정기/수시/최초)과 무관하게 모두 회차 진행(risk-regular.html)으로 이동하고, 방금 제출 행만 체크리스트(risk-checklist.html)로 이동한다 — 평가 유형·방법별 상세 진입 목적지의 일관화는 §7(RISK-04). 진행 상태(4종)와 결재 상태(4종)의 표준값·상호 관계 정의는 §7(RISK-05). 데이터 0건 전용 빈 화면 문구·UI는 없다(§7 RISK-08).

### 4-5. 방금 제출 평가 강조
- 구성: 체크리스트 평가를 제출하고 복귀하면, 제출 이력을 읽어 목록 맨 위에 최신순으로 누적해 보여 준다. 제출 행은 유형 칩 + 평가명 + (직전 제출 시 "방금 제출됨" 뱃지) + 메타(처리방법 라벨, 점검 통계 "총 N건 · O n · X n · NA n", 제출일시)를 표시하고, 관리대상·상태·기간·담당부서·담당자·결재 컬럼도 제출 데이터로 채운다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 체크리스트 평가 제출 후 복귀(방금 제출) | 제출 평가를 목록 맨 위에 추가·강조하고 해당 행으로 부드럽게 스크롤, 한 번만 강조(이후 새로고침 시 재강조하지 않음) | "✓ 평가가 정상 제출되어 목록에 추가되었습니다. (결재 진행 중)" 안내(약 3.6초) |
| 제출 평가 행 클릭 | 체크리스트 평가 화면으로 이동 | risk-checklist.html (SCR-RISK-005) |

- 상태: 제출 이력이 없으면 상단에 아무 행도 추가하지 않는다(별도 빈 안내 없음). 제출 행의 상태·결재 값은 제출 데이터가 정하고, "방금 제출됨" 강조는 복귀 직후 1회성이다. 제출 평가의 실제 저장 영속·결재 워크플로 연동·제출 후 상태 갱신 규칙은 §7(RISK-08·정책 연동)에서 확정한다.

## 6. 정책

- **검색·필터·정렬·페이징 동작** — 현재 필터 5종·검색·초기화·페이지 이동이 모두 화면에 배치만 되어 있고 동작이 연결되지 않았다. 검색 대상 필드(평가명만 / 담당자·대상 포함), 필터 조합 방식, 0.3초 자동 검색 적용, 컬럼 정렬 제공 여부, 페이지당 행 수·기본 정렬을 확정해야 한다(§7 RISK-02·RISK-03).
- **상태 필터 옵션 정합성** — 표 데이터에는 "반려" 평가가 존재하나 상태 필터 옵션에는 "반려"가 없다. 상태 필터 옵션과 실제 상태값(4종)을 일치시킬지 정해야 한다(§7 RISK-03).
- **행 클릭 상세 진입 일관성** — 현재 일반 행은 유형 무관하게 모두 회차 진행(risk-regular.html)으로, 방금 제출 행만 체크리스트(risk-checklist.html)로 이동한다. 정기/수시/최초·평가 방법별 상세 진입 목적지를 일관 규칙으로 정해야 한다(§7 RISK-04).
- **진행 상태 ↔ 결재 상태 표준값·매핑** — 진행 상태 4종(사전준비/진행중/완료/반려)과 결재 상태 4종(미요청/결재 진행중/완료/반려)의 표준값과 둘 사이 관계를 정의해야 한다(§7 RISK-05).
- **정기평가 중복 생성 방지** — 동일 연도 정기평가가 있어도 [정기평가 생성]에 중복 차단·경고가 없다. 차단·경고 정책을 정해야 한다(§7 RISK-06).
- **연도 정기평가 배너 수치 출처** — 배너의 건수·검토완료·진행률이 고정 문구로 표시된다. 실제 진행 현황과 연동할지 정해야 한다(§7 RISK-07).
- **빈/0건/오류 상태** — 전체 빈 목록·검색 결과 0건·데이터 로드 오류의 표시 문구·UI가 정의되어 있지 않다. 표준 안내를 정해야 한다(§7 RISK-08).
- **제출 데이터 영속·결재 연동** — 방금 제출 평가의 실제 저장 방식, 결재 워크플로 연동, 제출 후 상태 갱신 규칙을 정해야 한다(§7 RISK-08).
- **화면명 표기 통일** — 화면명("위험성평가 목록")과 노출 제목("평가 목록")이 다르다. 정식 표기를 통일해야 한다(§7 RISK-09).
