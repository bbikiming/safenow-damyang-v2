## 4. 화면 구성별 정리

### 4-1. 도구모음
- 구성: 연도 선택, 사유 필터(전체 + 3사유), [수시평가 등록].

### 4-2. 목록 표
- 구성: 발생일 · 사유(색상 배지) · 부서 · 내용·첨부 · 상태 · 관리.
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [이력] | 등록·검토 타임라인 모달 | — |
| [검토완료] (미검토 건) | 상태를 검토완료로 전환, 검토일 기록 | "검토완료" 처리 |

- 상태: 사유는 재해사고 발생·설비물질 변경·기타로 구분해 색으로 강조한다.

### 4-3. 등록 모달
- 구성: 부서(조직도 단일 선택), 사유, 발생일(기본 오늘), 내용, 첨부파일.
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [조직도] | 부서 인라인 조직도에서 부서 1개 선택 | 부서명 표시·부서 식별자 저장 |
| [파일 첨부] | 첨부 파일명 추가(프로토타입) | 지원 형식·용량 안내 표기 |
| [등록] | 수시평가 등록 | "수시평가 등록 · 재난안전과 알림" |

### 4-4. 상태 어휘
- 등록됨(등록 직후) / 검토완료(재난안전과 검토 후, 검토일 표기).

## 6. 정책

- **정기와 구분:** 정기평가는 연 1건·부서 다건·보고서 검수·개선조치 자동 생성 흐름인 반면, 수시평가는 사유별 다건·부서 단건·검토완료 단순 흐름이며 개선조치는 자동 생성하지 않는다.
- **후속 개선조치:** 수시평가에서 파생되는 개선조치는 개선조치 화면(rsk-imp.html)에서 수동으로 등록·관리한다.
- **실시 사유:** 재해사고 발생, 설비·물질 변경, 그 밖의 사유(작업방법 변경 등)를 실시 대상으로 한다. (근거: 사업장 위험성평가에 관한 지침 — 수시평가 실시 사유)
