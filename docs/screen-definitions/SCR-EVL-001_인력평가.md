# 화면 정의서 — 인력 평가 (대메뉴)

> 화면 ID: SCR-EVL-001 ~ 003 · 화면 유형: 목록·등록(평가 실행) / 대시보드(현황) / 목록·수정(설정) · 기획 관점(v6)
> 확정이 필요한 항목은 §7 미결사항에 모았습니다.
> v1.2: 프로토타입 대조 검수 반영 — 결재상신 문서(부서 단위 결과 보고)와 개인 평가표 문서를 구분, 매트릭스 결과 칩·템플릿 링크·배점 검증의 현재 동작을 현행화, 평가자 재지정·항목 마스터 정본 등 화면-고유 미결을 §7에 추가. §6 중 "(개발 반영)" 표기는 확정 기획이나 현재 프로토타입에 아직 반영되지 않은 항목이다.

## 1. 화면 정보

| 항목 | 내용 |
|---|---|
| 화면 ID | SCR-EVL-001(인력 평가) · SCR-EVL-002(평가 현황) · SCR-EVL-003(평가 설정) |
| 화면명 | 인력 평가 |
| URL 경로 | evl-eval.html / evl-list.html / evl-settings.html |
| 상위 메뉴 | 인력 평가(대메뉴) — 좌측 메뉴(SNB)에 하위 3개 화면 |
| 화면 유형 | 목록·등록(평가 실행) / 대시보드(현황) / 목록·수정(설정) |
| 관련 SFR | SFR-009 (기존 화면 코드: EVL02-E / EVL01-V / EVL03-S) |

**목적·설명** — 안전보건관리책임자와 관리감독자가 안전보건 관계 법령에 따른 업무를 충실히 수행하는지 **반기 1회**(상반기 4~6월 / 하반기 10~12월) 평가하고(왜: 중대재해처벌법 시행령 제4조제1항제5호 · 산업안전보건법 제15조/제16조 이행), 결과를 온나라 전자결재로 상신하는 업무를 지원한다. 대메뉴 하위 3개 화면으로 구성되며, 결재 상신은 온나라 전자결재와, 평가 항목 근거는 의견청취(SCR-OPN-001)·위험성평가 등 시스템 내 운영 실적과 연계된다.

- **인력 평가**: 부서별 피평가자 매트릭스에서 평가자를 지정하고, 항목별 평가(양호/보통/미흡)를 등록하며, 부서 단위 평가 결과 보고를 결재 문서로 상신한다.
- **평가 현황**: 평가 사이클(연도·반기) 기준 진행 현황을 KPI·역할별·부서별로 집계해 마감 관리를 지원한다.
- **평가 설정**: 역할별 평가 항목 마스터(항목명·평가 기준·배점)를 관리한다. 변경은 다음 사이클부터 적용된다.

| 역할 | 접근(조회) | 주요 기능 | 비고 |
|---|---|---|---|
| 시스템관리자 | 허용(제안) | 전체(제안) | |
| 평가 총괄(안전보건 담당) | 허용(제안) | 평가자 지정, 결재 상신, 현황 조회(제안) | |
| 평가자(지정된 자) | 허용(제안) | **본인이 평가자로 지정된 건만 평가 등록** | 화면에 구현된 유일한 권한 분기 |
| 피평가자/열람자 | 조회(제안) | 본인 결과 열람 범위 §7 | |

- 평가 등록 버튼은 해당 건의 평가자가 현재 사용자일 때만 활성화된다(타인 지정 건은 비활성 + "배정된 평가자만 등록할 수 있습니다" 도움말, 평가자 미지정 건은 등록 버튼 자체가 없다). 매트릭스 하단에 "평가등록은 배정된 평가자 본인만 활성화됩니다"와 현재 로그인 사용자를 상시 안내한다.
- 그 외에는 역할별 노출·기능 차이가 구현되어 있지 않다(전 역할 동일 화면). 평가 설정 화면은 상단에 "**CEO·GM·SHM 접근 가능**" 안내가 있으나 실제 접근 차단은 없다(전 역할 진입 가능). 실제 접근 제어 범위는 §7(EVL-01).

## 4. 화면 구성별 정리

### 4-A. 인력 평가 (SCR-EVL-001)

#### 4-A-1. 평가개요·법적 근거
- 구성: 관련 법조항(중처법 시행령 §4①5호 · 산안법 §15/§16), 평가주기(반기 1회 — 상·하반기 기간), 평가일자 안내(평가 등록 시 자동 기록, 실제 평가일은 매트릭스·이력의 '평가일자' 열 참조), 평가방법(항목별 세부평가기준 양호·보통·미흡 점검). 우측 상단 [결재상신] 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [결재상신] | 현재 선택 부서의 평가 결과 보고 문서 미리보기(4-A-5) | |

- 상태: 안내 영역으로 별도 상태 없음. 결재상신 가능 여부는 선택 부서의 완료 평가 유무에 따른다(E3).

#### 4-A-2. 피평가자 매트릭스
- 구성: 부서 선택 → 표(구분 책임자/감독자 · 피평가자(이름·팀) · 상반기[평가자/평가일자/평가] · 하반기[동일]). 하단에 평가등록 권한 안내(현재 로그인 사용자 표시).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 부서 선택 변경 | 해당 부서 매트릭스로 갱신 | |
| [평가자 선택] (미지정 셀) | 조직도 모달(4-A-6) | |
| [평가등록] (본인 지정 건만 활성) | 평가표 모달(4-A-3) | |

- 상태(셀 단위 단계): **평가자 미지정**([평가자 선택] 버튼, 평가일자·평가 '-') → **지정·미평가**(평가자 이름 + [평가등록] — 본인이면 활성, 타인이면 비활성) → **완료**(평가자·평가일자·결과 칩). 결과 칩은 표시 전용이며 완료 평가 열람은 평가 이력 [보기]로 한다. 평가자가 한번 지정되면 매트릭스에서 변경할 수 없다(재지정 절차는 §7 EVL-10).

#### 4-A-3. 평가표 모달
- 구성: 제목(반기·역할 평가표) + 피평가자/평가자 표시 + 평가 기준 안내(양호/보통/미흡 정의와 종합 산출 규칙) + 항목별 3택(책임자 8항목 / 감독자 5항목) + 진행 표시("평가 항목 X/Y") + [닫기]/[저장]. 완료 평가 열람 시에는 [저장] 대신 [PDF 미리보기]가 노출되고 종합 결과와 "조회 전용"이 표시된다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 항목 판정 선택 | 진행 표시(X/Y) 즉시 갱신 | |
| [저장] | 종합 결과 자동 산출 후 저장, 매트릭스·이력 갱신 | 미평가 잔존 시 확인창 후 저장. "평가 저장 완료 — 종합 [결과]" |
| [닫기]·바탕 클릭·ESC | 저장 없이 닫힘(입력 소실, E6) | |
| [PDF 미리보기] (열람 시) | 개인 평가표 문서 미리보기(4-A-5) | |

- 상태: 등록 모드(판정 가능)와 조회 전용 모드(판정 비활성)를 구분한다. 확정 기획(개발 반영): 저장 버튼을 **[임시저장]과 [평가 완료]로 분리** — 미평가 항목이 있으면 임시저장만 가능(진행 중 집계), 전 항목 판정 후에만 평가 완료(종합 산출·이력 등재·결재 대상 포함). 상세 규칙은 §6.

#### 4-A-4. 평가 이력
- 구성: 완료 건수 표시 + 필터(사이클/부서/구분/결과 + 피평가자 이름 검색, [초기화]) + 표(평가일자·사이클·부서·구분·피평가자(팀 병기)·평가자·결과 칩·[보기]).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 필터·검색 변경 | 목록·건수 즉시 갱신 | 0건 시 "평가 이력이 없습니다" |
| [초기화] | 전체 필터·검색어 초기값 복원 | |
| [보기] | 읽기 전용 평가표 모달(4-A-3 조회 전용) | |

- 상태: 이력은 완료 평가만 누적한다(반기 사이클 누적). 빈 상태 문구 있음. 완료 평가는 수정·삭제 불가(§6 정정 절차).

#### 4-A-5. 결재 문서 / PDF 미리보기
문서가 두 종류이며 진입점이 다르다.

- **① 부서 평가 결과 보고(결재상신용)** — [결재상신]에서 진입. 결재 안내 배너(결재선 안내) + 공문 형식 본문: 평가 개요 표(부서·평가 대상 인원·작성자·작성일·근거 법령) + 완료 평가 목록 표(사이클·역할·피평가자·평가자·평가일자·결과) + 결과 분포 요약(양호/보통/미흡 건수) + 결재란(작성·검토·결재). 도구: [PDF 저장/인쇄] [온나라로 결재 상신] [닫기].
- **② 개인 평가표(조회용)** — 평가 이력 [보기] → [PDF 미리보기]에서 진입. 평가 개요(대상자·소속/역할·사이클·평가자·평가일자) + 항목별 평가 표 + 종합 결과(산출 규칙 병기) + "평가완료" 도장 표시 + 법적 근거 각주 + 결재란(평가자·부서장·결재). 도구: [PDF 저장/인쇄] [닫기].

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [온나라로 결재 상신] (①에서만) | 확인창(결재선: 평가자 → 부서장 → 군수) → 상신 처리(시뮬레이션) | "온나라 전자결재로 결재 상신되었습니다" |
| [PDF 저장/인쇄] | 문서 영역만 인쇄/PDF 저장 | |

- 상태: ①은 선택 부서 완료 평가 0건이면 상신 버튼 비활성(E3). 상신 후 결재 진행 상태 관리·반려 수신은 미구현(§7 EVL-07).

### 4-B. 평가 현황 (SCR-EVL-002)

#### 4-B-1. 사이클 선택
- 구성: 연도 선택 + [상반기]/[하반기]/[전체] 토글 + 평가 기간 표시 + 마감 D-Day 배지.
- 상태: 선택 상태(강조)는 전환되지만 **집계 수치는 예시 고정값**이다. 선택 사이클 기준 재집계는 §6(개발 반영).

#### 4-B-2. KPI·마감 배너
- 구성: KPI 4종 — 대상자(책임자·감독자 구성) / 완료(진행률 바) / 진행 중(평균 진행 항목) / 미시작(미시작 부서 요약). 마감 임박 배너 — 마감 D-Day·마감일·미시작 건수 강조 + [미시작 평가 시작 →].

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [미시작 평가 시작 →] | 인력 평가 화면으로 이동 | evl-eval.html |

- 상태: "진행 중" 집계는 임시저장(작성 중) 개념을 전제로 한 표시다 — 현재 인력 평가 화면에는 진행 중 상태가 없으므로(저장 즉시 완료) 임시저장 분리(§6)와 함께 정합화한다.

#### 4-B-3. 역할별 진행 카드 (책임자 / 감독자)
- 구성: 역할명 + 항목 수 배지 + "템플릿 →" 링크 + 진행도 바(완료/대상·백분율) + 완료·진행중·미시작 3분할 + 완료자 평균 점수(100점 만점).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| "템플릿 →" | **현재: 준비 중 안내 표시.** 확정 기획: 평가 설정(SCR-EVL-003)의 해당 역할 항목으로 이동(§6, 개발 반영) | |

- 상태: 평균 점수는 완료 평가의 환산 점수 평균(§6 점수 환산 기준).

#### 4-B-4. 부서별 진행률
- 구성: 부서별 가로 진행 바 목록(완료/전체, 진행률 구간에 따른 색 구분, 0건 부서는 경고 표시). 일부 부서명에 ★ 표시가 있으나 의미가 정의되어 있지 않다(§7 EVL-13).
- 상태: 진행률 0% 부서는 경고 색·표기로 강조된다.

### 4-C. 평가 설정 (SCR-EVL-003)

#### 4-C-1. 안내·접근
- 구성: 상단 배너 — "CEO·GM·SHM 접근 가능. 평가 항목 마스터·배점 변경은 **다음 사이클부터 적용**됩니다. 진행 중인 평가는 영향 없음."
- 상태: 안내만 있고 실제 접근 차단은 없다(§7 EVL-01).

#### 4-C-2. 평가 항목 마스터 (역할별 카드 2개)
- 구성: 역할명·설명(평가 대상 직급, 근거 법조항) + 항목 수 + 총 배점 + 항목 행(번호·항목명·평가 기준 가이드·배점·비중 바·[편집]) + [항목 추가]. **항목 삭제 기능은 없다**(추가·편집만, §7 EVL-11).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [편집] / [항목 추가] | 항목 편집 모달 — 역할(변경 불가)·항목 번호(자동 부여, 변경 불가)·항목명(필수)·평가 기준(평가자에게 표시되는 안내문)·배점(0~100 숫자) | |
| 편집 모달 [저장] | 해당 행 갱신(추가 시 맨 아래 행 추가) + 카드 상단 항목 수·총 배점 재계산 | "항목이 저장/추가되었습니다. 다음 사이클부터 적용됩니다." 항목명 미입력 시 "항목명을 입력하세요" |
| 편집 모달 [취소]·바탕 클릭 | 저장 없이 닫힘 | |

- 상태: 총 배점은 항목 배점 합계로 실시간 표시된다. 합계가 100이 아니어도 현재는 저장을 막지 않는다(E5, §6 배점 무결성으로 차단 예정). **주의 — 이 화면의 항목 마스터와 평가표 모달(4-A-3)의 실제 평가 항목이 서로 다른 목록이다**(설정: 운영지표형 문안 / 평가표: 산안법 직무 문안). 정본 확정·연동은 §7 EVL-09.

## 6. 정책

- **평가 주기**: 반기 1회 — 상반기(4~6월) / 하반기(10~12월). 사이클 단위로 평가·집계한다.
- **종합 결과 자동 산출**: 미흡 항목이 1개라도 있으면 「미흡」, 전 항목 양호면 「양호」, 그 외 「보통」. 판정자가 임의로 종합 결과를 정할 수 없다. 산출 규칙은 평가표·결재 문서에 병기한다.
- **평가자 지정 원칙**: 평가 등록은 해당 건의 지정 평가자 본인만 가능하다. 평가자 지정은 조직도 기반으로 한다. 지정 후 변경(재지정) 절차는 §7 EVL-10에서 확정한다.
- **결재 문서 이원화**: 결재 상신 대상은 **부서 단위 평가 결과 보고**(선택 부서의 완료 평가 목록·결과 분포)이고, 개인 평가표 문서는 이력 열람용 PDF로 제공한다. 부서 단위 상신 외 전 부서 일괄 상신 필요 여부는 §7 EVL-12.
- **임시저장과 평가 완료의 분리(확정, 개발 반영)**: 미평가 항목이 있으면 [임시저장]만 가능하며 해당 건은 "진행 중"으로 집계된다. 전 항목 판정 후 [평가 완료] 시에만 종합 결과를 산출하고 평가 이력에 등재하며 결재 상신 대상에 포함한다. 임시저장 건은 평가자 본인이 이어서 작성할 수 있다(모달을 닫아도 임시저장분은 유지 — 현재의 입력 소실(E6) 해소).
- **완료 평가 불변·정정 절차(확정)**: 완료된 평가는 읽기 전용이며 수정·삭제할 수 없다. 오입력 정정이 필요하면 평가 총괄이 **정정 재평가**를 개설한다 — 새 평가가 기존 평가를 대체하고, 기존 평가는 이력에 "정정됨" 표시로 보존된다(감사 추적). 정정 재평가는 해당 사이클의 결재 상신 전까지만 허용하며, 상신 이후의 정정은 결재 회수·반려 절차(§7 EVL-07)를 따른다.
- **점수 환산 기준(확정)**: 항목 점수 = 항목 배점에 판정 환산율(양호 100% / 보통 50% / 미흡 0%)을 적용해 역할별 100점 만점으로 합산한다. 평가 현황의 "평균 점수"는 완료 평가의 환산 점수 평균이다. 다만 **정본 판정은 3단 종합 결과**(자동 산출 규칙)이며, 점수는 부서·기간 비교용 보조 지표로만 사용한다.
- **배점 무결성(확정, 개발 반영)**: 역할별 배점 합계가 100점이 아니면 항목 저장을 차단하고, 편집 화면에 실시간 합계를 표시한다(현재는 고정 안내문만 있고 저장이 허용됨).
- **현황 집계 연동(확정, 개발 반영)**: 평가 현황의 연도·반기 선택 시 해당 사이클 기준으로 KPI·역할별·부서별 집계를 재산출한다(현재 프로토타입은 예시 고정값). 매트릭스 결과 칩에서 완료 평가 바로 열람, 역할 카드 "템플릿 →"의 평가 설정 이동도 같은 단계에서 반영한다.
- **미시작 독려 알림(확정, 개발 반영)**: 사이클 마감 D-14·D-7·D-1에 미시작 건의 평가자에게 알림을 발송하고, D-7부터는 해당 부서장을 참조로 포함한다. 임시저장 후 14일 경과 건도 동일하게 재알림한다. 발송 채널은 시스템 공통 알림 정책 확정에 따른다.
- **마스터 변경의 적용 시점**: 평가 항목·배점 변경은 다음 사이클부터 적용되고, 진행 중인 평가에는 영향을 주지 않는다.
- **결재**: 결재선은 평가자 → 부서장 → 군수. 선택 부서에 완료 평가가 있어야 상신할 수 있다. 프로토타입에서 온나라 연계는 안내 표시로 대체된다.
- **법적 근거 표기**: 평가표·결재 문서에 중대재해처벌법 시행령 제4조제1항제5호 및 산업안전보건법 제15조·제16조를 명기한다.
