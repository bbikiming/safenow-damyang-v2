# 화면 정의서 — 점검표(F5) 폼

> 화면 ID: SCR-EDOC-005 · 화면 유형: 공통 입력 폼(팝업 / 문서 상세에서는 인라인) · 기획 관점(v6)
> 확정이 필요한 항목은 §7 미결사항에 모았습니다.

## 1. 화면 정보

| 항목 | 내용 |
|---|---|
| 화면 ID | SCR-EDOC-005 |
| 화면명 | 점검표(F5) 폼 — 체크리스트(부적정 시 개선조치 자동 등록) |
| URL 경로 | 전용 URL·메뉴 없음 — 5개 호출 화면에 종속된 공통 컴포넌트. 호출처: 경영방침(menu.html?m=policy) · 도급관리(menu.html?m=contract) · 이행관리(menu.html?m=comply) · 안전보건교육(edu.html) · 문서 상세(doc-detail.html) 및 업무/이행 목록 행 |
| 상위 메뉴 | 별도 메뉴 항목 없음 — 위 호출 화면 각각의 하위 동작("점검표 작성")으로 진입 |
| 화면 유형 | 공통 입력 폼 — 호출처가 팝업이면 모달, 문서 상세에서는 좌측 패널 인라인 폼 |

**목적·설명** — 점검일·점검자·점검 항목(O/X/해당없음 + 비고 또는 텍스트형)·증빙 첨부를 입력하는 표준 점검표다. 전용 메뉴 없이 5개 호출 화면이 각자의 점검 목적에 맞는 항목 세트·제목·메뉴라벨을 주입해 같은 폼을 띄운다. 작성중 → 등록완료 → 확정 3단계로 진행하며, 확정 시 부적정(X) 판정 항목을 개선조치로 자동 등록하고 온나라 결재요청 안내로 전환한다. 점검자는 자유 입력이 아니라 조직도 트리에서만 선택하며, 경영방침 점검표에서는 항목별 관련 법령 상세를 별도 창으로 열어볼 수 있다. 호출 형태(모달 / 문서 상세 인라인)에 따라 푸터 버튼 구성과 저장 후 동작(닫힘 / 그 자리 갱신)이 달라진다. 연관 화면: 개선조치 목록(SCR-EDOC 계열 자동 유입 대상), 평가서(F6, SCR-EDOC-006 — 도급관리에서 점검표와 나란히 노출되는 별도 폼), 업무 등록·수정 마법사(SCR-DOC-007).

## 4. 화면 구성별 정리

### 4-1. 헤더 영역 (폼종류·상태·메뉴라벨)
- 구성: 폼종류 칩("점검표") + 상태 칩(작성중 / 등록완료 / 확정 · 온나라 상신) + 메뉴라벨 칩(호출처가 주입한 "{대메뉴} · {주기}", 예: "경영방침 · 반기").

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| (표시 전용) | 현재 폼 종류·진행 상태·소속 메뉴를 한눈에 안내 | — |

- 상태: 저장본이 없으면 작성중으로 표시된다. 상태 칩은 저장된 상태에 따라 자동으로 바뀐다.

### 4-2. 연동 정보 카드 (옵션)
- 구성: 호출처가 연동 정보를 주입한 경우에만 "🔗 연동 정보 — {내용}" 한 줄을 헤더 아래 표시(문서 상세 인라인에서 소속 세트 정보 표시). 주입이 없으면 영역 자체가 나타나지 않는다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| (표시 전용) | 이 점검표가 어느 세트·문서에 연동되는지 안내 | — |

- 상태: 연동 정보가 없으면 미표시.

### 4-3. 입력 폼 — 점검일 / 점검자 / 점검 항목 / 증빙 첨부
- 구성: 라벨 + 입력칸 4행. 점검일(날짜), 점검자(조직도 선택), 점검 항목(체크리스트), 증빙 첨부(파일 영역).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 점검일 입력 | 날짜 선택. 저장값이 없으면 기본값 "2026-06-11"이 자동으로 채워져 있음(빈칸 아님) | — |
| [조직도] (점검자) | 조직도 선택 창을 폼 위에 띄움(§4-7) — 점검자 칸은 직접 타이핑 불가, 조직도에서만 채워짐 | 입력 도움말 "조직도에서 점검자를 선택하세요" |
| 항목 [O]·[X]·[해당없음] (점검 항목) | 같은 행에서 한 버튼만 선택(다른 선택 자동 해제). X 선택 시 빨강 강조 | 체크리스트 하단 고정 안내 "X 판정 항목은 확정 시 개선조치로 자동 등록됩니다."(빨강) |
| 비고/사유 입력 (점검 항목) | 각 행의 텍스트 입력("비고 / X 사유" 또는 "비고 및 판단 근거") | — |
| "관련 근거 · {법령} ⓘ" (객체형 항목) | 관련 법령 상세 창을 폼 위에 띄움(§4-8) | — |
| 증빙 첨부 영역 클릭 | 드롭존("파일을 끌어다 놓거나 클릭하여 첨부 (다중 가능)") 클릭 | 안내 "파일 첨부 (프로토타입 — 다중 가능)"만 표시, 실제 업로드 없음 |

- 점검 항목 표시 분기: 호출처가 객체형 세트(경영방침 policy)를 주입하면 각 행 상단에 영역 칩 + 항목명 + 결과유형 칩(O/X 또는 텍스트) + 관련 근거 버튼이 함께 표시된다. 결과유형이 "텍스트"인 항목은 O/X 버튼 대신 자유 텍스트 입력칸("점검 결과를 입력하세요")이 나온다. 결과유형은 항목마다 미리 지정되어 있다. 나머지 호출처(도급관리·이행관리·안전보건교육·문서상세)는 단순 문자열 항목으로, 모든 항목이 O/X/해당없음 + 비고 형식이다.
- 상태: 확정 상태에서도 점검일·점검 항목 등 입력칸은 읽기전용으로 잠기지 않는다(현재 동작 — 잠금 정책은 §6·§7). 입력값에 대한 형식·필수 검증은 없다(§5).

### 4-4. 증빙 첨부 목록 (경영방침 점검표에 한해 데이터 주입 시)
- 구성: 호출처가 첨부 목록을 주입한 경우(현재 경영방침에서만)에만 드롭존 아래에 "첨부파일 목록" 테이블 표시 — 컬럼: 파일명 / 파일 설명 / 등록일 / 관리. 관리 칸에 행마다 [설명 수정]·[삭제] 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [설명 수정] | (현재) 안내만 표시, 실제 수정 없음 | "파일 설명 수정 (프로토타입)" |
| [삭제] | (현재) 안내만 표시, 실제 삭제 없음 | "파일 삭제 (프로토타입)" |

- 상태: 첨부 목록 데이터가 주입되지 않은 호출처(도급관리·이행관리·안전보건교육·문서상세)에서는 이 테이블이 나타나지 않는다.

### 4-5. 처리 이력 영역 (옵션)
- 구성: 저장 이력이 있을 때만 "처리 이력" 타임라인 표시. 행마다 시각 + 이벤트 문구. 이벤트: "임시저장", "등록 (작성자: 박안전)", "확정 · 온나라 결재 상신".

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| (표시 전용) | 임시저장·등록·확정 시점마다 이력이 누적되어 시간순 표시 | — |

- 상태: 이력이 하나도 없으면(신규 빈 폼) 영역 미표시.

### 4-6. 푸터 버튼 영역 (상태별 분기, 모달 / 인라인 차이)
- 구성: 진행 상태와 호출 형태(모달 / 인라인)에 따라 버튼 구성이 달라진다.

| 상태 | 모달 버튼 | 인라인 버튼 |
|---|---|---|
| 작성중 | [취소] [임시저장] [등록] | [임시저장] [등록] |

### 4-8. 관련 법령 상세 창 (객체형 항목 한정)
- 구성: 경영방침 점검표의 "관련 근거 · {법령} ⓘ" 클릭 시 폼 위에 별도 창. 제목 "관련 법령 상세", 법령명·조·항·호·제목·조문 본문 표시.

### 4-9. 부적정(X) → 개선조치 자동 등록
- 구성: 확정 시 점검 항목 중 X 판정된 모든 항목마다 개선조치 1건을 생성한다(별도 입력 영역 없음, 확정의 부수 동작).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 확정 처리 | X 항목마다 개선조치 1건 생성 — 제목=항목명(비고 있으면 "— {비고}" 덧붙임), 발생 메뉴=메뉴라벨, 발생 문서=폼 제목, 담당자=기본 "미지정", 기한=고정 "2026-07-31", 초기 상태="접수" | (X건 있으면) "X 항목 N건이 개선조치로 자동 등록되었습니다" |

- 개선조치 흐름: 접수 → 계획 → 진행 → 완료확인 → 종결. 생성된 개선조치는 개선조치 목록에 누적되어 다른 화면에서 조회된다. "해당없음" 항목과 텍스트형 항목은 개선조치 생성 대상이 아니다(X만).
- 상태: 현재 기한·담당자가 고정값이며 산정 규칙은 미정(§6·§7). "해당없음" 항목의 집계·점수화 처리도 미정(§7).

### 4-10. 온나라 결재요청 안내 (확정 시 전환)
- 구성: 확정 시 폼을 닫고 같은 자리에 안내 팝업으로 전환한다(동시에 2개가 뜨지 않음). 📨 아이콘 + "온나라로 결재 요청을 보냈습니다" + 폼 제목 + 자동 생성 문서번호(예 "온나라-2026-XXXX") + "결재선: 팀장 → 과장 → 부군수" + "결재 완료 시 문서 상태가 자동으로 갱신됩니다. (연계 시뮬레이션)" + [확인].

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| (확정의 후속) | 결재요청 안내 표시(문서번호·결재선) | 현재는 안내만 — 실제 결재 연동·반려 처리 없음 |
| [확인] | 안내 팝업 닫고 호출 화면으로 복귀 | 호출 화면 상태 칩이 "확정 · 온나라 상신"으로 갱신 |

- 상태: 문서번호는 더미 자동 생성값이다. 실제 결재 결과 수신·반영은 미구현(§6·§7).

## 5. 유효성 검사

본 화면에는 입력 폼(점검일·점검자·점검 항목·비고·증빙)이 있으나, 현재 필수·형식 검증과 오류 안내가 전혀 정의되어 있지 않다. 빈 점검일(기본값은 들어가나 비울 수 있음)·점검자 미선택·미점검·비고 공란으로도 임시저장·등록·확정이 모두 가능하며, 미저장 종료 경고나 확정 전 확인 절차도 없다. 아래는 입력 항목과 제안 규칙이며 확정은 §7.

| 입력 항목 | 필수 | 형식/규칙 | 오류 안내 |

## 6. 정책

- **입력 필수·미점검 차단** — 현재 점검일·점검자·점검 항목·비고에 대한 필수·형식 검증이 없어 빈 폼·미점검으로도 진행된다. 필수 여부, 전 항목 응답 강제, 미점검 누락 차단을 정해야 한다(§7).
- **확정 사전 확인** — 현재 확정·상신은 사전 확인 없이 즉시 비가역 처리(개선조치 생성 + 온나라 상신)된다. 확정 전 확인 절차 도입 여부를 정해야 한다(§7).
- **확정 문서 잠금** — 현재 확정 후에도 점검일·점검 항목 등 입력칸이 읽기전용으로 잠기지 않는다. 확정본 잠금 여부를 정해야 한다(§7).
- **개선조치 자동 등록 규칙** — 현재 X 항목 개선조치의 기한은 고정 "2026-07-31", 담당자는 기본 "미지정", 초기 상태는 "접수"로 생성된다. 기한·담당자 산정 규칙, 항목 근거 전이 여부, "해당없음" 항목의 집계·점수화 처리를 정해야 한다(§7).
- **호출처별 항목 세트 관리** — 점검 항목 구성·폼 제목·메뉴라벨·점검자 기본값·첨부 목록은 호출 화면이 주입한다(경영방침=객체형 7항목+점검자 기본값+첨부 2건, 도급관리=4항목, 이행관리=6항목, 안전보건교육=4항목, 문서상세=기본 3항목). 항목 세트의 정본 관리·갱신 주체를 정해야 한다(§7).
- **온나라 결재 결과 반영** — 현재 결재요청은 안내만이며 문서번호는 더미값이다. 실제 연계, 승인·반려 수신 후 상태 갱신·되돌림(개선조치 롤백 포함), 문서번호 채번 규칙을 정해야 한다(§7).
- **개정(새 버전)** — 현재 [개정]은 안내만이며 실제 새 버전이 생성되지 않는다. 버전 생성·이력 보존·상태 초기값 규칙을 정해야 한다(§7).
- **작성자·결재선** — 현재 작성자("박안전")·결재선(팀장 → 과장 → 부군수)이 고정값이다. 실제 작성자·결재선 구성 규칙을 정해야 한다(§7).
