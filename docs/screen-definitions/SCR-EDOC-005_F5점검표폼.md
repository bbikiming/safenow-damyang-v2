| 6 | (확정 문서 재진입) [개정] | 새 버전 작성 안내(현재 프로토타입은 안내만) |

## 4. 화면 구성별 정리

### 4-1. 점검표 입력
- 구성: 점검일, 점검자, 점검 항목(호출 화면별 항목 구성 — O/X/해당없음 + 비고, 또는 텍스트형), 증빙 첨부.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 항목 O/X/해당없음 선택 | 항목별 단일 선택, 부적정(X)은 강조 | 안내문 "부적정 항목은 확정 시 개선조치로 자동 등록됩니다" |
| 증빙 첨부 클릭 | 파일 첨부 | 현재 프로토타입은 안내만(실제 업로드 없음) |

- 상태: 점검일·점검자·항목 입력은 형식·필수 검증이 없음(빈 값·미점검도 진행 가능) — 규칙 §5·§6·§7.

### 4-2. 상태 진행 (작성중 → 등록완료 → 확정)
- 구성: 상단 상태 표시 + 단계별 버튼 + 처리 이력. 단계에 따라 버튼이 달라진다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [임시저장] | 현재 단계 유지하며 입력 저장 | "임시저장되었습니다 — 작성중 상태로 유지" |
| [등록] (작성중) | 작성중 → 등록완료 | "등록되었습니다 — 확정 전까지 수정 가능" |
| [확정·온나라 결재 상신] (등록완료) | 등록완료 → 확정(§4-3·§4-4 동반) | (§4-4 안내) |
| [개정] (확정 후) | 새 버전 작성 | "개정 — 새 버전이 작성중 상태로 생성됩니다" (현재 안내만) |
| [취소]/[닫기] | 팝업 닫기 | 미저장 변경 경고 없음(§6·§7) |

- 상태: 작성중 / 등록완료 / 확정. 확정 후 입력란 잠금(읽기전용) 여부는 미정(§6·§7).

### 4-3. 부적정(X) → 개선조치 자동 등록
| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 확정 처리 | 부적정(X) 판정 항목마다 개선조치를 자동 등록 | "X 항목 N건이 개선조치로 자동 등록되었습니다" |

- 상태: 개선조치 자동 등록 시 기한·담당자 산정 규칙, "해당없음" 항목 처리는 미정(§6·§7).

### 4-4. 온나라 결재 상신 (안내)
| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 확정·상신 | 온나라 결재요청 안내 팝업 표시(문서번호·결재선: 팀장→과장→부군수) | 현재는 안내만(실연계·반려 처리 §6·§7) |

### 4-5. 첨부파일 목록 (경영방침 점검표 한정)
| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 첨부 [설명 수정]/[삭제] | 첨부 목록 수정·삭제 | 현재 프로토타입은 안내만 |

## 5. 유효성 검사

본 화면에는 입력 폼이 있으나, 현재 필수·형식 검증과 오류 안내가 정의되어 있지 않다(빈 점검일·미점검 항목으로도 등록·확정 가능). 아래는 입력 항목과 확정이 필요한 규칙이며, 상세는 §7.

| 입력 항목 | 필수(제안) | 형식/규칙 | 오류 안내 |
|---|---|---|---|
| 점검일 | 필수(제안) | 날짜 | 미정 |
| 점검자 | 필수(제안) | — | 미정 |
| 점검 항목 | 전 항목 응답 필요 여부 미정 | O/X/해당없음 또는 텍스트 | 미정 |
| 비고 | 부적정(X) 시 사유 필수 여부 미정 | — | 미정 |
| 증빙 첨부 | 선택 | 파일 형식·용량·개수 규칙 미정 | 미정 |

## 6. 정책

- **입력 필수·미점검 차단** — 점검일·점검자·점검 항목의 필수 여부, 전 항목 응답 강제, 미점검 누락 차단 정책 확정 필요(§7).
- **확정 사전 확인** — 확정·상신은 사전 확인 절차 없이 즉시 비가역 처리(개선조치 생성·상신). 확정 전 확인 절차 필요 여부 확정 필요(§7).
- **확정 문서 잠금** — 확정 후 입력란을 읽기전용으로 잠글지 확정 필요(§7).
- **개선조치 자동 등록 규칙** — 부적정(X) 항목의 개선조치 기한·담당자 산정 규칙, 항목 근거 전이 여부, "해당없음" 항목의 집계·점수화 처리 확정 필요(§7).
- **온나라 결재 결과 반영** — 실제 연계, 승인·반려 수신 후 상태 갱신·되돌림(개선조치 롤백 포함), 문서번호 채번 규칙 확정 필요(§7).
- **개정(새 버전)** — 개정 시 버전 생성·이력 보존·상태 초기값 규칙 확정 필요(§7).
- **작성자·결재선** — 현재 작성자·결재선이 고정값. 실제 작성자/결재선 구성 규칙 확정 필요(§7).
