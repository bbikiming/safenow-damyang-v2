## 4. 화면 구성별 정리

### 4-1. 필터 바
- 구성: 검색(이름·부서), 부서, 구분, 고용형태, 명단 출처, 채용연도.
- 우측 액션: [엑셀 업로드], [직접 등록]. 결과 건수는 "필터결과 / 전체 N명".

### 4-2. 명단 표
- 구성: 이름(출처 배지: 인사연동/직접등록/엑셀업로드) · 부서 · 구분 · 고용형태(계약개월 병기) · 채용일 · 액션.
- 상태: 출처가 인사연동이면 액션이 "읽기 전용", 그 외는 [수정]·[삭제]. 명단에서 제외한 인원은 표시하지 않는다.

### 4-3. 등록/수정 모달
- 구성: 이름, 부서(조직도), 구분(4종), 고용형태(4종), 채용일, 계약기간(기간제만).
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [등록] | 직접등록으로 명단에 추가 | "{이름} 근로자 등록 완료" |
| 인사연동 인원 수정 시도 | 차단 | "인사연동 근로자는 수정할 수 없습니다." |
| [삭제] | 확인 후 명단에서 제외(이력 보존) | "근로자를 명단에서 제외했습니다." |

### 4-4. 엑셀 업로드 모달
- 구성: 대상 부서(조직도), 파일 선택.
- 상태: 업로드 시 해당 부서에 인원을 추가하고 출처를 엑셀업로드로 표시한다(프로토타입은 표본 데이터로 시연).

## 6. 정책

- **명단 출처 3종:** 인사연동(읽기전용)·직접등록·엑셀업로드로 출처를 구분해 표시한다. 인사연동분은 수정·삭제할 수 없다.
- **삭제는 보존 처리:** 근로자 삭제는 이력을 보존하는 제외(비활성) 처리로, 완전 삭제하지 않는다.
- **모집단 파생:** 신규 등록 인원은 별도 절차 없이 채용시교육 미이수로 파생되고 이수현황 모집단에 즉시 반영된다.
- **대상 관리:** 이 명단이 정기·채용시·기타·이수현황 전 교육 화면의 대상자 산정 기준이다.
