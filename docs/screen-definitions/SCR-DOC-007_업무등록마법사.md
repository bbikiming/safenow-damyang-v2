> 확정이 필요한 항목은 §7 미결사항에 모았습니다.

| 7 | (수정) [편집] | 기존 값이 채워진 팝업 → 변경 후 [저장] |
| 8 | (삭제) [삭제] → 확인 팝업에서 [삭제] | 목록·문서 상세에서 제거 |

## 4. 화면 구성별 정리

### 4-1. 단계 표시(스텝) / 이동
- 구성: 4단계 진행 표시(분류 · 처리유형 · 상세입력 · 확인), [이전]·[다음], 마지막 단계는 [등록 완료](수정 시 [저장]), 수정 모드에서는 [삭제] 함께 노출.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [다음] | 현재 단계 필수값 검사 후 다음 단계로 | 미입력 시 해당 항목 안내 |
| [이전] | 입력값을 유지한 채 이전 단계로 | — |
| [등록 완료]/[저장] | 입력 내용 저장 | 팝업 닫힘 + "등록/수정되었습니다" |
| [취소]·[닫기] | 입력 취소 | 원래 화면 유지 |

- 상태: 현재 진행 단계(분류→처리유형→상세입력→확인)를 항상 표시한다. 등록 모드와 수정 모드를 구분하며, [삭제]는 수정 모드에서만 노출한다.

### 4-2. 1단계 — 분류
- 구성: 법령 분류(대메뉴) **탭 선택**, 세트 선택, PDCA 단계 선택(P 계획·D 실행·C 점검·A 조치) **단일 선택**.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 대메뉴 탭 클릭 | 해당 분류의 세트 목록으로 갱신 | — |
| 세트 선택 | 선택한 세트의 PDCA 구성을 안내로 표시 | — |
| PDCA 단계 선택 | 한 단계만 선택 | — |

- 상태: 세트를 고르지 않으면 "세트 미지정"으로 등록할 수 있음(안내).

### 4-3. 2단계 — 처리유형 선택
- 구성: 첨부파일 / 전자문서 / 프로그램 중 하나 선택(아이콘·한 줄 설명 포함).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 처리유형 선택 | 3단계의 상세 입력 폼 구성을 결정 | "선택에 따라 다음 단계 입력 폼이 달라집니다" |

### 4-4. 3단계 — 상세 입력 (담당 공통 + 처리유형별 분기)
- 공통(담당): 담당부서는 조직도에서 담당자를 지정하면 자동 표시(직접 선택하지 않음), [담당자 지정] 버튼으로 조직도에서 담당자를 고른다. 담당자는 세 처리유형 모두 필수.
- 첨부파일: 문서명 · 담당 · 주기 · 법적 근거(직접 입력) · 파일 첨부(파일명·설명 입력 후 [파일 추가] → 첨부파일 목록에 누적, 행별 삭제) · 보존 연한.
- 전자문서: 문서명 · 담당 · 연계 양식 선택(표준 양식 또는 전자문서 빌더 양식, 선택 양식 미리보기, [새 양식 만들기]로 전자문서 빌더 이동) — 그 외 항목은 전자문서 양식 안에서 입력.
- 프로그램: 담당 · 연결 메뉴 선택 → 연결 화면(뎁스) 선택 — 문서명은 선택한 화면 이름으로 자동 설정.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [담당자 지정] | 조직도 팝업 열림 → 1명 선택 시 담당부서·담당자 동시 설정 | 버튼에 선택한 담당자 표시 |
| [파일 추가](첨부파일) | 입력한 파일명·설명을 첨부파일 목록에 추가 | 파일명 미입력 시 안내 |
| 목록의 [삭제](첨부파일) | 해당 파일을 목록에서 제거 | — |
| 연계 양식 변경(전자문서) | 선택 양식의 미리보기 항목을 갱신 | — |
| [새 양식 만들기](전자문서) | 전자문서 빌더 화면으로 이동 | admin-presets.html |
| 연결 메뉴 변경(프로그램) | 해당 메뉴의 화면(뎁스) 목록으로 갱신 | — |

- 상태: 신규 등록 문서의 초기 상태는 "미시행"이다. 전자문서는 등록 이후 문서 상세에서 작성중 → 등록완료 → 확정(결재 상신) 흐름으로 이어진다(SCR-DOC-005 · SCR-EDOC 계열 연계). 처리유형을 바꾸면 직전 처리유형에서 입력하던 항목은 비워진다.

### 4-5. 4단계 — 확인·저장
- 구성: 입력 요약(법령 분류·세트·PDCA·처리유형·문서명·연계 대상·담당) + [저장].

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 요약 검토 | 저장 전 입력 내용을 한눈에 확인 | — |
| [등록 완료]/[저장] | 문서 저장 후 목록·상세 반영 | 팝업 닫힘 + 안내 |

- 상태: 저장 직후 결과가 목록·상세에 즉시 반영된다(신규는 해당 분류·세트·PDCA 위치에 추가, 수정은 기존 항목 갱신, 삭제는 목록·상세에서 제거).

### 4-6. 조직도 담당자 선택 팝업
- 구성: 부서별 구성원 목록에서 1명 선택. 선택 시 담당부서·담당자를 함께 설정하고 팝업을 닫는다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 부서 펼치기 | 해당 부서 구성원 표시 | — |
| 구성원 선택 | 담당부서·담당자 설정 후 팝업 닫힘 | "담당자 지정" 안내 |

### 4-7. 삭제 확인 팝업
- 구성: 대상 문서명 표시 + [삭제]·[취소]. 사용자 등록 문서만 삭제 대상.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [삭제] | 문서 제거 후 목록·상세 갱신 | "삭제되었습니다" |
| [취소] | 삭제 취소 | 팝업 닫힘 |

## 6. 정책

- **사용자 등록 문서 구분** — 새로 등록한 문서는 기본 제공 문서와 구분해 관리하며, 수정·삭제는 사용자 등록 문서에 한한다(기본 제공 문서는 조회만). 적용 범위는 §7에서 확정.
- **처리유형에 따른 상세 동작** — 문서 상세에서 첨부파일=미리보기·다운로드, 전자문서=입력 폼, 프로그램=연결 화면으로 이동으로 갈린다(SCR-DOC-004·005·006 연계).
- **전자문서 양식 연계** — 전자문서는 표준 양식 또는 전자문서 빌더에서 만든 양식 중에서 고른다(SCR-EDOC 계열·프리셋 양식 관리 연계).
- **프로그램 연결 대상** — 시스템 내 화면(메뉴 → 화면)을 골라 연결한다.
- **분류 기준** — 업무 목록과 동일하게 법적의무 재분류 기준을 정본으로 한다.
- **반영 범위** — 등록·수정·삭제 결과는 업무 목록·이행 목록·문서 상세에 동일하게 반영된다.
