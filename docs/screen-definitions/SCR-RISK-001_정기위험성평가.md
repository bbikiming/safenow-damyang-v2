- 연도별 정기 위험성평가를 **연 1건 원칙**으로 관리하는 화면이다. 별도 상세 화면 없이 **대상 부서 선정 → 용역업체 보고서 업로드·검수 → 부서별 조치기한 배분·전달 → 개선조치 완료·재촉**까지 한 화면에서 처리한다.
- 상단에서 연도를 고르고, 해당 연도에 평가가 없으면 생성 마법사로 만들며, 있으면 요약 카드와 부서별 조치·이력 탭으로 진행 상황을 관리한다.
- 과거 "위험성평가 상세(rsk-detail.html)"는 이 화면으로 통합되어, 구 상세 링크는 이 목록으로 자동 이동한다(하위호환).

## 4. 화면 구성별 정리

### 4-1. 상단 도구모음
- 구성: 연도 선택, 상태별 우측 영역(평가 없음=[정기평가 생성] / 진행중=연 1회 등록 원칙 안내 / 완료=완료일 표기), 시연 초기화.
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [정기평가 생성] | 생성 마법사 시작 | 해당 연도 평가가 이미 있으면 "이미 존재" 안내로 차단 |
| [시연 초기화] | 확인 후 시연 데이터 초기 상태로 복귀 | 당해년 미등록·전년 완료 상태로 복원 |

### 4-2. 빈 상태(미등록 연도)
- 구성: 안내 문구, 진행 단계 칩(① 대상 부서 선정 → ② 점검일자 지정), [정기평가 생성].
- 상태: 설문지는 생성 후 첨부한다는 안내를 함께 표기한다.

### 4-3. 요약 카드(평가 존재 시)
- 구성: 유형 배지(정기), 제목, 상태(진행중/완료), 메타(대상 부서 수·완료 부서 수 / 공통 설문지·부서 적용 수 / 보고서 첨부 여부 / 완료일), 진행률(개선조치 완료/전체).

### 4-4. 부서별 조치 탭
- 구성: 용역업체 보고서 블록, 부서별 조치 표(가로 스크롤), 지적사항 없는 부서 제외 안내.
- 보고서 블록 단계별 동작:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [보고서 업로드] (미첨부) | 보고서에서 부서별 지적 항목을 추출하고 검수 단계로 전환 | "N개 부서 M건 항목을 추출했습니다" |
| [보고서 취소] (검수 중) | 확인 후 첨부·검수결과 초기화 | — |
| [보고서 교체]·[보고서 첨부] (전달 완료) | 재추출 없이 보고서 파일만 교체 | "개선 건수·조치 내역은 유지됩니다" |

- 부서별 조치 표 구성: 부서 · 점검일 · 설문지(공통본/부서별본/첨부) · 개선 건수(완료/전체) · 조치기한 · 상태(조치전/조치완료·전달일) · 관리(검수 진행/관리/보기·재촉).
- 상태: 조치기한·개선 기한이 기준일을 넘기면 강조 표기하고 [재촉]을 노출한다.

### 4-5. 검수 인라인 패널(검수 중)
- 구성: 부서별 펼침, 부서별 편집 표(유해위험요인 · 분류 · 원인 · 개선조치)와 행 추가·삭제.
- 정책: 검수 단계에서는 항목 내용만 다루고, 조치기한·확인은 다음 단계(조치기한 설정)로 넘긴다.
- 상태: 추출 항목이 0건이면 [검토완료]가 비활성되고 "최소 1건 입력" 안내가 뜬다.

### 4-6. 조치기한 설정(검토완료 → 전달)
- 구성: 일괄 조치기한 + 일괄 적용, 부서별 조치기한 표.
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [전달 실행] | 유효 항목이 있는 부서에 개선조치 생성·배분, 조치기한 통보 | 기한 미지정 부서가 있으면 "N개 부서 미지정"으로 차단 |
| 지적 0건 부서 | "지적사항 없음(조치 대상 제외)"으로 자동 조치완료 처리 | 목록·완료율 집계에서 제외 |

### 4-7. 점검설문지 첨부(등록 이후)
- 구성: 공통 설문지(전 부서 적용) 첨부·변경·삭제, 부서별 설문지(해당 부서만 대체) 첨부.
- 상태: 완료된 평가는 첨부를 잠근다.

### 4-8. 생성 마법사(2단계)
- 구성: (1단계) 대상 부서 선정 — 인사정보 연동 목록에서 다중 선택(기본 전체선택), (2단계) 점검일자 — 일괄/부서별 지정.
- 상태: 각 단계 유효성 미충족 시 차단(§5). 생성 시 대상 부서에 점검예정일을 통보하고, 설문지는 이후 목록에서 첨부한다.

### 4-9. 이력 탭
- 구성: 평가 이력과 각 개선조치 이력(알림 제외)을 병합한 타임라인.

## 6. 정책

- **연 1건 원칙:** 정기 위험성평가는 연도별 1건을 원칙으로 하고, 목록과 상세를 한 화면에 통합한다. 별도 상세 화면(구 rsk-detail.html)은 폐지하고 하위호환 링크만 이 화면으로 유도한다. (근거: 산업안전보건법 제36조, 사업장 위험성평가에 관한 지침)
- **설문지 후행 첨부:** 설문지 확정 지연으로 일정 통보가 막히지 않도록, 마법사에서 설문지 단계를 제거하고 생성 후 목록에서 첨부한다.
- **보고서 기반 흐름:** 공정별 위험성 추정·평가기법 전환 화면 없이, 용역업체 보고서 업로드·자동 추출·검수 방식으로 지적 항목을 확정한다.
- **지적 0건 자동 제외:** 지적사항이 없는 부서는 "조치 대상 제외"로 자동 처리하고 완료율 집계에서 뺀다.
- **재평가 연동:** 개선조치(감소대책)가 완료되면 해당 대상이 재평가 대기로 전환되고, 조치 대상 부서가 모두 완료되면 평가가 자동 완료된다.
- **기준일:** 기한 초과 판정에 쓰는 기준일 정책은 화면·집계 간 일관되게 확정한다(§7).
