**목적·설명** — PDCA 세트(업무 묶음) 단위로 업무문서 현황을 한눈에 보는 목록 화면이다. 대메뉴별로 세트를 묶고, 세트 안의 문서를 PDCA 단계(계획·실행·점검·조치)로 정리해 보여준다. 상단에서 분류 기준(기존 분류 ↔ 법적의무 재분류)을 전환할 수 있고, 재분류 기준에서는 세트별 PDCA 구성의 적정성(정상·주의·오류)을 함께 표시한다. 문서를 클릭하면 문서 상세로 이동하며, 문서의 처리 유형(전자문서·첨부파일·프로그램)에 따라 상세에서의 작업이 달라진다. 본 화면은 조회·검색·이동 중심이며, 목록에서 문서를 직접 등록·수정하지는 않는다.

## 3. 진입 / 이탈 경로 및 사용자 시나리오

**진입 / 이탈**

| 구분 | 연결 화면 | 조건 / 트리거 |
|---|---|---|
| 진입 | 업무문서 > 업무 목록 | 메뉴 클릭 |
| 진입 | 각 대메뉴 화면의 [문서] 탭 | 같은 목록 구성을 해당 대메뉴로 한정해 보여줌 |
| 진입 | 타 화면에서 특정 대메뉴로 이동 | 해당 대메뉴 탭이 선택된 상태로 진입 |
| 이탈 | 문서 상세(doc-detail.html) | 문서명 또는 행 액션 클릭 |
| (이탈 아님) | 동일 화면 재구성 | 분류 기준 전환 시 목록만 다시 구성 |

**사용자 시나리오**

| 단계 | 사용자 행동 | 화면 반응 |
|:--:|---|---|
| 1 | 업무 목록 진입 | 분류 기준(기본: 법적의무 재분류)으로 대메뉴별 세트 카드 + PDCA 단계별 문서 표 표시. 재분류 기준이면 검증 요약(정상·주의·오류) 배너 노출 |
| 2 | 대메뉴 탭 선택 | 해당 대메뉴의 세트만 표시 |
| 3 | 문서명·담당자 검색 / 상태·PDCA 단계 필터 | 즉시 결과를 좁힘. 결과가 없으면 "조건에 맞는 문서가 없습니다" 안내 |
| 4 | (재분류) '검토 대상만 보기' | 주의·오류로 표시된 세트만 노출 |
| 5 | 문서명 또는 행 액션 클릭 | 문서 상세로 이동(처리유형에 따라 입력/미리보기/이동 형태가 달라짐) |
| 6 | '목록 다운로드' | 현재 보이는 목록을 파일로 받기(현재 프로토타입은 안내만 표시) |

## 4. 화면 구성별 정리

### 4-1. 상단 — 분류 기준 전환 / 검증 요약
- 구성: 분류 기준 토글(기존 분류 ↔ 법적의무 재분류), (재분류 시) 검증 요약 배너(정상·주의·오류 건수).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 분류 기준 토글 | 선택한 기준의 세트 구성으로 목록 전체를 다시 구성(대메뉴 탭·검색·필터 초기화) | 화면 내 재구성(이동 없음) |
| (재분류) 검증 요약 배너 | 세트 PDCA 구성의 정상·주의·오류 건수를 요약 표시 | 주의·오류 세트는 카드에 경고 표시 |

- 상태: 기본 분류 기준과 검증 판정 규칙은 §6·§7 참조.

### 4-2. 검색·필터
- 구성: 대메뉴 탭(전체 + 대메뉴), 문서명·담당자 검색, 상태 필터(완료/진행/미시행), PDCA 단계 필터(계획·실행·점검·조치), (재분류) '검토 대상만 보기'.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 대메뉴 탭 | 해당 대메뉴 세트만 표시 | — |
| 문서명·담당자 검색 | 입력 즉시 부분일치로 좁힘 | 0건이면 "조건에 맞는 문서가 없습니다" |
| 상태 / PDCA 단계 필터 | 선택값에 맞는 문서만 표시 | — |
| 검토 대상만 보기(재분류) | 주의·오류 세트만 / 다시 누르면 전체 | — |

- 상태: 결과 0건 안내는 제공됨. 로딩·오류(데이터 조회 실패) 상태 처리는 미정(§7).

### 4-3. 세트 카드 목록
- 구성: 세트 헤더(대메뉴·세트명·문서 수·완료 수·선행 관계·(재분류) PDCA 구성/검증 표시) + PDCA 단계별 문서 행(문서명·담당자·부서·수정일·상태·액션).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 문서명 클릭 | 문서 상세로 이동 | doc-detail.html |
| 행 액션 클릭 | 문서 상세로 이동(처리유형별 라벨: 전자문서=입력 / 첨부파일=미리보기 / 프로그램=이동 / 기타=상세) | doc-detail.html |

- 상태: 세트에 배정되지 않은 문서는 "세트 미지정" 그룹으로 분리해 표시. 빈 PDCA 단계는 행을 만들지 않음(표시 정책은 §6·§7).

### 4-4. 목록 다운로드
- 구성: [목록 다운로드] 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 목록 다운로드 | 목록을 파일로 내려받기 | 현재 프로토타입은 안내 메시지만 표시(실제 내보내기 미구현) — 형식·범위 §6·§7 |

## 6. 정책

- **분류 기준 운영** — 기존 분류와 법적의무 재분류 중 무엇을 정본으로 운영할지, 두 기준 전환을 계속 노출할지 확정 필요(§7).
- **세트 ↔ 문서 매핑 기준** — 문서를 어떤 세트에 배정할지 정식 기준 확정 필요. 현재 일부 문서가 "세트 미지정" 상태이며, 미지정 문서의 분류·노출 정책 확정 필요(§7).
- **PDCA 단계·검증 판정 규칙** — 세트의 PDCA 구성 적정성을 정상·주의·오류로 판정하는 기준과, 주의·오류 세트의 후속 처리(누가 언제 보완) 확정 필요(§7).
- **상태(완료/진행/미시행) 판정 기준** — 문서 상태를 무엇으로 판정·집계하는지 기준 확정 필요(§7).
- **목록 다운로드** — 내려받기 형식·포함 컬럼·범위(현재 필터 반영 여부)·파일명 규칙 확정 필요(§7).
- **빈 PDCA 단계 표현** — 세트 구성상 단계는 있으나 문서가 없는 경우의 표기 정책 확정 필요(§7).
