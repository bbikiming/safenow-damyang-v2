# 화면 정의서 — 업무 목록(세트 리스트)

> 화면 ID: SCR-DOC-002 · 화면 유형: 목록 · 기획 관점(v6)
> 확정이 필요한 항목은 §7 미결사항에 모았습니다.

## 1. 화면 정보

| 항목 | 내용 |

**목적·설명** — PDCA 세트(업무 묶음) 단위로 업무문서 현황을 한눈에 보는 목록 화면이다. 대메뉴(법령 분류)별로 세트를 묶고, 세트 안의 문서를 PDCA 단계(계획·실행·점검·조치)로 정리해 보여준다. 상단에서 분류 기준(기존 분류 ↔ 법적의무 재분류)을 전환할 수 있으며, 기본은 법적의무 재분류 기준이다. 재분류 기준에서는 세트별 PDCA 구성의 적정성(정상·주의·오류)을 검증 배너로 함께 표시한다. 같은 세트·문서 데이터를 **두 가지 형태**로 본다 — 업무 목록(테이블형, docs-preset.html)과 이행 목록(격자 카드형, docs-exec.html)이며, 둘은 같은 구성 요소의 레이아웃 변형이다(데이터·필터·툴바·검증 배너 동일, 표시 형태만 다름).

목록은 조회·검색·이동에 더해 **등록·수정·삭제 진입점**을 제공한다 — 툴바의 [+ 업무 등록] 버튼으로 등록 마법사를 열고(SCR-DOC-007), 사용자가 등록한 문서 행에서는 [편집]·[삭제]를 직접 수행한다. 기본 제공(시드) 문서는 조회만 가능하다. 문서를 클릭하면 문서 상세로 이동하며, 문서의 처리 유형(전자문서·첨부파일·프로그램)에 따라 상세에서의 작업이 달라진다.

분류 기준별 규모는 화면 표기상 **법적의무 재분류(v2)=9대메뉴·53세트**, **기존 분류(v1)=51세트**로 안내된다.

## 4. 화면 구성별 정리

### 4-1. 상단 — 분류 기준 전환 / PDCA 검증 배너
- 구성: 분류 기준 토글(라벨 "분류 버전" + 버튼 2개: [기존 분류 (51세트)] / [법적의무 재분류 v2 (53세트)], 현재 기준 버튼에 선택 표시), (재분류 기준 + 전체 목록 화면에서만) PDCA 검증 배너. 대메뉴 화면에 끼워 넣은 형태(메뉴 임베드)에서는 분류 토글·검증 배너·대메뉴 탭을 표시하지 않는다.
- 검증 배너 문구: 모두 정상이면 "전체 N개 세트 모두 정상입니다 (오류 없음)", 주의·오류가 있으면 "PDCA 규칙 검증 — 정상 N · 주의 M · 오류 K (해당 세트ID 나열)". 현재 실측 표시는 주의 2건(발주 산업안전보건관리비 세트, 비상조치·대피·구호 세트, 둘 다 "점검(C) 후 조치(A) 단계 없음")·오류 0건이다. 주의·오류가 있을 때만 배너 우측에 [검토 대상만 보기] 토글이 나타난다(모두 정상이면 토글 없음).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 분류 기준 토글 | 선택한 기준의 세트 구성으로 목록 전체를 다시 구성(대메뉴 탭·검색·필터 모두 초기화) | 화면 내 재구성(이동 없음) |
| 검증 배너(재분류) | 세트 PDCA 구성의 정상·주의·오류 건수와 해당 세트ID를 요약 표시 | 주의·오류 세트는 카드 헤더에 경고 배지 표시 |
| [검토 대상만 보기] | 주의·오류 세트만 노출, 라벨이 [전체 보기]로 바뀜 | 다시 누르면 전체 노출 |

- 상태: 기본 분류 기준은 법적의무 재분류이며, 기존 분류는 토글 또는 URL 지정으로 본다. 검증 판정 규칙·후속 처리는 §6·§7 참조.

### 4-2. 툴바 — 검색·필터
- 구성: 대메뉴 탭([전체] + 9개 대메뉴), 문서명·담당자 검색 입력 + [검색] 버튼, 상태 셀렉트(상태 전체·완료·진행·미시행), PDCA 단계 필터 버튼군([전체]·P 계획·D 실행·C 점검·A 조치·미확정), 요약 텍스트("세트 N개 · 문서 M건 · 9개 대메뉴"), [+ 업무 등록] 버튼(재분류 기준 + 등록 기능 사용 시 노출), [목록 다운로드] 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 대메뉴 탭 | 해당 대메뉴 세트만 표시 | — |
| 문서명·담당자 검색 | 입력 즉시 부분일치로 좁힘([검색] 버튼은 표시용, 입력만으로 즉시 반영) | 0건이면 "조건에 맞는 문서가 없습니다" |
| 상태 셀렉트 | 선택한 상태(완료/진행/미시행)의 문서만 표시 | — |
| PDCA 단계 필터 | 선택 단계(P·D·C·A·미확정)의 문서만 표시 | — |
| [+ 업무 등록] | 등록 마법사 팝업 열림(현재 선택된 대메뉴 기본값), 저장 후 목록 재구성 | SCR-DOC-007 |
| [목록 다운로드] | 다운로드 안내 표시 | 4-5 참조 |

- 상태: 대메뉴 탭 + 검색 + 상태 + PDCA 단계 + (재분류 시) 검토 대상만 보기 조합이 함께 적용된다. 분류 기준을 바꾸면 모든 필터가 초기화된다. PDCA 필터에는 "미확정"(아직 단계가 확정되지 않은 문서) 항목이 포함되며, "단계 없음" 그룹은 표에는 나타나지만 필터 버튼에는 없다. 결과 0건 안내는 제공되나 로딩·조회 실패 상태 처리는 미정(§7).

### 4-3. 세트 카드 목록(테이블형)
- 카드 헤더 구성: 대메뉴 칩 + 세트명 + (기준문서 세트인 경우) "기준문서" 칩 + "N건 · 완료 M" + (재분류) PDCA 흐름 배지(예: P·D·C·A) + (재분류) 검증 배지(주의 ⚠ / 오류 ⛔, 검증 사유를 툴팁으로 표시) + (선행 관계가 있으면) "선행: …" 표기 + (세트 보조 설명이 있으면) 회색 보조 텍스트.
- 본문 표 구성: PDCA 단계(P 계획·D 실행·C 점검·A 조치·미확정·단계 없음) 순으로 그룹화하고, 단계 칸은 묶어서 표시한다. 문서가 있는 단계만 행을 만든다(빈 단계는 강제로 만들지 않음). 행 컬럼: 문서명(링크)·담당자·부서·수정일·상태칩·액션열.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 문서명 클릭 | 문서 상세로 이동 | 문서 상세(SCR-DOC-004~006) |
| 행 액션 클릭 | 문서 상세로 이동(처리유형별 라벨: 전자문서=[입력 →] / 첨부파일=[미리보기 →] / 프로그램=[이동 →] / 그 외=[상세 →]) | 문서 상세(SCR-DOC-004~006) |
| (사용자 등록 문서) [편집] | 기존 값이 채워진 수정 팝업 열림 | SCR-DOC-007 |
| (사용자 등록 문서) [삭제] | 삭제 확인 팝업 열림 → 4-6 참조 | — |
| (기본 제공 문서) | 편집·삭제 버튼이 표시되지 않음(조회 전용) | — |

- 상태(상태칩): 전자문서는 작성중 / 등록완료 / 확정 칩으로, 그 외 처리유형은 완료 / 진행 / 미시행 칩으로 표시된다. 완료 수 집계도 전자문서는 "확정" 기준, 그 외 처리유형은 "완료" 기준으로 센다.
- 상태(세트 미지정): 법적의무 재분류 기준의 기본 제공 문서는 모두 세트에 배정되어 있어 기본 제공 문서만으로는 "세트 미지정" 그룹이 생기지 않는다. 사용자가 세트를 지정하지 않고 등록한 문서만 "세트 미지정" 그룹으로 분리된다(해당 카드는 점선 테두리·"TODO 미지정" 표시).
- 상태(편집·삭제 대상): [편집]·[삭제]는 사용자 등록 문서에만 노출된다. 사용자 등록 문서는 새로고침 후에도 유지되고, 기본 제공 문서 원본은 바뀌지 않는다(§6).

### 4-4. 격자 카드(이행 목록, docs-exec.html)
- 구성: 업무 목록과 같은 세트·문서 데이터를 카드형으로 본다. 본문 하단에 "업무 목록과 동일한 세트·문서 데이터를 카드형으로 봅니다" 안내와 업무 목록(테이블형)으로 가는 링크가 있다.
- 카드 헤더: 세트명 + (검증 배지가 있으면 배지, 없으면 PDCA 흐름 또는 "마감: YY-MM-DD") + 진행률 바 + 진행률 %(완료/전체).
- 카드 본문: PDCA 단계 도트 + 단계별 문서 행(문서명 + 상태칩).
- 카드 푸터: 대표 담당자(이름·부서) + 대메뉴 칩(미지정 카드는 "TODO 미지정"). 미지정 카드는 점선 테두리.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 문서 행 클릭 | 문서 상세로 이동 | 문서 상세(SCR-DOC-004~006) |
| 업무 목록 링크 | 테이블형 업무 목록(docs-preset.html)으로 이동 | — |

- 상태: 분류 토글·검증 배너·대메뉴 탭·검색·필터·등록 진입점은 테이블형과 동일하게 동작하며, 표시 형태(진행률 바·% · PDCA 도트 · 대표 담당자 푸터)만 격자 카드 고유 요소다.

### 4-5. 목록 다운로드
- 구성: [목록 다운로드] 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [목록 다운로드] | 다운로드 안내 표시 | 현재 프로토타입은 고정 파일명·형식(엑셀 파일명, xlsx)이 안내에 그대로 표시되며 현재 필터는 반영되지 않음(실제 내보내기 미구현) — 형식·포함 컬럼·범위는 §6·§7 |

### 4-6. 삭제 확인 팝업(사용자 등록 문서)
- 구성: 대상 문서명·문서ID 표시 + 삭제 안내("삭제하면 목록·상세에서 제거됩니다. (되돌릴 수 없음)") + [삭제]·[취소]. 사용자 등록 문서만 대상이며, 삭제 동작은 등록 마법사와 같은 확인 흐름(SCR-DOC-007)을 사용한다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [삭제] | 문서 제거 후 업무 목록·이행 목록·문서 상세에서 즉시 제거 | "삭제되었습니다" |
| [취소] | 삭제 취소 | 팝업 닫힘 |

## 6. 정책

- **분류 기준 운영** — 기본은 법적의무 재분류 기준(화면 표기상 9대메뉴·53세트)이며, 기존 분류 기준(51세트)은 토글 또는 URL 지정으로 본다. 두 기준 전환을 계속 노출할지, 어느 기준을 정본으로 운영할지 확정 필요(§7).
- **세트 ↔ 문서 매핑 기준** — 법적의무 재분류 기준의 기본 제공 문서는 모두 세트에 배정되어 있어 기본 제공 문서만으로는 "세트 미지정"이 생기지 않는다. 사용자가 세트 없이 등록한 문서만 미지정 그룹으로 나타난다. 미지정 문서의 정식 분류·노출 정책 확정 필요(§7).
- **사용자 등록 문서 구분** — 사용자가 등록한 문서는 기본 제공(시드) 문서와 구분해 관리하며, [편집]·[삭제]는 사용자 등록 문서에 한한다(기본 제공 문서는 조회만). 사용자 등록 문서는 새로고침 후에도 유지되고 기본 제공 문서 원본은 변경되지 않는다. 등록·수정·삭제 결과는 업무 목록·이행 목록·문서 상세에 동일하게 즉시 반영된다(SCR-DOC-007 연계).
- **등록·수정·삭제 동선** — [+ 업무 등록]·[편집]은 업무 등록·수정 마법사(SCR-DOC-007)로, [삭제]는 같은 확인 흐름으로 연결된다.
- **처리유형에 따른 표시·동작** — 상태칩과 완료 집계가 처리유형에 따라 갈린다(전자문서=작성중/등록완료/확정 및 확정 기준 집계, 그 외=완료/진행/미시행 및 완료 기준 집계). 문서 상세에서도 첨부파일=미리보기·다운로드, 전자문서=입력 폼, 프로그램=연결 화면 이동으로 갈린다(SCR-DOC-004·005·006 연계).
- **PDCA 단계·검증 판정 규칙** — 세트 PDCA 구성 적정성을 정상·주의·오류로 판정하는 기준과 주의·오류 세트의 후속 처리(누가 언제 보완) 확정 필요. 현재 주의 사유는 "점검(C) 후 조치(A) 단계 없음" 유형이다(§7).
- **상태(완료/진행/미시행) 판정 기준** — 문서 상태를 무엇으로 판정·집계하는지 기준 확정 필요(§7).
- **목록 다운로드** — 내려받기 형식·포함 컬럼·범위(현재 필터 반영 여부)·파일명 규칙 확정 필요. 현재 프로토타입은 고정 파일명·형식 안내만 표시한다(§7).
- **빈 PDCA 단계 표현** — 세트 구성상 단계는 있으나 문서가 없는 경우의 표기 정책 확정 필요(§7).
- **이행 목록(격자 카드) 운영** — 업무 목록(테이블형)과 이행 목록(격자 카드형)은 같은 세트·문서 데이터의 표시 변형이다. 두 형태를 모두 유지할지, 진행률·마감일 표기 기준은 무엇으로 할지 확정 필요(§7).
