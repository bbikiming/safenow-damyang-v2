# 화면 정의서 — 메뉴 관리

> 화면 ID: SCR-ADMIN-005 · 화면 유형: 설정 · 기획 관점(v6)
> 확정이 필요한 항목은 §7 미결사항에 모았습니다.

## 1. 화면 정보

| 항목 | 내용 |
|---|---|
| 화면 ID | SCR-ADMIN-005 |
| 화면명 | 메뉴 관리 |
| URL 경로 | admin-menus.html |
| 상위 메뉴 | 시스템 관리 > 메뉴 관리 |
| 화면 유형 | 설정 (마스터-디테일) |

**목적·설명** — 시스템의 대메뉴 9개·중메뉴 31개 각각에 대해 "누가 이 화면을 볼 수 있고(보기), 등록·수정·삭제·상신까지 할 수 있는가(수정)"를 지정하는 접근 권한 설정 화면이다. 공통 권한 정의서가 정한 6등급 체계·초기 세팅 프리셋을 실제로 운영·변경하는 수단으로(왜: 로그인 연동 시 실제 접근 제어의 설정 데이터를 만드는 발판), 권한 관리(SCR-ADMIN-006)가 "누구인가"(등급 구성원)를 정의하면 이 화면은 "그 등급/부서/사용자가 어느 메뉴에 무엇을 할 수 있는가"를 메뉴 단위로 확정한다. 화면은 좌측 메뉴 트리와 우측 설정 패널(중메뉴 선택 시 기본 정보·접근 권한 / 대메뉴 선택 시 하위 요약·일괄 적용) 2개 영역으로 구성되며, 부서·사용자를 직접 지정할 때는 권한 관리와 공유하는 공용 조직도 선택기를 연다. 지정이 하나도 없는 메뉴는 기본 차단(시스템 관리자 제외)으로 처리한다. 실제 내비게이션 숨김·차단은 본개발 범위이며 이 화면은 설정만 다룬다.

## 4. 화면 구성별 정리

### 4-1. 상단 안내 바 · 데모 데이터 초기화
- 구성: 좌측에 화면 성격 안내 문구, 우측에 [데모 데이터 초기화] 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| (안내 문구) | 화면 성격 상시 표기 | "메뉴별 접근 권한을 설정합니다. 미설정 메뉴는 기본 차단(시스템 관리자 제외)이며, 실제 내비게이션 노출에는 영향을 주지 않습니다(프로토타입)." |
| [데모 데이터 초기화] | 확인 후 등급·지정·사용 여부를 초기 시드로 복원 | 확인: "메뉴·권한 데모 데이터를 초기 시드로 되돌립니다. 이 화면에서 변경한 내용이 모두 사라집니다. 계속하시겠습니까?" / 완료: "데모 데이터를 초기화했습니다." |

- 상태: 초기화 후에는 우측 패널 선택이 해제되어 "좌측에서 메뉴를 선택하면 접근 권한을 설정할 수 있습니다." 안내로 돌아간다.

### 4-2. 좌측 메뉴 트리
- 구성: 상단 검색 입력("메뉴명 검색"), 아래 대메뉴 9개 > 중메뉴 31개 트리(내비게이션 메뉴 구성과 단일 출처). 각 중메뉴에 상태 배지, 대메뉴에 펼침/접힘 화살표.
- 상태 배지 4종: `설정 N건`(지정 있음) · `미설정`(지정 0건 = 기본 차단) · `사용 안 함`(사용 여부 꺼짐) · `관리자 전용`(시스템 관리 하위 메뉴).

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 메뉴명 검색 입력 | 이름이 일치하는 중메뉴만 남기고 대메뉴는 펼쳐 노출. 일치하는 대메뉴명은 하위 전체 노출 | 결과 없으면 "검색 결과가 없습니다." |
| 대메뉴 화살표 | 해당 대메뉴 하위 중메뉴 펼침/접힘 | — |
| 대메뉴명 클릭 | 우측을 하위 요약 패널(4-8)로 전환 | 미저장 시 이탈 확인 |
| 중메뉴 클릭 | 우측을 그 메뉴 설정 패널(4-3~4-6)로 전환. 같은 메뉴 재클릭은 무시 | 미저장 시 이탈 확인 |

- 상태: 빈 상태(검색 미일치) "검색 결과가 없습니다.". 단계상태 = 각 중메뉴가 `설정 N건`→(전체 해제·저장)→`미설정`, 사용 여부에 따라 `사용 안 함`, 시스템 관리 하위는 항상 `관리자 전용`. 저장 시 배지가 즉시 갱신된다.

### 4-3. 우측 패널 — 기본 정보(중메뉴 선택 시)
- 구성: 메뉴명 · 대메뉴 · 경로 · 연결 화면 코드(모두 조회 전용) + 사용 여부 토글. 메뉴의 신설·삭제·순서 변경·명칭 수정은 이 화면 범위가 아니다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 사용 여부 토글(켬) | 이 메뉴를 사용으로 전환, 즉시 적용 | "메뉴 사용으로 설정되었습니다." / 텍스트 "사용" |
| 사용 여부 토글(끔) | 이 메뉴를 사용 안 함으로 전환(내비게이션 숨김 대상), 즉시 적용 | "메뉴 사용 안 함으로 설정되었습니다. (프로토타입 — 실제 내비게이션에는 영향 없음)" / 텍스트 "사용 안 함 (메뉴 숨김 대상 — 프로토타입은 표시만)" |

- 상태: 사용 여부는 접근 권한 지정과 무관하게 최우선 스위치다(꺼짐이면 전 등급에서 숨김 대상). 토글은 확정 저장을 기다리지 않고 즉시 반영되어 트리 배지가 `사용 안 함`으로 바뀐다.

### 4-4. 우측 패널 — 접근 권한(추가 수단)
- 구성: 전체 권한 자동 적용 안내줄 + 추가 방식 라디오("권한 등급 불러오기" / "부서·사용자 직접 지정") + 방식별 추가 속성창. 라디오는 메뉴별 배타 선택이 아니라 추가 입력 수단을 전환하는 토글이며, 두 방식으로 넣은 결과는 하나의 통합 목록(4-5)에 누적(합집합)된다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 전체 권한 안내줄 | 시스템 관리자 등급이 목록과 무관하게 자동 적용됨을 상시 표기 | "전체 권한 등급 1개(시스템 관리자)는 항상 접근·수정 가능합니다. 아래 목록과 무관하게 자동 적용됩니다." |
| "권한 등급 불러오기" 선택 | 등급 선택 목록 + [추가] 노출. 목록은 전체 권한 등급과 이미 추가된 등급을 제외. 남는 등급이 없으면 "추가할 등급 없음" | — |
| [추가](등급) | 선택 등급을 통합 목록에 지정 행으로 추가(기본 보기 체크·수정 해제) | 미선택 시 "추가할 등급을 선택하세요." |
| "부서·사용자 직접 지정" 선택 | [조직도에서 선택] 버튼 + 안내 "부서 또는 사용자를 복수 선택해 추가합니다." | — |
| [조직도에서 선택] | 공용 조직도 선택기(4-10) 열림 | 제목 "직접 지정 — {메뉴명}" |

- 상태: 통합 목록이 비면 "지정된 권한이 없습니다. 등급을 불러오거나 조직도에서 직접 지정하세요. (미설정 = 기본 차단, 시스템 관리자 제외)". 전체 권한 등급(시스템 관리자)은 등급 선택 목록·통합 목록 어디에도 나타나지 않는다.

### 4-5. 우측 패널 — 통합 지정 목록
- 구성: 유형(권한등급 / 부서 / 사용자) · 대상 · 보기 · 수정 · 관리 5열 표. 각 행에 유형 배지, 대상명, 보기·수정 체크박스, [제거]. 등급 행은 구성원 미리보기 펼침, 부서 행은 `하위 포함`/`직속만` 표기.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 수정 체크 | 해당 행 수정 부여 + 보기 자동 체크·잠금(수정 ⊃ 보기) | 하단 "'수정'은 등록·수정·삭제·상신을 포함합니다. 수정 체크 시 보기는 자동 포함됩니다." |
| 보기 체크 해제(수정 없음) | 보기 해제. 보기·수정 모두 해제 시 무권한 행 처리 | 행에 "권한 없음 — 저장 시 제거" 배지 |
| 등급 행 펼침 | 그 등급의 구성원(부서·인원) 미리보기 표시(조회 전용) | "구성원 미리보기 (N명)". 일반 공무원 등급은 "SSO 로그인 사용자 전원이 자동 포함됩니다." |
| [제거] | 해당 지정 행 삭제 | — |

- 상태: 부서 행은 조직도 선택 시의 포함 범위에 따라 `직속만` 또는 `하위 포함`으로 표기. 무권한 행은 강조 표시되며 저장 시 자동 제거된다(저장 결과 토스트에 제거 건수 포함). 단계상태 = 통합 목록 유무가 트리 배지 `설정 N건`↔`미설정`을 결정한다.

### 4-6. 적용 요약 · 저장 / 초기화 · 미저장 가드
- 구성: 하단에 "권한등급 N · 부서 N · 개인 N명" 요약 + [다른 메뉴 설정 복사]·[초기화]·[저장]. 미저장 변경이 있으면 접근 권한 제목 옆에 "저장 필요" 칩.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|

## 6. 정책

- **미설정 = 기본 차단** — 지정 행이 하나도 없는 메뉴는 시스템 관리자를 제외한 전원에게 기본 차단되고, 트리에 `미설정` 배지가 붙는다. "설정을 깜빡하면 열리는" 사고를 막는 화이트리스트 방식이다. 실제 내비게이션 숨김·대메뉴 노출 규칙(하위 중메뉴가 1개 이상 접근 가능할 때만 대메뉴 노출)의 강제 차단은 본개발 범위다(§7).
- **합집합·수정 우선** — 한 대상이 등급 경유와 직접 지정 등 여러 경로로 같은 메뉴에 권한을 받으면 가장 넓은 권한(수정 > 보기)을 실효값으로 삼는다. '수정'은 등록·수정·삭제·상신을 포함하며, 승인·반려·완료·공표 등 결재 단계 권한은 이 화면이 정하지 않고 각 화면 정책·온나라 결재선을 따른다(공통 권한 정의서 §4·§7).
- **전체 권한 등급의 자동 적용·설정 제외** — 전체 권한 등급(시스템 관리자)은 모든 메뉴에 자동 적용되며 메뉴별 지정 대상 목록에 나타나지 않는다. 관리자가 실수로 자기 접근을 없애는 셀프 잠금을 원천 차단한다.
- **명시 저장** — 접근 권한 변경은 [저장]으로만 확정한다(자동 저장 없음). 미저장 상태 이탈은 확인 다이얼로그로 보호한다. 반면 사용 여부 토글·일괄 적용·데모 초기화는 즉시 반영된다.
- **대체 방식 복사·일괄 적용** — [다른 메뉴 설정 복사]와 [하위 전체에 복사]는 대상 메뉴의 기존 설정을 대체하며, 실행 전 대상 수·대체 사실을 확인한다. 자기 자신·시스템 관리 메뉴는 소스/대상에서 제외하고, 미설정 소스를 고르면 기본 차단 경고를 추가한다. 시스템 관리 대메뉴는 일괄 적용 자체를 제공하지 않는다.
- **조직도 단일 출처** — 직접 지정에 쓰는 조직도는 경영방침(SCR-SBM-001)·의견청취(SCR-OPN-001)·인력 평가(SCR-EVL-001) 등 다른 화면의 조직도와 동일한 부서·인원을 참조하며, 부서 지정은 특정 시점 인원 스냅숏이 아니라 그 부서 소속 전체를 가리키는 동적 참조다.
- **프로토타입 표현 한계** — 로그인·실제 접근 강제가 없어 이 화면의 저장 결과·사용 여부 토글은 설정 데이터로만 존재하고 실제 내비게이션 노출에는 아직 영향을 주지 않는다(상단 안내로 명시).
- **연계** — 등급 정의·구성원은 권한 관리(SCR-ADMIN-006)에서 관리하고, 사용자별 실효 권한(등급+개별 지정 합집합)은 사용자 관리(SCR-ADMIN-001)의 접근 메뉴 역조회에서 확인한다. 초기 세팅 프리셋은 시스템 오픈 시 이 화면의 지정 행 기본값으로 일괄 제공된다.
