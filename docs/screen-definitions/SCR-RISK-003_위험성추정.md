# SCR-RISK-003 위험성 추정(공정별 실시)

> v6 · 기획 관점 · 위험성평가 모듈

---

## 1. 화면 정보

| 항목 | 내용 |
|---|---|
| 화면 ID | SCR-RISK-003 |
| 화면명 | 위험성 추정(공정별 실시) |
| URL 경로 | rsk-exec.html |
| 상위 메뉴 | 위험성평가 > 위험성평가 |
| 화면 유형 | 등록·수정 |

- 한 공정의 유해위험요인별 **위험성을 추정**하는 실시 화면이다. 작업공정에서 정리한 유해위험요인이 자동으로 호출되며 다시 입력하지 않는다.
- 추정 기법(빈도·강도, 3단계 판단법, 체크리스트법)을 전환해 위험성 등급과 허용/허용초과를 산정한다.
- 허용초과 항목에는 감소대책을 입력하면 **개선조치**로 배정되고, 조치 완료 후 **재평가**로 허용 여부를 재확인한다.
- 이전 3개 평가방법 화면(4M·매트릭스 / 체크리스트 / 유해위험요인별 O·X)을 하나의 실시 화면으로 통합한다.

## 4. 화면 구성별 정리

### 4-1. 헤더·기법 선택
- 구성: 공정명·담당 평가자·평가명, 추정 기법 선택(빈도·강도 / 3단계 판단법 / 체크리스트법).
- 상태: 완료 잠금 상태에서는 기법·입력이 비활성이며 보기 모드로 전환된다.

### 4-2. 유해위험요인 추정 표
- 구성: 유해위험요인(자동 호출)·법령근거, 빈도·강도(또는 판정), 위험성 등급, 허용 여부, 감소대책.
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 빈도·강도 입력 | 등급·허용 여부를 즉시 산정·표시 | 색으로 허용/허용초과 구분 |
| 허용초과 [대책 입력] | 감소대책 입력 패널을 펼친다 | 카드 안 인라인, 적층 팝업 없음 |

### 4-3. 감소대책 입력(허용초과 시)
- 구성: 대책 내용(표준안 자동 채움), 담당자, 조치 기한, 조치 전 사진.
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [개선조치 생성] | 감소대책을 개선조치로 배정 | "감소대책이 개선조치로 배정되었습니다" |

### 4-4. 완료·재평가
- 구성: [임시저장]·[이 공정 평가 완료], 조치 후 [재평가].
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [이 공정 평가 완료] | 입력·대책 충족 확인 후 잠금 | 미충족 시 부족 항목 안내 |
| [재평가] | 조치 후 상태로 재추정 시작 | 허용이면 종결, 허용초과이면 추가 대책 |

## 6. 정책

- **자동 호출:** 작업공정에서 정리한 유해위험요인을 그대로 불러오며 재입력하지 않는다.
- **기법·허용기준:** 추정 기법과 허용기준은 설정에서 관리하며, 평가 실시 시 전환할 수 있다. (근거: 위험성평가 고시)
- **허용초과 처리:** 허용초과 항목은 감소대책 입력이 있어야 평가를 완료할 수 있고, 감소대책은 개선조치로 일원화된다.
- **조치 후 재평가:** 조치 완료 후 동일 요인을 재추정해 허용 여부를 재확인하며, 이력은 보존한다. (근거: 산업안전보건법 제36조)
