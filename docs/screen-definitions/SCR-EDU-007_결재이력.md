## 4. 화면 구성별 정리

### 4-1. 요약 지표
- 구성: 전체 상신(건) · 결재중 · 결재완료 · 반려 4종.
- 상태: 필터와 무관하게 **전체 상신 건 기준**으로 집계한다(현재 조건의 결과 건수는 필터 바에 별도 표기).

### 4-2. 필터 바
- 구성: 검색어(문서번호·문서명·대상), 구분(전체/총괄/교육별/개인별), 상태(전체/결재중/결재완료/반려), 상신연도.
- 상신연도 옵션은 실제 상신 이력이 있는 연도만 제공한다.
- 우측: 결과 건수, [필터 초기화](적용 중인 조건 수 표시).

### 4-3. 이력 표
- 구성: 문서번호 · 상신일 · 구분 · 문서명 · 대상 · 결재선 · 상태 · 관리([문서] [상태]).
- 정렬: 상신일 최신순(같은 날짜는 나중에 상신한 건이 위).
- 구분 어휘: 총괄(연도·부서 범위 전체) / 교육별(교육 1건) / 개인별(대상자 1명).
- 대상 표기: 총괄은 부서 범위와 대상 연도, 교육별은 교육명, 개인별은 대상자와 소속 부서.
- 상태 어휘: 결재중 / 결재완료 / 반려. 같은 대상을 재상신해 최신이 아닌 건에는 **이전** 배지를 함께 표시한다.
- 대상이 삭제된 경우(교육 건 삭제·대상자 제외) 대상 칸에 삭제 사실을 표기해 이력 자체는 보존한다.

### 4-4. 행 액션
- 이벤트:

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [문서] | 상신 당시 조건(연도·부서·서식·대상)을 복원해 결재 문서 미리보기를 연다 | 문서 미리보기 |
| [상태] | 그 상신 건의 문서번호·상신일·결재선·상태 이력을 보여준다 | 결재 상태 |
| [결재 완료 처리] / [반려 처리] (최신 건) | 상태를 진행하고 목록·요약을 갱신한다 | "결재 상태: 결재완료 · 문서등록번호 회신 등록(목업)" |
| [상태] (이전 건) | 이력만 열람하고 상태 변경 버튼을 제공하지 않는다 | "이 문서는 이후 재상신되어 최신 건이 아닙니다" |

### 4-5. 빈 상태
- 상신 이력이 하나도 없을 때: 각 교육 화면의 상신 진입점에서 문서를 올리면 이 화면에 쌓인다는 안내를 표시한다.
- 조건에 맞는 건이 없을 때: 조건을 좁혀 결과가 없음을 알린다.

## 6. 정책

- **조회 전용:** 이 화면은 상신 건을 만들지 않는다. 문서 작성·상신은 각 교육 화면의 진입점에서만 하고, 여기서는 열람·상태 확인·재상신만 한다.
- **이력 보존:** 재상신하면 이전 건을 덮어쓰지 않고 별도 이력으로 남기며, 최신이 아닌 건은 상태를 되돌릴 수 없다(결재 이력의 사후 변조 방지).
- **상태 단일 출처:** 여기서 진행한 결재 상태는 교육 카드·상세·이수현황의 결재 상태 표시와 같은 값을 바라보므로, 어느 화면에서 바꾸든 전 화면에 동일하게 반영된다.
- **시연 회신:** 결재완료·반려 처리는 프로토타입 시연용이며, 실제 연계에서는 온나라가 결재 결과와 문서등록번호를 회신한다.
