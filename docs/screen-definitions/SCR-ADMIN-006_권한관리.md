# 화면 정의서 — 권한 관리

> 화면 ID: SCR-ADMIN-006 · 화면 유형: 등록·수정(마스터-디테일) · 기획 관점(v6)
> 확정이 필요한 항목은 §7 미결사항에 모았습니다.

---

## 1. 화면 정보

| 항목 | 내용 |
|---|---|
| 화면 ID | SCR-ADMIN-006 |
| 화면명 | 권한 관리 (ADM02-S) |
| URL 경로 | admin-roles.html |
| 상위 메뉴 | 시스템 관리 > 권한 관리 (하위 3번째 · 메뉴 관리 다음) |
| 화면 유형 | 등록·수정 (좌 목록 + 우 상세 마스터-디테일) |

**목적·설명** — 시스템이 인정하는 **권한 등급**을 정의하고, 각 등급에 **누가 속하는가**(부서·개인 구성원)를 지정·관리하는 화면이다. 도입 근거는 중대재해처벌법 대응 통합관리시스템의 접근 통제를 운영 가능한 형태로 만드는 것으로, 공통 권한 정의서가 등급 체계(6등급)와 초기 세팅 값의 단일 출처라면 본 화면은 그 등급의 **구성원 배정을 실제로 바꾸는 운영 수단**이다. 화면은 상단 안내 바 + **좌측 권한 등급 목록** + **우측 등급 상세**(기본 정보·속성·구성원·적용 현황)의 3영역과, 구성원 추가에 쓰이는 **공용 조직도 선택기**(단일 모달)로 구성된다. 이 화면은 "누구인가"만 다루고, 그 등급이 **어느 메뉴를 보기/수정할 수 있는가는 메뉴 관리(SCR-ADMIN-005)에서 메뉴별로 설정**한다 — 이 경계(공통 권한 정의서의 결정 D1)가 화면 전반의 안내 문구로 반복 노출된다. 상단 고정 안내: "권한 등급과 구성원을 관리합니다. 보기·수정 권한은 메뉴 관리에서 메뉴별로 설정합니다."

## 3. 진입 / 이탈 경로 및 사용자 시나리오

**진입 / 이탈**

| 구분 | 연결 화면 | 조건 / 트리거 |
|---|---|---|
| 진입 | 시스템 관리 GNB > 권한 관리 | 메뉴 클릭 → 좌 등급 목록 로드, 우 패널은 미선택 안내 |
| 진입 | 사용자 관리(SCR-ADMIN-001) 하단 등급 요약 | "권한 관리로 이동" 링크(사용자 관리 측 진입점) |
| 화면 내 이동 | 좌 등급 카드 ↔ 우 상세 | 등급 카드 선택 시 우 상세 로드(미저장 변경 있으면 이탈 확인) |
| 화면 내 이동 | 우 상세 → 조직도 선택기 | [+ 조직도에서 추가] → 단일 모달 |
| 연계(참조만) | 메뉴 관리(SCR-ADMIN-005) | 적용 현황·안내 문구가 "메뉴 관리에서 설정"을 가리킴(자동 이동 아님) |
| 이탈 | 다른 GNB 메뉴 | 미저장 변경이 있으면 이탈 확인 다이얼로그, 없으면 그대로 이동 |

**사용자 시나리오**

| 단계 | 사용자 행동 | 화면 반응 |
|:--:|---|---|
| 1 | 권한 관리 진입 | 좌측에 등급 목록(기본 6등급), 우측은 "좌측에서 등급을 선택하거나 [+ 등급 추가]로 새 등급을 만드세요." 안내 |
| 2 | 등급 카드 선택 | 우측에 기본 정보·속성·구성원·적용 현황이 로드됨 |
| 3 | [+ 조직도에서 추가] | 조직도 선택기(단일 모달) 열림 — 제목 "구성원 추가 — {등급명}" |
| 4 | 부서·개인 복수 선택 후 [적용 (N건)] | 구성원 목록에 추가, 결과 안내 토스트, 우 상세가 '저장 필요' 상태로 전환 |
| 5 | [저장] | 변경 확정 후 "저장되었습니다." · 좌 목록의 구성원 수 갱신 |
| 6 | [+ 등급 추가] | 우 패널이 신규 등급 입력 모드로 전환(모달 아님) |
| 7 | 등급명·설명·속성 입력 후 [저장] | 등급명 검증 통과 시 "등급이 추가되었습니다." · 좌 목록에 새 등급 표시 |
| 8 | (삭제) [등급 삭제] → 확인 | 적용 메뉴 수를 경고에 표기, 확인 시 "등급이 삭제되었습니다." + 메뉴 지정 연쇄 제거 |

**분기 로직**

- **선택 상태에 따른 우 패널 분기** — 아무 등급도 선택하지 않으면 우 패널은 안내만 표시. 기존 등급을 고르면 상세(기본 정보·속성·구성원·적용 현황·액션)를, [+ 등급 추가]를 누르면 동일 위치에 **신규 등급 모드**를 연다. 신규 모드는 카드 제목이 "새 등급"이며 삭제 버튼과 적용 현황 표가 없고, 적용 현황 자리에는 "등급을 저장한 뒤 메뉴 관리에서 이 등급을 메뉴에 지정하면 여기에 표시됩니다." 안내가 놓인다.
- **속성 플래그에 따른 구성원·적용 현황 분기** — '전체 권한'을 켜면 적용 현황이 표 대신 "전체 권한 등급 — 전 메뉴(N개)에 자동 접근·수정합니다."로 바뀐다. '구성원 자동(전 직원)'을 켜면 구성원 영역이 개별 목록 대신 "SSO 로그인 사용자 전원이 자동 포함됩니다. 개별 구성원 지정은 사용하지 않습니다."로 대체된다.
- **보호 등급 분기** — 보호(전체 권한 속성) 등급은 등급명·설명·속성 입력이 잠기고(읽기 전용), 삭제 버튼이 노출되지 않으며, 구성원 최소 1명 유지 규칙이 적용된다. 보호 여부는 등급명이 아니라 **전체 권한 플래그가 켜졌는가**로 판정한다(기본 제공 시스템 관리자 등급이 이에 해당).

**엣지케이스**

- **미저장 이동(dirty guard)** — 우 상세에서 값이 바뀐 뒤(저장 전) 다른 등급 카드로 이동하거나 [+ 등급 추가]를 누르면 "저장하지 않은 변경사항이 있습니다. 이동하면 변경 내용이 사라집니다. 계속하시겠습니까?" 확인을 거친다. 취소하면 현재 등급에 머문다.
- **초기화** — [초기화]는 편집 중 변경분을 되돌린다(기존 등급은 저장값으로, 신규 모드는 빈 값으로). "변경 내용을 되돌렸습니다." 안내.
- **속성 조합 경고** — '전체 권한'과 '구성원 자동(전 직원)'을 동시에 켜면 상세에 인라인 경고가 뜬다(저장은 차단하지 않음). 한쪽을 해제하면 경고가 사라진다.
- **구성원 중복** — 이미 구성원인 대상을 다시 추가하면 중복 제외 후 안내한다. 부서와 그 부서 소속 개인이 함께 들어오면 별도 안내한다(실효 권한은 합집합).
- **빈 상태** — 등급 미선택 시 우 패널 안내. 구성원이 없는 등급은 "구성원이 없습니다. [+ 조직도에서 추가]로 부서·사용자를 선택하세요." 표시. 적용된 메뉴가 없는 등급은 적용 현황에 "아직 적용된 메뉴가 없습니다." 표시.

## 4. 화면 구성별 정리

### 4-1. 상단 안내 바
- 구성: 좌측 고정 안내 문구 + 우측 [데모 데이터 초기화] 버튼.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| (표시) | 화면 목적과 D1 경계를 항상 안내 | "권한 등급과 구성원을 관리합니다. 보기·수정 권한은 메뉴 관리에서 메뉴별로 설정합니다." |
| [데모 데이터 초기화] | 확인 후 초기 시드로 등급·구성원·메뉴 지정 복원 | 확인: "권한 등급 데모 데이터를 초기 시드로 되돌립니다. 변경 내용이 모두 사라집니다. 계속하시겠습니까?" · 완료: "데모 데이터를 초기화했습니다." |

- 상태: 데모 초기화는 프로토타입 시연용 복원 수단이다. 실행 시 선택·편집 상태가 모두 해제되고 우 패널은 미선택 안내로 돌아간다.

### 4-2. 좌측 권한 등급 목록
- 구성: [+ 등급 추가] 버튼 + 등급 카드 목록. 기본 6등급 — 시스템 관리자(보호) · 군수 · 부군수·국장 · 과장·팀장 · 전담부서(중대재해 총괄) · 일반 공무원. 각 카드는 등급명, 구성원 수 요약, 적용 메뉴 수 요약을 표기한다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [+ 등급 추가] | 우 패널을 신규 등급 입력 모드로 전환 | 미저장 변경 시 이탈 확인 |
| 등급 카드 선택 | 우 패널에 해당 등급 상세 로드, 선택 카드 강조 | 미저장 변경 시 이탈 확인 |

- 구성원 수 표기 규칙: 개별 구성원 등급은 "구성원 N명", 구성원 자동 등급은 "구성원 자동(전 직원)". 부서 구성원은 하위 포함 여부에 따라 실인원을 합산한 값이다.
- 적용 메뉴 수 표기 규칙: 전체 권한 등급은 "전체 메뉴", 그 외는 "적용 N개"(메뉴 관리에서 이 등급이 지정된 중메뉴 수). 보호 등급 카드에는 "보호" 배지를 부기한다.
- 상태: 목록은 항상 최소 6개(기본 등급)를 표시하며 신규 추가 시 카드가 늘어난다. 빈 목록 상태는 없다(기본 등급 삭제 불가·자동 등급 상시 존재).

### 4-3. 우측 — 기본 정보 · 속성
- 구성: 카드 제목("기본 정보" / 신규 모드는 "새 등급") + 상태 배지, 등급명(필수), 설명, 속성 체크박스 2개('전체 권한' · '구성원 자동(전 직원)').

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 등급명·설명 입력 | 편집 중 상태로 전환('저장 필요') | 등급명 도움말: "예: 안전관리자" / 설명 도움말: "등급의 용도를 설명하세요" |
| '전체 권한' 체크 | 적용 현황이 자동 적용 안내로 전환 + 안내 노출 | "전체 권한 — 메뉴별 설정 없이 모든 메뉴에 접근·수정합니다. (메뉴 관리의 개별 지정 대상에서 제외)" |
| '구성원 자동' 체크 | 구성원 영역이 자동 포함 안내로 전환 | "SSO 로그인 사용자 전원이 자동 포함됩니다. 개별 구성원 지정은 사용하지 않습니다." |
| 두 속성 동시 체크 | 인라인 경고 노출(저장 차단 없음) | "전 직원에게 모든 메뉴의 보기·수정 권한이 부여되는 조합입니다. 의도한 설정인지 확인하세요." |

- 상태 배지: 편집 중이면 "저장 필요", 보호 등급이면 "보호 등급 · 속성 잠금". 보호 등급 본문에는 "보호 등급입니다. 등급명·속성 변경과 삭제가 잠겨 있으며, 구성원은 최소 1명을 유지해야 합니다. (셀프 잠금 방지)" 안내가 상시 표시된다.
- 상태: 보호 등급은 등급명·설명 입력과 속성 체크가 모두 잠긴 읽기 전용이다. 신규 등급 모드는 기본 정보만 비어 있는 상태로 시작한다.

### 4-4. 우측 — 구성원
- 구성: 구성원 수 요약 배지 + [+ 조직도에서 추가] 버튼 + 구성원 행 목록(부서/사용자 유형 배지 · 대상명 · 제거). 구성원 자동 등급은 목록 대신 자동 포함 안내로 대체된다.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [+ 조직도에서 추가] | 조직도 선택기(단일 모달) 열림(§4-7) | 제목 "구성원 추가 — {등급명}" |
| 조직도 [적용 (N건)] | 선택 부서·개인을 구성원 목록에 추가 | "N건 추가"(중복 시 "· N건 중복 제외" 추가, 부서·소속 개인 겹침 시 "· 부서와 그 소속 개인이 중복 지정되어 있습니다(실효 권한은 합집합)" 추가) |
| 구성원 행 [제거] | 해당 부서·개인을 목록에서 제거 | 보호 등급 마지막 1명 제거 시 차단: "보호 등급은 구성원을 최소 1명 유지해야 합니다. (셀프 잠금 방지)" |

- 구성원 행 표기: 부서 구성원은 "부서" 배지 + 부서명 + 하위 포함 여부 배지("하위 포함" / "직속만"). 사용자 구성원은 "사용자" 배지 + 이름 + 소속 부서·직책.
- 상태: 구성원 없음 → "구성원이 없습니다. [+ 조직도에서 추가]로 부서·사용자를 선택하세요." · 구성원 자동 등급 → 자동 포함 안내(개별 추가 버튼·목록 없음).

### 4-5. 우측 — 적용 현황
- 구성: "이 등급이 적용된 메뉴 N개" 요약 + 대메뉴 > 중메뉴 · 보기 · 수정 4열 읽기 전용 표 + D1 경계 안내.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| (표시) | 이 등급이 메뉴 관리에서 지정된 중메뉴와 보기·수정 값을 읽기 전용으로 나열 | 안내: "보기·수정 권한은 메뉴 관리에서 메뉴별로 설정합니다. 아래는 읽기 전용 현황입니다." |

- 표 구성: 대메뉴 · 중메뉴 · 보기(O/-) · 수정(O/-). 값은 메뉴 관리(SCR-ADMIN-005)의 지정 결과를 반영한 읽기 전용이며 이 화면에서 편집하지 않는다.
- 상태: 신규 등급 → "등급을 저장한 뒤 메뉴 관리에서 이 등급을 메뉴에 지정하면 여기에 표시됩니다." · 전체 권한 등급 → "전체 권한 등급 — 전 메뉴(N개)에 자동 접근·수정합니다." · 지정 없음 → "아직 적용된 메뉴가 없습니다."

### 4-6. 우측 — 액션(저장·초기화·삭제)
- 구성: [등급 삭제](기존·비보호 등급만) · [초기화] · [저장].

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| [저장] | 등급명 검증 후 변경 확정 | 신규: "등급이 추가되었습니다." / 기존: "저장되었습니다." (오류는 §5) |
| [초기화] | 편집 중 변경분을 저장값(신규는 빈 값)으로 되돌림 | "변경 내용을 되돌렸습니다." |
| [등급 삭제] | 적용 메뉴 수 경고 확인 후 등급 삭제 + 메뉴 지정 연쇄 제거 | 적용 중: "이 등급은 N개 메뉴에 적용 중입니다. 삭제하면 해당 메뉴에서 지정이 제거됩니다. 계속하시겠습니까?" / 미적용: "이 등급을 삭제하시겠습니까?" · 완료: "등급이 삭제되었습니다." |

- 상태: [등급 삭제]는 신규 모드와 보호 등급에서는 노출되지 않는다. 저장 후에는 '저장 필요' 상태가 해제되고 좌 목록이 갱신된다. 삭제 후 우 패널은 미선택 안내로 돌아간다.

### 4-7. 공용 조직도 선택기 (단일 모달)
- 성격: 메뉴 관리의 '직접 지정'과 권한 관리의 '구성원 추가'가 **함께 쓰는 공용 컴포넌트**다. 아래는 권한 관리 컨텍스트(구성원 추가) 기준 기술이다. 화면에 떠 있는 모달은 항상 1개이며, 우 상세 패널은 모달이 아닌 인라인 영역이므로 선택기가 열려도 모달은 정확히 1개다.
- 구성: 좌측 조직도 트리(검색 + 전체 펼치기/접기) + 우측 선택 목록(바스켓) + 하단 [취소] · [적용 (N건)]. 제목은 호출 컨텍스트 표기 — 구성원 추가 시 "구성원 추가 — {등급명}".

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 검색 입력 | 부서명·이름·직책으로 필터, 매칭 상위 조직 자동 펼침·매칭 강조·무관 가지 숨김, 입력 후 짧은 지연 후 반영 | 결과 없음: "검색 결과가 없습니다." |
| [전체 펼치기] / [전체 접기] | 트리 전체를 펼치거나 접음 | — |
| 부서 체크 | 선택 목록에 부서 추가(직속 인원 수 표기) | 도움말: "부서·이름·직책 검색" |
| '하위 포함' 체크 | 하위 조직 전체를 동적 포함, 포함 인원수 실시간 갱신, 하위 부서·개인 자동 체크·비활성 | 하위 조직 있는 노드·부서 체크 시에만 활성(기본 비활성) |
| 개인 체크 | 선택 목록에 개인 추가 | — |
| 선택 목록 [×] / [제거] | 해당 항목을 선택에서 제거 | — |
| [적용 (N건)] | 선택 대상을 구성원 목록에 반영 후 모달 닫힘 | 0건이면 비활성 |
| [취소] | 선택을 버리고 닫음 | 재오픈 시 초기 상태로 시작 |

- 트리 기본 상태: 국 단계까지 펼쳐 모든 부서명이 보이도록 제공(부서 계층 노출). 이미 구성원인 대상은 "추가됨" 배지 + 비활성으로 재추가를 막는다(부서 단위로 포함된 하위 대상 포함).
- 직위 노드 예외: 군수·부군수 등 직위 노드는 부서 단위 선택 대상이 아니며 소속 개인만 체크할 수 있다.
- 하위 포함 규칙: 하위 조직이 있는 부서에만 노출되고, 그 부서를 직접 체크했을 때만 활성화된다(기본 비활성). 부서 체크를 해제하면 하위 포함도 함께 해제·비활성되고 리셋된다. 켜면 하위 부서·개인이 자동 체크·비활성되어 중복 선택을 막는다.
- 상태: 선택 0건이면 바스켓은 "좌측에서 부서·인원을 선택하세요."이고 [적용]은 비활성. 선택이 있으면 바스켓에 유형 배지(부서/사용자)·대상명·하위 포함 배지를 표기하고 상단 "선택 목록 N건"과 [적용 (N건)]이 실시간 갱신된다.

## 6. 정책

- **등급은 "누구"만 정의(D1 경계)** — 권한 관리는 등급과 그 구성원(누가)만 다루고, 그 등급이 어느 메뉴를 보기/수정하는가(무엇을)는 메뉴 관리(SCR-ADMIN-005)에서 메뉴별로 설정한다. 적용 현황 표와 상단 안내가 이 경계를 반복 노출한다. 이 경계 문구의 최종 표현은 §7에서 확정.
- **보호 규칙은 등급명이 아닌 플래그 기준** — 삭제 불가·구성원 최소 1명 유지는 특정 등급명이 아니라 '전체 권한' 플래그가 켜진 모든 등급에 적용한다. 이는 시스템 관리자 등급이 개명되거나 관리자 등급이 여러 개로 늘어나도 셀프 잠금 방지가 깨지지 않게 하기 위함이다. 신규 등급의 두 플래그 허용 범위·경고 강도는 §7에서 재확인.
- **부서 지정 = 동적 포함** — 구성원에 부서를 추가하면 특정 시점 인원을 고정하는 스냅숏이 아니라 그 부서 소속 전체를 가리키는 참조다. 전입자는 자동 포함, 전출자는 자동 제외된다(SSO·조직도 동기화 전제). 개인 단위 지정은 자동 갱신 대상이 아니며 인사이동 시 사용자 관리(SCR-ADMIN-001)에서 정리한다.
- **실효 권한 합집합** — 한 사용자가 여러 등급·개별 지정에 겹쳐도 실효 권한은 합집합(수정 > 보기)이다. 부서와 그 소속 개인이 함께 지정돼도 중복은 제외하고 합집합으로 해석한다.
- **조직도 단일 출처** — 이 화면의 조직도 선택기는 경영방침 점검자·의견청취 위원·인력평가 평가자 선택 등 다른 화면과 동일한 조직도(부서·인원)를 바라본다. 조직 구성 변경은 모든 화면에 동일하게 반영된다.
- **초기 세팅 값의 출처 분리** — 기본 6등급과 초기 구성원·초기 메뉴 지정은 공통 권한 정의서(등급 체계·초기 세팅 프리셋)를 단일 출처로 한다. 이 화면은 오픈 이후의 구성원 변경 수단이며, 시연용 복원은 [데모 데이터 초기화]로 제공한다.
- **명시적 저장** — 구성원·속성 변경은 자동 저장하지 않고 [저장]으로 확정하며, 미저장 이동 시 확인 다이얼로그를 거친다.
