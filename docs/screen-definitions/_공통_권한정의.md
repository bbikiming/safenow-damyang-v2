# 공통 권한 정의

> 권한(등급별 접근·기능 허용 범위)을 각 화면 정의서에 중복하지 않고 **이 문서 한 곳에서 관리**합니다.
> 각 화면 정의서의 §2 권한 정보는 본 문서 **§6 화면별 권한 매핑의 해당 [SCR-ID] 행**을 참조합니다.

---

## 1. 문서 목적 · 연계 규칙

- **단일 출처(SSOT)**: 역할·접근·기능 권한은 본 문서에서만 확정·관리한다.
- **연계 방식**: 각 화면 정의서 §2 권한 정보 → 본 문서 **§6 [SCR-ID] 행**을 참조한다. 화면 정의서는 1줄 요약만 두고, 권한 표를 따로 복제하지 않는다.
- **상속**: §6에 개별 행이 없는 화면은 본 문서 **§5 화면군 마스터 매트릭스**의 해당 그룹 값을 따른다.
- **확장**: 새 화면 정의서를 작성할 때 §6에 그 화면의 [SCR-ID] 행 1줄을 추가한다.
- **확정 절차**: 권한이 확정되면 (1) §5/§6의 `[확인필요]`를 확정값으로 갱신, (2) §7 미결사항에서 제거, (3) 영향 화면 정의서 §2의 고유 예외만 점검.
- **운영 반영(v2 신설)**: 운영 시 본 문서의 권한은 시스템 관리 > **메뉴 관리(SCR-ADMIN-005)·권한 관리(SCR-ADMIN-006)** 화면에서 관리·변경된다. 본 문서는 그 두 화면이 시스템 오픈 시점에 반영해야 할 **등급 체계(§3)·초기 세팅 값(§5)** 의 단일 출처이며, 오픈 이후의 실제 변경은 두 화면에서 이루어진다.

---

## 2. 권한 표기 규칙

| 표기 | 의미 |
|---|---|
| O | 접근/기능 사용 가능 |
| X | 접근/기능 사용 불가 |
| 읽기전용 | 화면·데이터는 보이나 쓰기(등록·수정·삭제·결재) 불가 |
| 숨김 | 메뉴·버튼·화면 자체를 노출하지 않음 |
| 비활성 | 노출되나 클릭·입력 불가 |
| 화면정책 (v2 신설) | 이 문서·메뉴 권한 매트릭스가 값을 정하지 않고 **개별 화면 정책(화면 정의서 §2)·온나라 결재선**을 그대로 따름(승인·반려·완료·공표 등 결재 단계 권한) |
| [확인필요] | 정책 미확정 → 기획/발주처 확정 필요(§7) |

> **현 상태 주의**: 아래 매트릭스의 O/X는 시스템 오픈 시점에 적용되는 **초기 세팅 프리셋**이다(§5). 현재 프로토타입에는 로그인·역할·접근제어가 구현되어 있지 않아 실제로는 전 사용자에게 모든 화면·기능이 동일하게 노출된다(§4).

## 3. 역할 정의 (v2 — 6등급 체계)

> 2026-07-08 "시스템 관리 > 메뉴 관리·권한 관리" 기획 확정(`../planning/기획-시스템관리-메뉴권한-검토-v1.md`)에 따라 구 4역할 체계를 폐지하고 아래 6등급으로 개정한다. 등급은 시스템 관리 > **권한 관리(SCR-ADMIN-006)** 화면에서 구성원(부서·사용자)을 관리하고, 실제 메뉴별 보기·수정 권한은 **메뉴 관리(SCR-ADMIN-005)** 화면에서 등급 경유 지정 또는 부서·사용자 개별 지정으로 부여한다(§4).

| 등급 | 구성원 | 초기 프리셋 · 속성 |
|---|---|---|
| 시스템 관리자 | 전담팀 관리자(개인) | 전체 권한(모든 메뉴 자동 적용) · 보호 등급(삭제 불가·구성원 최소 1명 유지) |
| 군수 | 개인 1 | 경영책임자(중대재해처벌법상) — 초기 프리셋: 전 메뉴 보기 |
| 부군수·국장 | 개인 소수 | 초기 프리셋: 전 메뉴 보기 |
| 과장·팀장 | 직책자 개인 지정 원칙(부서 단위로 지정하면 부서원 전원이 포함되므로 지양) | 초기 프리셋: 전 메뉴 보기 |
| 전담부서(중대재해 총괄) | 재난안전과 중대재해팀(부서, 하위 조직 포함) | 초기 프리셋: 전 업무 메뉴 수정 |
| 일반 공무원 | 자동 포함(전 직원 · SSO 로그인 사용자 전원) | 초기 프리셋: 전 메뉴 보기 + 의견청취·건의함 수정 |

### 구 역할 → 신 등급 매핑

| 구 역할(v1) | 신 등급(v2) | 비고 |
|---|---|---|
| 시스템 관리자 | 시스템 관리자 | 동일 |
| 담당자 | 전담부서(중대재해 총괄) 또는 메뉴별 개별 지정 | 중대재해팀 고유 업무는 전담부서 등급, **부서별 업무 담당자(예: 보건소 담당자 → 안전보건교육)는 등급이 아니라 메뉴 관리의 개별 지정**으로 부여 |
| 부서장(관리책임자) | 과장·팀장 | 동일 대상, 명칭만 직위 기준으로 구체화 |
| 열람자 | 일반 공무원 | '열람자' 등급은 폐지되고 일반 공무원(자동 전 직원)으로 흡수 |
| (없음) | 군수 · 부군수·국장 | 상위 열람 계층 신설 — 구 체계에는 대응 역할이 없었음 |

> **호환 규칙**: 기존 화면 정의서(§2 권한 정보 요약, 아래 §6 화면별 매핑의 기존 행)에 남아있는 구 역할 표기(`담당자`·`부서장`·`열람자` 등)는 위 매핑표로 해석한다. 각 화면 정의서를 재작성·개정하는 시점에 신 등급 표기로 갱신하며, 이 개정만으로 기존 정의서 문구를 일괄 치환하지는 않는다.

## 4. 공통 적용 원칙

- **현재 구현 상태**: 로그인·역할·접근제어가 없어 전 사용자에게 동일 노출되며, 사용자·작성자·결재선은 고정 표시값이다. 본 문서의 권한은 구현 목표이자 확정 대상이다.
- **권한 설정의 단위(v2)**: 보기·수정 권한은 **메뉴 × 지정(등급 또는 개별 부서·사용자)** 단위로 메뉴 관리 화면에서 설정한다. 권한 관리 화면의 등급은 "**누가**"(구성원)만 정의하며, "**무엇을**"(그 메뉴에서 보기/수정 가능 여부)은 메뉴 관리의 지정 행에서 정한다 — 등급 자체에는 메뉴별 권한 값을 붙이지 않는다.
- **등급 지정 · 개별 지정 병존(v2)**: 하나의 메뉴에 등급 경유 지정과 부서·사용자 개별 지정이 동시에 걸릴 수 있다(예: 전담부서 등급의 수정 + 보건소 담당자 개별 지정의 수정 + 상위 등급들의 보기가 한 메뉴에 동시 적용). 동일 대상에 권한이 중복 부여되면 **높은 권한 우선(합집합)** 을 적용한다.
- **'수정'의 범위(v2)**: '수정' 권한은 등록·수정·삭제·상신을 포함한다. 승인·반려·완료·공표·발송 등 **결재 단계 권한은 이 매트릭스가 정하지 않고 각 화면 정책(화면 정의서 §2)과 온나라 결재선**을 따른다(§2 "화면정책" 표기, §7 PERM-004·005).
- **미설정 메뉴 기본 정책(v2)**: 메뉴 관리에서 아무 지정도 없는 메뉴는 시스템 관리자를 제외하고 **기본 차단**한다. 접근 불가 메뉴는 GNB·LNB에서 **숨김** 처리하며, 대메뉴는 하위 중메뉴가 하나라도 접근 가능해야 노출한다.
- **데이터 스코프(Phase 2)**: "본인 담당만/소속 부서만/전체"와 같은 데이터 노출 범위는 이번 메뉴·권한 체계의 범위에서 **제외**하고 Phase 2 과제로 다룬다(§7 PERM-002). 그 전까지 각 화면은 현재 정책대로 별도 운용한다.
- **조회성 vs 쓰기성**: 대시보드·통계·정보센터 등 조회성 화면은 전 등급 열람을 기본으로 하고(§5), 등록·수정·삭제·상신 등 쓰기성 기능은 전담부서·시스템 관리자를 기본으로 한다.
- **시스템관리 영역**: 사용자 관리·메뉴 관리·권한 관리·알림 관리·프리셋 양식 관리·연계 관리 등 시스템 관리 화면은 시스템 관리자 전용을 기본으로 한다(§5 "시스템 관리" 행).
- **역할 부여 방식(v2)**: 등급·개별 지정 모두 **SSO 조직도 연동을 전제로 한 수동 지정**이다(권한 관리·메뉴 관리 화면에서 조직도로 선택). 연계로 자동 부여되지 않는다. 단, 일반 공무원 등급만 예외로 **SSO 로그인 사용자 전원이 자동 포함**된다.
- **일반 공무원 예외**: 일반 공무원의 초기 프리셋은 전 메뉴 보기(읽기전용)이나, **의견청취·건의함은 예외로 등록(수정) 권한**을 가진다(전 직원 제출이 전제인 기능 — §5-1 각주, §6 SCR-OPN-001).

## 6. 화면별 권한 매핑 (SCR-ID 키) — 연계 지점

> 각 화면 정의서 §2는 아래 해당 [SCR-ID] 행을 참조한다. 행이 없는 화면은 §5 그룹 매트릭스를 상속한다.
> **v2 표기 안내**: 아래 표의 `담당자`·`부서장`·`열람자` 등 구 역할 표기는 §3 "구 역할 → 신 등급 매핑"으로 해석한다. 각 화면 정의서 재작성 시 신 등급 표기로 갱신한다.
