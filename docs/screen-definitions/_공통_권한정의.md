# 공통 권한 정의

> 권한(역할별 접근·기능 허용 범위)을 각 화면 정의서에 중복하지 않고 **이 문서 한 곳에서 관리**합니다.
> 각 화면 정의서의 §2 권한 정보는 본 문서 **§6 화면별 권한 매핑의 해당 [SCR-ID] 행**을 참조합니다.

---

## 1. 문서 목적 · 연계 규칙

- **단일 출처(SSOT)**: 역할·접근·기능 권한은 본 문서에서만 확정·관리한다.
- **연계 방식**: 각 화면 정의서 §2 권한 정보 → 본 문서 **§6 [SCR-ID] 행**을 참조한다. 화면 정의서는 1줄 요약만 두고, 권한 표를 따로 복제하지 않는다.
- **상속**: §6에 개별 행이 없는 화면은 본 문서 **§5 화면군 마스터 매트릭스**의 해당 그룹 값을 따른다.
- **확장**: 새 화면 정의서를 작성할 때 §6에 그 화면의 [SCR-ID] 행 1줄을 추가한다.
- **확정 절차**: 권한이 확정되면 (1) §5/§6의 `[확인필요]`를 확정값으로 갱신, (2) §7 미결사항에서 제거, (3) 영향 화면 정의서 §2의 고유 예외만 점검.

---

## 2. 권한 표기 규칙

| 표기 | 의미 |
|---|---|
| O | 접근/기능 사용 가능 |
| X | 접근/기능 사용 불가 |
| 읽기전용 | 화면·데이터는 보이나 쓰기(등록·수정·삭제·결재) 불가 |
| 숨김 | 메뉴·버튼·화면 자체를 노출하지 않음 |
| 비활성 | 노출되나 클릭·입력 불가 |
| [확인필요] | 정책 미확정 → 기획/발주처 확정 필요(§7) |

> **현 상태 주의**: 아래 매트릭스의 O/X는 **목표 정책안**이다. 현재 프로토타입에는 로그인·역할·접근제어가 구현되어 있지 않아 실제로는 전 사용자에게 모든 화면·기능이 동일하게 노출된다(§4).

---

## 3. 역할 정의

| 역할 | 책임 범위(제안) | 비고 |
|---|---|---|
| 시스템 관리자 | 시스템 전반(사용자·권한·설정·연계·로그) 관리 | 시스템관리 화면 전용 |
| 담당자 | 본인 배정 업무(점검·평가·조치·문서 작성·상신) 수행 | 데이터 스코프(본인 담당 범위) [확인필요] |
| 부서장(관리책임자) | 소관 업무 총괄·승인·결재 | 직위-역할 매핑 [확인필요] |
| 열람자 | 조회 전용(대시보드·보고서 등) | 외부 점검·감사용 운용 여부 [확인필요] |

> 역할명·직위 매핑·다중 역할(겸직)·역할 부여 방식(연계 자동/수동 지정)은 §7에서 확정한다.

---

## 4. 공통 적용 원칙

- **현재 구현 상태**: 로그인·역할·접근제어가 없어 전 사용자에게 동일 노출되며, 사용자·작성자·결재선은 고정 표시값이다. 본 문서의 권한은 구현 목표이자 확정 대상이다.
- **데이터 스코프**: 담당자·부서장은 본인 배정 시설/사업장 범위만 보고 다루는 것을 원칙으로 한다(범위·적용 여부 [확인필요]).
- **조회성 vs 쓰기성**: 대시보드·통계·정보센터 등 조회성 화면은 전 역할 열람을 기본으로, 등록·수정·삭제·결재 등 쓰기성 기능은 열람자를 제외한다(세부 [확인필요]).
- **시스템관리 영역**: 사용자·권한·연계·시스템 설정 화면은 시스템 관리자 전용을 기본으로 한다.
- **결재·공표**: 상신·승인·반려·완료·공표 등 단계 권한은 역할별로 구분하며, 구체 주체는 §7에서 확정한다.

## 6. 화면별 권한 매핑 (SCR-ID 키) — 연계 지점

> 각 화면 정의서 §2는 아래 해당 [SCR-ID] 행을 참조한다. 행이 없는 화면은 §5 그룹 매트릭스를 상속한다.

| SCR-ID | 화면 | 접근(조회) | 핵심 기능 허용(제안) | 비고 |
|---|---|---|---|---|
| SCR-SBM-001 | 경영방침 | 담당부서 · 부서장 · 열람자(읽기) | 등록·수정·상신·게시 = 담당자 / 승인·반려 = 부서장 [확인필요] | 현재 결재 상태를 권한 없이 임의 전환 가능 → 실제 권한 게이트 [확인필요] |
| SCR-DOC-002 | 업무 목록(세트 리스트) | 전 역할 조회(제안) | 다운로드·상세이동 = 담당자 이상 [확인필요] | 담당 범위 한정 노출 여부 [확인필요] |
| SCR-DOC-007 | 업무 등록·수정(마법사) | 전 역할 조회(제안) | 등록·수정·삭제 = 담당자 이상 / 삭제는 사용자 등록 문서 한정 [확인필요] | 본인 담당 범위 한정 등록·수정 여부, 부서장 삭제 허용 범위 [확인필요] |
| SCR-RISK-001 | 위험성평가 목록 | 전 역할 조회(제안) | 정기/수시 평가 등록 = 담당자 이상 [확인필요] | 부서·담당자별 노출 범위 [확인필요] |
| SCR-EDOC-005 | 점검표(F5) 폼 | 점검 수행자 · 부서장(승인) · 열람자(읽기) | 작성·등록 = 담당자 / 확정·상신 = 담당자 / 승인 = 부서장 [확인필요] | 작성자·결재선이 고정값 → 실제 구성 규칙 [확인필요] |
| SCR-OPN-001 | 의견청취 (SCR-OPN-001~003 공통) | 종사자(등록·본인 조회) · 담당자 · 부서장 · 열람자(읽기) [확인필요] | 의견 처리(조치 선택·완료·반려·재처리) = 담당자 / 회의록 확정·점검결과지 승인 = 부서장 [확인필요] | 오프라인 대리 등록 담당자 범위, 종사자에게 처리 기능 비노출 여부 [확인필요] |
| SCR-EVL-001 | 인력 평가 (SCR-EVL-001~003 공통) | 평가 총괄 · 평가자 · 부서장 [확인필요] | 평가 등록 = 지정 평가자 본인만(구현됨) / 평가자 지정·결재 상신 = 평가 총괄 / 평가 설정 접근 = CEO·GM·SHM(안내만, 차단 미구현) [확인필요] | 피평가자 본인 결과 열람 범위 [확인필요] |

> 그 외 화면(54개 중 미작성분)은 §5 그룹 매트릭스를 따르며, 화면 정의서 작성 시 본 표에 행을 추가한다.
