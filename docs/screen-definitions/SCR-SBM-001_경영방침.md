# 화면 정의서 — 경영방침

> 화면 ID: SCR-SBM-001 · 화면 유형: 목록·등록·상세 · 기획 관점(v7.1 — 유효성·경우의 수·정합성 보강)
> 본 정의서는 프로토타입 v2의 **현재 구현 상태**(버전 목록형 경영방침 관리 + 현행본 기준 이행점검)를 기준으로 작성되었다. v7.1에서 유효성 검사(§5)·예외·경계 시나리오(§3)·데이터 정합성(§6)을 기획 관점에서 보강했다. 확정이 필요한 항목은 §7 미결사항에 모았다.

## 1. 화면 정보

| 항목 | 내용 |
|---|---|
| 화면 ID | SCR-SBM-001 |
| 화면명 | 경영방침 |
| URL 경로 | menu.html?m=policy |
| 상위 메뉴 | 안전보건관리체계 > 경영방침 |
| 화면 유형 | 목록 · 등록(제정/개정/최초) · 상세 |
| 구성 파일 | menu.html, menu.js(policy 모듈), edoc.js / edoc-templates.js(이행점검 F5·법령사전), common.js(모달·토스트) |
| 관련 SFR | SFR-005 (안전·보건 목표와 경영방침) |

**목적·설명** — 담양군의 안전·보건 목표와 경영방침을 **버전 단위(제정/개정/최초)** 로 등록·관리하고, 각 버전을 온나라 결재에 상신하여 확정하며, 확정된 **현행본**에 대해 반기 1회 이행점검과 게시·주지(게시 전/후 증빙)를 수행하는 안전보건관리체계 최상위 업무 화면이다.

화면은 공통 셸(페이지 타이틀 + 요약 바) 위에 얹히며, 그 아래 **3개 서브탭**(경영방침 / 이행 점검 / 게시 현황)으로 골격을 이룬다. 각 서브탭은 **목록(list)** 을 기본으로 하고, 행을 클릭하면 **공유 상세(detail)** 로 진입한다(세 탭이 동일 상세를 공유). 경영방침 탭에서는 추가로 **등록 화면(form)** 으로 전환된다.

핵심 설계 원칙 두 가지:
1. **현행본 = 결재 승인완료된 확정 문서** — 버전 숫자가 아니라 *결재가 승인완료된 문서 중 시행일이 가장 최근인 버전*을 현행본으로 본다. 이행점검 작성·게시·증빙 등록은 **현행본에서만** 가능하다.
2. **이전 버전은 이력 열람 전용** — 현행본이 아닌 버전은 점검·게시·증빙의 신규 작성/수정/삭제가 불가하며, 점검표·게시·증빙을 **읽기만** 할 수 있다.

반기 이행점검에서 부적합(X) 판정된 항목은 개선조치(SCR-IMP 계열)로 자동 연계된다.

현재 프로토타입의 데이터는 `window.POLICY_STATE`(인메모리 SSOT)에 보관되어 **SPA 세션 동안 유지**되며(버전 추가·게시·증빙·점검 결과 반영), 전체 새로고침 시 초기 시드로 리셋된다. 단, 이행점검표 인스턴스는 e-Doc 엔진에 별도 보관된다(세션 종료 후에도 유지). 실제 영구 저장·발송·온나라 연계 규칙은 §6·§7에서 확정한다.

연관 화면: 이행관리(SCR-CMP 계열, 점검 상태 표시·역진입), 개선조치(SCR-IMP 계열, 부적합 항목 유입), 업무 목록(docs-preset.html?menu=policy).

## 4. 화면 구성별 정리

### 4-1. 공통 셸 — 페이지 타이틀 / 요약 바 (경영방침 고유 아님)
- 구성: 페이지 타이틀("경영방침") + 서브타이틀("안전보건관리체계 > 경영방침 · SFR-005 · 문서 N건"). 그 아래 **요약 바**(전 메뉴 공통) — "이행률 N%" 진행바(완료 문서 비율, 70↑ 양호·40↑ 주의·그 외 위험 색)·"시기도래 N건/없음"·"담당부서 재난안전과 중대재해팀".
- 상태: 요약 바 수치는 문서 데이터 기반 동적 표시. 별도 로딩 없이 즉시 표시. 모든 메뉴가 공유하는 공통 셸로 경영방침 고유 영역이 아니다.
- **변경 이력**: 이전 버전 정의서의 상단 [업무 기능]/[문서] 2탭은 **제거**되었다. 경영방침 화면은 곧바로 업무 기능(3 서브탭)으로 진입하며, 문서 분류 보기는 업무문서 > 업무 목록(docs-preset.html)에서 제공한다.

### 4-2. 서브탭 바 (경영방침 / 이행 점검 / 게시 현황)
- 구성: 3개 서브탭 버튼(`.psub-tabs`). 현재 활성 탭이 강조된다. 클릭 시 해당 목록으로 전환되며, 상세 화면은 세 탭이 공유한다(상세에서 [‹ 목록]을 누르면 직전 활성 탭의 목록으로 복귀).

| 트리거 | 동작 결과 |
|---|---|
| [경영방침] | 버전 목록(4-3) 표시 |
| [이행 점검] | 이행점검 목록(4-6) 표시 |
| [게시 현황] | 게시 현황 목록(4-9) 표시 |

- 상태: 라우팅은 인메모리 SSOT(`POLICY_STATE.tab/view/ver`)로 관리된다. 서브탭 전환 시 view는 목록(list)으로 초기화된다.

### 4-3. 경영방침 목록 (서브탭: 경영방침)
- 구성: 카드 제목 "경영방침 (N건)" + [+ 등록] 버튼. 표 컬럼 — 버전(`vX.X` + 제정/개정/최초 칩) · 방침명 · 반기 점검(최근 점검 완료일 또는 '미점검') · 온나라 결재 상태(미상신/결재중/승인완료/반려 칩). 행 클릭 시 상세로 진입.
- 등록된 방침이 하나도 없으면 빈 안내 + [최초 등록] 버튼만 표시된다.
- 시드 데이터: v2.0(제정·승인완료·현행), v1.1(개정·승인완료), v1.0(최초·승인완료).

|---|---|---|
| 행 클릭 | 상세(4-5)로 진입 | — |
| [+ 등록] | 등록 구분 선택 팝업(4-4) | 버전이 없으면 곧바로 '최초'(v1.0) 등록 화면으로 |

### 4-4. 경영방침 등록 (구분 선택 팝업 → 등록 화면)
- **구분 선택 팝업**: 현재 최신본 기준으로 [개정](+0.1)/[제정](+1.0)을 선택. 각 버튼에 다음 버전 번호와 성격(개정=소폭 보완·갱신, 제정=전면 재수립)이 표기된다. 버전 번호 규칙 — 최초=1.0, 개정=직전 +0.1, 제정=정수부 +1.0.
- **등록 화면(form)**: 상단 취소 바("경영방침 {구분} 등록 · v{번호}") + 입력 그리드.
  - 목표(슬로건, 텍스트, 필수 — 미입력 시 "목표를 입력하세요" 토스트로 저장 차단)
  - 세부 방침(줄 단위 GUI — 행별 입력 + ×삭제, [+ 세부 방침 추가]로 빈 줄 추가·포커스. 저장 시 빈 줄은 자동 제외)
  - 점검자(읽기전용 표시 + [조직도] 버튼 → 조직도 트리 오버레이)
- 방침 제호는 "담양군 안전·보건 경영방침"으로 고정. 시행일은 등록일로 채워지고 결재 상태는 '미상신'으로 시작.

| 트리거 | 동작 결과 | 이동 / 메시지 |
|---|---|---|
| 팝업 [개정]/[제정] | 구분·버전 확정 후 등록 화면으로 | — |
| [+ 세부 방침 추가] | 세부 방침 입력 줄 1개 추가 + 포커스 | — |
| 세부 방침 줄 [×] | 해당 줄 제거 | — |
| [조직도] | 조직도 트리 오버레이(부서·이름) → 선택 시 점검자 칸 채움 | — |
| [저장] | 목표 미입력 시 차단 / 정상 시 새 버전을 목록 최상단(unshift)에 추가하고 상세로 이동 | "경영방침 vX (구분) 등록 완료" |
| [취소]/[‹ 취소] | 목록으로 복귀 | — |

- 상태: 저장된 버전은 인메모리 SSOT에 즉시 반영되어 목록·상세에서 보인다(세션 유지, 전체 새로고침 시 초기화 — §6). 입력 검증은 목표 필수 외 미정의(§5·§7).

### 4-6. 이행 점검 목록 + 상세 점검 이력 (서브탭: 이행 점검)
- **목록(renderListCheck)**: 상단 안내 배너 — "결재 승인완료된 **현행본만** 이행 점검을 작성할 수 있습니다. 이전 버전은 **점검 이력 열람만** 가능합니다. 현행본: vX". 표 컬럼 — 버전 · **상태(현행/이력 칩)** · 방침명 · 점검 여부(완료/미점검) · 점검일 · 점검 결과(적합/부적합/보완 필요 칩) · 점검표(액션). 푸터에 [현행본 이행점검 작성].
- **점검표 컬럼 액션 분기**:

- **상세 내 '반기 이행 점검 이력' 섹션**: 회차·점검일·점검자·결과 표 + 회차별 [점검표 보기]. 현행본이면 [이행점검 작성] 버튼, 이전 버전이면 버튼 없이 🔒 "이전 버전입니다. 점검 이력 열람만 가능하며 신규 점검 작성·수정은 현행본에서만 할 수 있습니다." 안내.

### 4-9. 게시 현황 목록 + 상세 게시 섹션 (서브탭: 게시 현황)

| [확정 · 온나라 결재 상신] | 상태 '확정' + X 항목마다 개선조치 자동 생성 + 결재요청 안내 | "X 항목 N건이 개선조치로 자동 등록되었습니다" |

- 상태: 작성중 → 등록완료 → 확정. 점검표 인스턴스는 e-Doc 엔진에 보관되며 변경 시 화면이 즉시 갱신된다. 전 항목 판정 강제·X 사유 필수 여부는 미정(§5·§7). X 판정 자동 개선조치는 기한·담당 산정 규칙 미정(§7).

### 4-13. 마스터 관리 모달군 (코드상 정의·현재 상세에서 직접 노출은 [작성 가이드]만)
- 구성: 근거법령·고려사항 관리(법령/조항/세부내용 표 + 구분/고려사항 표, 행 추가/삭제), 점검표 관리(영역/항목/결과유형/근거 표, 행 추가/삭제), 점검표 관련 근거(조문 상세) 설정 모달이 코드(PG.policyBasis/policyChecklist 등)로 정의되어 있다. 각 [+ 추가]는 단일 오버레이 규칙으로 입력 팝업을 띄운다.
- 입력 검증: 대부분 미입력값을 기본값(`'설정 시'`, `'O / X'` 등)으로 대체해 검증 없이 행이 추가된다. 단 **조문 상세의 '근거 표기'는 필수**("근거 표기를 입력하세요" 토스트로 차단)이며, **점검표 관련 근거 설정은 현행본에서만** 가능하다("현행(결재 승인완료) 방침에서만 …" 차단).
- 상태: 재개편된 상세 화면에서는 **[작성 가이드] 배지만 노출**되며, 근거법령·고려사항·점검표 관리 진입 배지는 제거되었다(코드 핸들러는 유지). 각 모달의 [저장]은 현재 안내만 표시하고 실제 데이터(점검표 F5 프리셋 등)에 반영되지 않는다(§7). 점검표 관리 모달의 항목 수(6)와 실제 F5 점검표(7) 불일치는 §6·§7.

## 6. 정책 (업무 규칙 · 데이터 정합성)

- **현행본 판정 기준(단일성)** — 현행본은 `결재 승인완료` 버전 중 **시행일 최신본**으로 산출한다(버전 숫자 아님). 시스템 관점에서 현행본은 항상 0건 또는 1건이어야 한다. 결재 승인된 버전이 둘 이상이면 시행일이 가장 최근인 것을 현행으로 보며, **동일 시행일·다중 승인본**의 우선순위와 **현행 전환 시점**(승인 즉시 vs 시행일 도래)은 확정 필요(§7 SBM-04).
- **버전 번호 정합성** — 최초=1.0, 개정=+0.1, 제정=정수부 +1.0의 자동 채번이다. 버전 번호는 중복되지 않아야 하며(자동 채번이라 충돌 가능성은 낮으나), 수동 지정·연도 기반 등 대안 채택 시 **중복 차단 규칙**을 함께 정의해야 한다(§7 SBM-11).
- **편집 잠금 경계** — 현행본만 이행점검 작성·게시 추가/삭제·증빙 추가/삭제가 가능하고, 이전 버전은 점검표·게시·증빙을 읽기만 한다(목록 액션·상세 버튼·직접 호출 모두 동일 가드). 이전 버전 본문(목표·세부방침) 수정 가능 여부는 미정(현재 본문 직접 수정 UI 없음, 개정으로 새 버전 생성 — §7 SBM-05).
- **점검 결과 ↔ 개선조치 일관성** — 이행점검 확정 시 X(부적합) 항목마다 개선조치가 자동 생성된다. 점검표의 X 건수와 자동 생성 개선조치 건수가 일치해야 하며, 기한·담당자 산정 규칙(현재 데모값)을 확정해야 한다(§7 SBM-08).
- **점검항목 마스터 단일화** — 점검표 관리 모달(6항목)과 실제 F5 점검표(7항목)가 다르며 관리 모달 편집이 F5에 반영되지 않는다. 점검항목을 단일 마스터로 통합하고, 관리 모달의 F5 반영·노출 여부를 정해야 한다(개발 시 반영).
- **필수 라벨 ↔ 실제 강제 정합** — 게시 위치·게시일에 '필수' 라벨이 있으나 실제로는 빈 값이 자동 대체되어 강제되지 않는다. 라벨과 검증 동작을 일치시켜야 한다(개발 시 반영).
- **저장 경계(영구화)** — 버전 추가·게시·증빙·점검 결과는 인메모리 SSOT(`POLICY_STATE`)에 반영되어 세션 동안 유지되나 전체 새로고침 시 초기 시드로 리셋된다. 이행점검표 인스턴스만 e-Doc 엔진에 별도 보관되어 세션 종료 후에도 유지된다. 실서비스 영구 저장 대상·시점·연동 범위는 개발 시 반영한다(개발 진행 중 자연 해소 — 미결 제외).
- **상신·게시 선행조건** — 미상신·미점검 상태에서도 상신·게시가 가능하다. 상신 가능 선행 상태(등록 완료 등), 결재 승인 후에만 게시·공표를 허용할지 확정 필요(§7 SBM-06).
- **온나라 결재 전환·채번** — 결재 상태 전환은 시뮬레이션이며 권한·결재선·선행조건이 적용되지 않는다. [확정] 정식 문서번호는 **온나라 채번을 정본**으로 하며(실연계 시 자동 수신, 미연계 시 결재 완료 후 담당자 수동 입력, 결재 전 '미채번'), 실제 온나라 연동·승인/반려 자동 갱신 스펙은 §7 SBM-07에서 관리한다.
