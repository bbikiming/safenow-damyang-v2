# 99. 미결사항 목록 (Outstanding Items)

> 담양군 중대재해예방 시스템 프로토타입-v2 전 화면 코드 분석 기반.
> 각 화면 분석의 `undefinedPolicies`(미확정 정책)와, 개발자가 임의 판단하게 되는 5개 취약영역
> (① 버튼 동작·이동경로 / ② 유효성·오류메시지 / ③ 권한 차이 / ④ 상태 처리 / ⑤ 미확정 정책)을 통합했습니다.
> 동일 정책이 여러 화면에 영향을 주는 경우 **하나의 공통 항목**으로 통합하고 영향 범위에 전 화면을 표기했습니다.

## 개요

- **총 미결 항목 수: 35건**
- 카테고리별: 버튼동작 7건 · 유효성 4건 · 상태 5건 · 권한 4건 · 정책 15건
- **공통 전제(전 화면 동일):** 프로토타입-v2는 빌드/프레임워크 없는 정적 HTML + 전역 IIFE 스크립트(layout.js·common.js·edoc.js 등) 구조이며, 데이터는 대부분 하드코딩 또는 localStorage 목업입니다. 서버/DB·인증·외부 연계가 실제로 구현된 곳은 없습니다.
- **3대 전역 미구현(아래 표에서 PER-01 / POL-01 / POL-02로 통합):**
  1. **권한·로그인·접근제어 전무** — 모든 메뉴·버튼이 누구에게나 동일 노출, 사용자/결재선/작성자명 하드코딩.
  2. **입력 유효성 검증 전무** — `collect()`가 무검증 수집, required/형식/범위/정합성·오류 메시지 거의 없음.
  3. **외부 연계·알림 전부 시뮬레이션** — 온나라/e호조/FMS/조직도/SMS·메일이 토스트·안내 팝업 수준.

## 우선 처리 권고 (개발 임의판단 위험 Top)

1. **POL-06 데이터 정합성 결함** — data.js 문서에 `status` 필드가 없어 통계 이행표·이행률이 전부 0% 출력. 통계/대시보드 핵심 수치가 무의미한 상태로, 상태 데이터 모델 확정이 시급합니다.
2. **PER-01 / PER-03 권한 부재** — 인증·역할이 전무하여 누구나 확정·종결·마감·삭제 가능. 결재성 업무의 신뢰성에 직결됩니다.
3. **STT-02 / STT-04 비가역 확정 + 온나라 미연동** — 확인 절차 없이 확정 시 이수 가산·개선조치 자동생성·상신이 즉시 발생하나 반려 롤백 경로가 없습니다.
4. **BTN-05 / STT-03 상세 식별자·상태 권위 소스** — 행 클릭이 고정 화면으로 가고 상태가 URL/localStorage로만 결정되어 라우팅·상태 모델 재설계가 필요합니다.
5. **VAL-01~04 유효성 전무** — 빈 값·비정합 입력(출석>대상, 종료일<시작일 등)으로도 등록·확정 가능합니다.
