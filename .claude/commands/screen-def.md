너는 담양군 중대재해 프로토타입의 **화면 정의서**를 작성한다. 대상: **$ARGUMENTS**

반드시 아래 순서를 지킨다. 임의로 건너뛰지 않는다.

1. **하네스·기준 읽기** — 다음을 먼저 읽는다:
   - `docs/screen-definitions/_작성하네스.md` (작성 하네스 — 절대 원칙·절차·깊이 기준·체크리스트·금지 토큰)
   - `docs/screen-definitions/_화면정의서_템플릿_v6.md` (7섹션 템플릿)
   - `docs/screen-definitions/_규칙.md` (자동 규칙)
   - `docs/screen-definitions/SCR-DOC-007_업무등록마법사.md` (**골드 스탠다드** — 이 깊이·톤을 목표로)

2. **프로토타입 관찰** — 대상 화면의 실제 HTML/JS를 **직접 읽어** 진입점·영역·버튼/동작·메시지·상태·입력 폼·다른 화면 연계를 수집한다. **추측 금지**. 프로토타입에 있는 동작은 빠뜨리지 말고, 없는 기능은 지어내지 않는다.

3. **작성** — 7섹션 화면 정의서를 **골드 수준 깊이**로, **기획 관점만**(코드 토큰 금지) 작성한다. 미확정은 본문에 흩지 말고 §7에 `담당·기한`과 함께 모은다. 관련 화면은 SCR-ID로 상호참조한다.

4. **등록** — `docs/screen-definitions/` 에 `SCR-...md`로 저장한다(파일명은 `_inventory_rows.json`의 `defFile`과 일치). 신규 화면이면 `_inventory_rows.json`에 행을 추가하고, `_공통_권한정의.md` §6 화면별 권한 매핑에 해당 `[SCR-ID]` 행을 추가한다.

5. **검증·재생성** — `python3 docs/screen-definitions/_lint.py <파일>` 로 위반 0을 확인한다(위반 시 수정 후 재검사). 그다음 `python3 docs/screen-definitions/_build-data.py` 로 뷰어 데이터를 재생성한다.

6. **자가검증·보고** — `_작성하네스.md` §4 체크리스트를 항목별로 점검했음을 확인하고, 무엇을 어떤 근거(프로토타입의 어느 동작)로 적었는지 간단히 보고한다.

원칙: 기획 의도와 화면 설계만 적는다. 구현 방식은 개발이 정한다. 프로토타입과 어긋나게 쓰지 않는다.
