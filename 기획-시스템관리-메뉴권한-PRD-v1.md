| ID | 결정 사항 | 확정 내용 |
|---|---|---|
| D1 | 보기·수정 체크의 위치 | 메뉴 관리의 지정 행에서 설정한다. 권한 관리는 등급 정의·구성원 관리만 전담하며 체크박스를 두지 않는다. '전체 권한'은 등급 속성으로 재정의해 시스템 관리자 전용으로 모든 메뉴에 자동 적용하고, 메뉴별 설정 대상에서 제외한다. |
| D2 | 등급 불러오기·직접 지정의 관계 | 두 방식은 병존하며 결과는 합집합이다. 라디오는 메뉴별 배타 선택이 아니라 추가 방식을 전환하는 입력 수단이고, 적용 결과는 하나의 통합 목록에 누적된다. 동일 대상에 권한이 중복되면 높은 권한(수정)이 우선한다. |
| D3 | 기본 권한 등급 체계 | 시스템 관리자 · 군수 · 부군수·국장 · 과장·팀장 · 전담부서(중대재해 총괄) · 일반 공무원 6등급을 기본 제공한다. 부서별 업무 담당자는 등급이 아니라 메뉴 관리의 개별 지정으로 처리한다. |
| D4 | 미설정 메뉴 기본 정책 | 지정 행이 없는 메뉴는 시스템 관리자를 제외한 전원에게 기본 차단되고 GNB에서 숨겨진다. 시스템 오픈 시 초기 세팅 프리셋을 일괄 제공한다. |
| D5 | '수정' 권한의 범위 | 등록·수정·삭제·상신을 포함한다. 승인·반려·공표 등 결재 단계 권한은 각 화면 정책과 온나라 결재선 영역으로 존치한다. |
| D6 | 데이터 스코프 | 본인/부서/전체 등 데이터 노출 범위는 이번 범위에서 제외하고 Phase 2 후보로 남긴다. |
| D7 | 시스템 관리 하위 재편 | '사용자·권한'을 '사용자 관리'로 존치하고(사용자별 실효 권한 역조회 추가) '메뉴 관리'·'권한 관리'를 신설한다. |
| D8 | 조직도 선택기 형태 | 복수 선택형 단일 모달로 만든다. 계층 트리 체크, 선택 바스켓, 하위 부서 포함 옵션을 포함한다. |

## 1. 배경 · 목적

`_공통_권한정의.md`는 역할·접근 권한의 단일 출처(SSOT)로 만들어졌으나, 실제로 역할을 누구에게 어떻게 부여할지를 다루는 화면이 없다. `admin-users.html`은 사용자별 권한·담당 대메뉴를 표로 보여주지만 값이 고정 표기이며 실제 부여·회수 UI가 없다. 이 때문에 화면정의서 8종이 참조하는 권한 체계가 "정의는 있지만 운영 수단이 없는" 상태로 남아 있었다.

이번 기획은 "권한 관리(누구인가) + 메뉴 관리(어디에 접근하나)" 2개 화면을 신설해 권한 배정을 실제로 운영 가능한 형태로 만든다. 이는 로그인 연동에 따른 실접근 제어(본개발 범위)의 설정 데이터를 만드는 발판이며, `_공통_권한정의.md`의 미결사항 중 다음 항목을 해소하거나 경계를 명확히 한다.

| PERM ID | 항목 | 본 PRD의 처리 |
|---|---|---|
| PERM-001 | 역할-직위/부서 매핑 | 해소 — §5 6등급 정의 + 구 4역할 → 신 6등급 매핑표 |
| PERM-002 | 데이터 스코프 | 범위 외 유지 — §2, Phase 2 후보(D6) |
| PERM-003 | 다중 역할·겸직 | 해소 — 겸직 허용 + 실효 권한 합집합 원칙을 명문화(§4·§5) |
| PERM-004 | 결재 단계 권한(상신·승인·반려) | 범위 외 유지 — 화면 정책·온나라 결재선 영역으로 존치(D5, §2) |
| PERM-005 | 삭제·완료·공표 권한 | 부분 해소 — 삭제는 '수정' 게이트에 포함(D5). 완료·공표는 화면 정책 영역으로 유지 |
| PERM-006 | 열람자(감사용 읽기 전용) 운용 | 미해소 — 신규 등급 생성으로 유사 구성은 가능하나 운용 정책 자체는 미확정(§12 후속 검토) |
| PERM-007 | 역할 부여 방식(자동·수동) | 해소 — 등급 불러오기(구성원 자동 포함)와 직접 지정(수동)의 병존으로 확정(§4·§5) |
| PERM-008 | 접근제어 구현 범위(메뉴·데이터) | 부분 해소 — 메뉴·버튼 노출 제어는 확정(§4·§6·§7). 데이터 접근 제어(스코프)는 Phase 2(D6) |

본 PRD가 확정되면 `_공통_권한정의.md`는 v2로 개정되어야 하며, 개정 범위는 §12(후속 과제)에 명시한다.

## 2. 범위

### 2-1. 포함 (In-Scope)

- 메뉴 관리 화면 신규(ADM01-S / SCR-ADMIN-005)
- 권한 관리 화면 신규(ADM02-S / SCR-ADMIN-006)
- 공용 조직도 선택기 신규(복수 선택형, D8) — 메뉴 관리·권한 관리 공용
- 사용자 관리 화면 개편(기존 `admin-users.html`, '사용자·권한'에서 명칭 변경) — 권한·담당 대메뉴 컬럼의 산출값화, 사용자별 실효 권한 역조회 추가
- 초기 세팅 프리셋(대메뉴 9개 × 중메뉴 31개 전체에 대한 기본값)

### 2-2. 제외 (Out-of-Scope)

검토 문서 §5의 범위 경계 5문장을 그대로 채택한다.

1. **메뉴 권한 vs 기능 단위 권한** — 이 기능은 메뉴의 접근(보기)·편집(수정) 게이트까지만 다룬다. '수정'은 등록·수정·삭제·상신을 포함하며, 승인·반려·공표 등 결재 단계 권한은 각 화면 정책(화면정의서 §2)과 온나라 결재선의 영역으로 존치한다(PERM-004·005의 부분 해소).
2. **메뉴 권한 vs 데이터 스코프** — "본인 담당만 / 소속 부서만 / 전체" 같은 데이터 노출 범위(PERM-002)는 이번 범위에서 제외하고 각 화면 정책으로 남긴다. 지정 행에 '조회 범위' 셀렉트를 추가하는 확장은 Phase 2 후보다.
3. **프로토타입 표현 한계** — 프로토타입에는 로그인·접근 강제가 없으므로(`_공통_권한정의.md` §4), 두 화면은 설정 UI·상호작용의 목업이며 실제 차단은 구현 목표로만 기술한다.
4. **연계 문서 개정** — 이 기획이 확정되면 `_공통_권한정의.md`를 v2로 개정한다. §3 역할 정의는 새 6등급 체계로, §5 매트릭스는 초기 세팅 프리셋으로 재규정하고, §7의 PERM-001·007·008을 해소 처리한다. 화면정의서 8종의 §2가 이 문서를 참조하므로 일관성 유지가 필수다.
5. **기존 화면 재편** — 시스템 관리 하위 '사용자·권한'(SFR-015)은 '사용자 관리'로 존치(SSO 동기화 + 사용자별 실효 권한 역조회)하고, '메뉴 관리'·'권한 관리'를 신설한다. 사용자 목록의 '권한'·'담당 대메뉴' 컬럼은 새 체계의 산출값(읽기 전용)으로 재정의한다.

추가로 다음 두 가지도 범위 밖임을 명시한다.

- 메뉴 자체의 신설·삭제·순서 변경 — 메뉴는 개발 산출물이며, 메뉴 관리 화면은 이미 존재하는 메뉴의 권한만 설정한다(§6 M-5).
- 화면 내부의 개인 단위 데이터 필터링(예: 인력평가에서 지정 평가자 본인만 작성 가능한 제약)은 메뉴 수정 권한과 별개로 각 화면 자체 정책이 유지한다. 메뉴 수정 권한은 "이 화면에 들어와 쓰기 기능을 쓸 수 있는가"까지만 결정한다.

## 3. 용어 정의

| 용어 | 정의 |
|---|---|
| 권한 등급(등급) | 권한 관리에서 정의하는 접근 수준의 이름표. 구성원(부서·개인)을 갖는다. 기본 6종은 §5 참조. |
| 지정 | 특정 메뉴에 보기·수정 권한을 부여하는 개별 행위 또는 그 결과. 경로에 따라 '등급 경유 지정'과 '직접 지정'으로 나뉜다. |
| 직접 지정 | 권한 관리의 등급을 거치지 않고, 메뉴 관리에서 조직도 선택기로 특정 부서·사용자에게 곧바로 보기·수정을 부여하는 방식. |
| 지정 행 | 메뉴 관리 화면의 통합 목록에 나타나는 한 줄. 유형(등급 / 부서 / 사용자) · 대상 · 보기 값 · 수정 값을 가진다. |
| 개별 지정 | 등급이 아니라 지정 행(직접 지정)으로 특정 메뉴 1개에만 권한을 부여하는 운용 패턴. 부서별 업무 담당자처럼 등급으로 묶기 어려운 대상에 사용한다(D3). |
| 전체 권한 | 등급 속성 플래그. 켜진 등급은 모든 메뉴에 보기·수정이 자동 적용되며, 메뉴 관리의 지정 대상 목록에는 나타나지 않는다. 기본 제공 시스템 관리자 등급에 적용된다. |
| 구성원 자동(전 직원) | 등급 속성 플래그. 켜진 등급은 조직도에서 개별 추가할 필요 없이 로그인 가능한 전 직원이 자동으로 구성원이 된다. 기본 제공 일반 공무원 등급에 적용된다. |
| 보기 | 해당 메뉴(중메뉴) 화면에 진입해 내용을 조회할 수 있는 권한. |
| 수정 | 보기를 포함하며, 등록·수정·삭제·상신까지 수행할 수 있는 권한(D5). |
| 합집합 원칙 | 한 대상(등급/부서/사용자)이 여러 경로로 같은 메뉴에 권한을 받을 때, 가장 넓은 권한(수정 > 보기)을 실효값으로 삼는 원칙(D2). |
| 실효 권한 | 등급 경유 지정과 직접 지정을 모두 합집합한 뒤 산출되는 최종 접근 가능 메뉴와 그 수준. |
| 미설정 | 어떤 지정 행도 없는 메뉴의 상태. 시스템 관리자를 제외한 전원에게 기본 차단되고 GNB에서 숨겨진다(D4). |
| 초기 세팅 프리셋 | 시스템 오픈 시 최초 1회 일괄 적용되는 지정 행의 기본값 모음(§10). 이후 메뉴 관리에서 자유롭게 재설정할 수 있다. |
| 하위 부서 포함 | 조직도 선택기에서 부서 선택 시, 그 부서에 속한 하위 조직 전체를 함께 포함하는 옵션(P-6). |
| 역조회 | 사용자 1명을 기준으로, 그가 속한 등급과 개별 지정을 합산해 최종 접근 가능한 메뉴 목록을 보여주는 기능(§9). |

> 표기 대응 — 본 문서의 '보기'는 `_공통_권한정의.md`의 '읽기전용'에, '수정'은 'O'(쓰기 가능)에 대응한다. '미설정'은 같은 문서의 'X'·'숨김'에 대응한다.

## 4. 권한 모델

권한 체계는 2계층으로 나뉜다.

```
[권한 관리]                                [메뉴 관리]
 등급 정의 -----------------------------    메뉴(중메뉴) 1개당 지정 행 N개
  - 구성원(부서·개인) 관리                    - [등급]   전담부서   보기[x] 수정[x]
  - '전체 권한' 속성 (시스템 관리자 전용)       - [부서]   보건소     보기[x] 수정[x]
  - '구성원 자동' 속성 (일반 공무원 전용)       - [사용자] 김안전     보기[x] 수정[ ]
        |
        └─ "등급 불러오기"로 메뉴 관리에 반영 ──┘
           (등급 선택 시 지정 행 자동 생성 + 구성원 미리보기)

실효 권한 = 그 사용자가 속한 모든 지정 행(등급 경유 + 직접 지정)의 합집합
           동일 대상에 보기·수정이 겹치면 수정이 우선한다.
```

- **합집합**: 실효 권한은 항상 합집합이다. 등급을 불러와 '보기'를 주고 동시에 특정 부서에 '수정'을 직접 지정하면, 그 부서 구성원은 '수정'을 갖는다(§4 상단 다이어그램 예시).
- **기본 차단**: 지정 행이 하나도 없는 메뉴는 시스템 관리자를 제외한 누구도 접근할 수 없다(D4). 이는 "설정을 깜빡하면 열려버리는" 사고를 막기 위한 화이트리스트 방식이다.
- **GNB 노출 규칙**: 중메뉴는 그 사용자의 실효 권한이 '보기' 이상일 때만 노출된다. 대메뉴는 하위 중메뉴 중 1개 이상이 노출 대상일 때만 노출된다. 그 외에는 숨김이며 비활성(회색 처리) 방식은 쓰지 않는다(M-2).
- **전체 권한의 특수 취급**: 전체 권한 등급은 메뉴 관리의 어떤 지정 목록에도 나타나지 않는다. 대신 메뉴 관리 화면에 "전체 권한 등급 N개는 자동 적용됩니다"라는 안내만 노출한다. 이는 관리자가 실수로 자기 접근을 제거하는 셀프 잠금 사고를 원천 차단하기 위함이다(D1).

## 5. 권한 등급 체계

### 5-1. 권한 등급 정의 (기본 6종)

| # | 등급 | 구성원 | 비고 |
|---|---|---|---|
| 1 | 시스템 관리자 | 전담팀 관리자(개인) | 전체 권한 플래그 · 삭제 불가 · 구성원 0명 불가 |
| 2 | 군수 | 개인 1 | 경영책임자(중대재해처벌법상) — 초기 프리셋: 전 메뉴 보기 |
| 3 | 부군수·국장 | 개인 소수 | 초기 프리셋: 전 메뉴 보기 |
| 4 | 과장·팀장 | 직책자 개인 지정 | 부서 단위로 지정하면 부서원 전원이 포함되므로 개인 지정 원칙을 운용한다(§5-3) |
| 5 | 전담부서(중대재해 총괄) | 재난안전과 중대재해팀(부서) | 발주 원안 '담당부서'의 개명 — 업무 메뉴 전반 수정 |
| 6 | 일반 공무원 | 구성원 자동(전 직원) | 대시보드·통계 등 보기 + 의견청취·건의함 등록 |

> 군수와 부군수·국장을 분리 유지하는 이유 — 메뉴 접근 관점의 실효 권한은 현재 동일(전 메뉴 보기)하지만, 군수는 경영책임자로서 별도 표기의 상징·보고 가치가 있어 분리를 유지한다. 결재 권한은 온나라 연계로 외부화되어 있어 시스템 등급과는 무관하다.

### 5-2. 구 4역할 → 신 6등급 매핑

|---|---|
| 시스템 관리자 | 시스템 관리자 |

### 5-3. 운용 규칙

- **과장·팀장 개인 지정 원칙** — 이 등급에 부서를 통째로 넣으면 부서원 전원이 과장·팀장 권한을 갖게 되는 오류가 생긴다. 반드시 직책자 개인 단위로 구성원을 추가한다. 권한 관리 화면은 이 등급에 부서 단위 추가를 시도하면 안내 문구로 개인 지정을 권고한다.
- **겸직 허용 · 합집합** — 한 사용자가 여러 등급에 동시에 속할 수 있다(예: 과장·팀장이면서 전담부서 소속). 실효 권한은 모든 소속 등급과 개별 지정을 합산한 합집합이다(P-4, PERM-003 해소).
- **부서 지정 = 동적 포함** — 등급 구성원이나 메뉴 관리 개별 지정에 부서를 추가하면, 이는 특정 시점의 인원 목록을 고정하는 스냅숏이 아니라 그 부서에 속한 인원 전체를 동적으로 포함하는 규칙이다. 전입자는 자동으로 포함되고 전출자는 자동으로 제외된다(P-5, SSO·조직도 동기화 전제). 개인 단위로 추가한 지정은 이 자동 갱신 대상이 아니므로, 인사이동이 발생하면 사용자 관리 화면에서 개인 지정을 정리하는 운용 흐름을 따른다.
- **전체 권한 · 구성원 자동 플래그의 일반화** — 두 플래그는 특정 등급 이름에 고정된 것이 아니라 등급 속성이므로, 원칙적으로 신규 등급에도 설정할 수 있다. 다만 보호 규칙(§7 P-3)은 등급명이 아니라 "전체 권한이 켜진 모든 등급"에 적용되도록 일반화한다. 이는 시스템 관리자 등급이 이름을 바꾸거나 관리자 등급이 여러 개로 늘어나는 경우에도 셀프 잠금 방지 규칙이 깨지지 않도록 하기 위함이다.
- **시스템 관리자 보호** — 기본 제공 시스템 관리자 등급은 삭제할 수 없고 구성원을 최소 1명 유지해야 한다(P-3).

| 우 패널(대메뉴 선택 시) | 하위 일괄 적용 | 하나의 지정(등급 또는 직접 지정)을 선택해 하위 중메뉴 전체에 합집합으로 추가 |

### 인터랙션 플로우

1. 좌측 트리에서 검색 또는 스크롤로 메뉴를 찾는다.
2. 중메뉴를 클릭하면 우측에 그 메뉴의 기본 정보와 현재 지정 목록이 로드된다. 대메뉴를 클릭하면 하위 중메뉴 요약표가 로드된다.
3. (중메뉴) 라디오로 '권한 불러오기'를 선택하면 등급 선택 속성창이 열린다. 등급을 고르면 지정 행이 통합 목록에 추가되고 보기·수정 기본값은 비어 있어 관리자가 직접 체크한다.
4. (중메뉴) 라디오로 '직접 지정'을 선택하면 조직도 선택기(§8)가 열린다. 부서·사용자를 복수 선택해 적용하면 각각 지정 행으로 통합 목록에 추가된다.
5. 통합 목록의 각 행에서 보기·수정을 체크한다. 수정을 체크하면 보기가 자동으로 함께 체크되고 잠긴다(N-1).
6. 지정 행을 제거하거나, [다른 메뉴 설정 복사]로 다른 중메뉴의 설정을 통째로 가져올 수 있다(복사 시 확인 다이얼로그, M-7).
7. [저장]을 눌러야 반영된다. 저장하지 않고 좌측 트리의 다른 노드를 클릭하면 이탈 확인 다이얼로그가 뜬다(M-4).
8. (대메뉴) 하위 중메뉴 권한 요약에서 기준 메뉴를 선택하고 [하위 전체에 복사]를 누르면, 기준 메뉴의 설정을 하위 중메뉴 전체에 복사한다. 실행 전 "기준 메뉴의 설정을 하위 N개 메뉴에 복사하며 각 메뉴의 기존 설정은 대체됩니다"라는 확인 다이얼로그를 거친다(N-2).

### 유효성 · 가드 규칙

| ID | 규칙 |
|---|---|
| M-1 | 지정 행이 없는 메뉴는 시스템 관리자를 제외한 전원에게 기본 차단되며, 트리 노드에 '미설정' 배지가 표기된다. 시스템 오픈 시 §10의 초기 세팅 프리셋이 일괄 적용된다. |
| M-2 | 접근 불가 메뉴는 GNB에서 숨김 처리한다(비활성이 아니다). 대메뉴는 하위 중메뉴가 1개 이상 접근 가능할 때만 노출된다. |
| M-3 | 권한 설정은 중메뉴(화면) 단위로만 한다. 대메뉴 노드에는 지정 행을 만들지 않고, 하위 중메뉴 권한 현황 요약과 [하위 일괄 적용]만 제공한다. |
| M-4 | 저장은 메뉴별 명시적 [저장]/[초기화] 버튼으로 확정한다. 미저장 상태에서 트리의 다른 노드를 클릭하면 이탈 확인 다이얼로그를 띄운다. |
| M-5 | 메뉴명·경로·연결 화면 코드·설명은 조회 전용이며, 메뉴의 신설·삭제·순서 변경은 이 화면의 범위가 아니다(메뉴는 개발 산출물). |
| M-6 | 지정 목록 하단에 "권한등급 n · 부서 n · 개인 n" 적용 요약을 표기한다. |
| M-7 | [다른 메뉴 설정 불러오기(복사)]는 합집합으로 추가되며, 이미 존재하는 대상은 높은 권한을 우선해 병합하고 실행 전 확인 다이얼로그를 거친다. |
| N-1 | 수정 체크 시 보기가 자동으로 함께 체크되고, 보기만 남기고 수정만 해제하는 것은 가능하나 보기를 해제하려면 수정도 함께 해제해야 한다(수정 ⊃ 보기, P-1 규칙을 메뉴 관리 지정 행에 적용). 보기·수정이 모두 해제된 지정 행은 해제 시점에 안내하고 저장 시 자동 제거한다(무권한 행 방지). |
| N-2 | [하위 일괄 적용]과 [다른 메뉴 설정 복사]는 기준 메뉴의 지정 전체를 대상 메뉴에 복사하며 **기존 설정을 대체**한다. 실행 전 대상 메뉴 수와 대체 사실을 확인받는다. 기준 메뉴가 미설정이면 선택 목록에 "(미설정)"으로 표기하고, 실행 시 대상 메뉴의 지정이 모두 제거되어 미설정(기본 차단)이 된다는 경고를 추가로 노출한다. |
| N-3 | 메뉴 사용 여부 토글을 끄면 시스템 관리자를 포함한 전 등급의 GNB에서 숨겨진다. 접근 권한 지정과 무관하게 최우선으로 적용되는 스위치다. |

## 7. 화면 스펙 — 권한 관리 (ADM02-S)

### 레이아웃

좌측 등급 목록 + 우측 등급 상세의 마스터-디테일 구조로, 메뉴 관리와 골격을 공유한다.

```
+-- 좌: 등급 목록 ---------+  +-- 우: 등급 상세 -------------------------------+
| [+ 등급 추가]            |  | [기본 정보] 등급명 · 설명                       |
| 시스템 관리자 (보호·1명)  |  |   전체 권한 [ ]   /   구성원 자동(전 직원) [ ]  |
| 군수 (1명)               |  | [구성원]                    [+ 조직도에서 추가] |
| 부군수·국장 (4명)         |  |  [부서] 재난안전과 중대재해팀        [제거]     |
| 과장·팀장 (23명)          |  |  [사용자] 박안전(재난안전과)         [제거]     |
| 전담부서 (부서 1)         |  | [적용 현황] 이 등급이 적용된 메뉴 N개           |
| 일반 공무원 (자동)        |  | [변경 이력]                                    |
|                          |  |                              [저장] [초기화]   |
+---------------------------+  +-------------------------------------------------+
```

등급 추가 폼은 새 모달을 띄우지 않고 우측 패널 안에 인라인으로 연다. 구성원 추가에 필요한 조직도 선택기(§8)도 페이지 레벨 단일 모달로 열리므로, 등급 상세 패널과 겹쳐도 모달은 항상 1개다.

### 요소 정의

| 영역 | 요소 | 설명 |
|---|---|---|
| 좌 목록 | [+ 등급 추가] | 우측 패널에 등급 생성 인라인 폼을 연다(모달 아님) |
| 좌 목록 | 등급 카드 | 등급명 + 구성원 수 요약. 전체 권한이 켜진 등급에는 '보호' 표기를 부기한다. |
| 우 패널 | 기본 정보 | 등급명(필수·중복 금지), 설명 |
| 우 패널 | 전체 권한 토글 | 켜면 모든 메뉴에 자동 보기·수정, 메뉴 관리의 지정 대상에서 제외 |
| 우 패널 | 구성원 자동(전 직원) 토글 | 켜면 구성원 개별 추가 UI가 "전 직원 자동 포함"으로 대체 |
| 우 패널 | 구성원 목록 | 부서·사용자 항목 + [제거]. [+ 조직도에서 추가]로 조직도 선택기(§8)를 연다. |
| 우 패널 | 적용 현황 | 이 등급이 지정 행으로 걸려 있는 메뉴 수(참조 수) |
| 우 패널 | 변경 이력 | 최근 변경(일시·처리자·변경 내용) 진입점. 상세 데이터 모델은 §12 후속 과제 |
| 우 패널 | 저장 · 초기화 | 명시적 확정 버튼 |

### 인터랙션 플로우

1. [+ 등급 추가]를 누르면 우측 패널에 등급명·설명 입력 인라인 폼이 열린다. 저장 시 등급명 필수·중복 금지 검증을 거친다(N-1).
2. 등급 카드를 클릭하면 우측에 상세가 로드된다.
3. [+ 조직도에서 추가]로 조직도 선택기(§8)를 열어 부서·사용자를 복수 선택하면 구성원 목록에 추가된다. 같은 등급에 부서와 그 부서 소속 개인을 함께 추가하면 중복 안내가 뜬다(P-4).
4. 전체 권한 토글을 켜면 구성원 목록은 유지하되(전체 권한은 접근 범위 속성이지 구성원 자동 포함 속성이 아니다), 이 등급은 메뉴 관리의 지정 대상 목록에서 빠진다는 안내가 뜬다.
5. 구성원 자동 토글을 켜면 구성원 목록 영역이 "전 직원 자동 포함 — 개별 추가 비활성"으로 바뀐다.
6. 전체 권한과 구성원 자동을 동시에 켜면, 전 직원이 전체 메뉴에 자동 접근하게 된다는 강한 확인 경고가 뜬다(N-3).
7. 등급을 삭제하면 적용 중인 메뉴 수를 경고에 표기하고, 확인 후 삭제하면 해당 메뉴들의 지정 행도 함께 제거된다(P-2).

### 유효성 · 가드 규칙

| ID | 규칙 |
|---|---|
| P-2 | 메뉴에 적용 중인 등급을 삭제하려 하면 "N개 메뉴에 적용 중 — 삭제 시 해당 메뉴에서 지정이 제거됩니다"를 확인해야 삭제된다. 등급 목록에 참조 수(적용 메뉴 수) 배지를 표기한다. |
| P-3 | 전체 권한 플래그가 켜진 등급은 삭제할 수 없고, 구성원이 1명일 때 마지막 구성원의 제거도 차단한다(셀프 잠금 방지). 기본 제공 시스템 관리자 등급이 이에 해당한다. |
| P-4 | 같은 등급에 부서와 그 부서 소속 개인을 중복 추가하면 안내한다. 겸직은 허용하며 실효 권한은 합집합(최댓값)이다. |
| P-7 | 등급 상세 하단에 [변경 이력] 진입점을 두어 최근 변경(일시·처리자·변경 내용)을 확인할 수 있게 한다. 상세 데이터 모델과 전용 조회 화면은 §12 후속 과제로 이관한다. |
| N-1 | 등급명은 필수 입력이며 기존 등급명과 중복될 수 없다. |
| N-2 | 전체 권한·구성원 자동 플래그는 신규 등급에도 설정할 수 있다. 다만 보호 규칙(P-3)은 특정 등급명이 아니라 "전체 권한이 켜진 모든 등급"에 적용된다. |
| N-3 | 한 등급에서 전체 권한과 구성원 자동을 동시에 켜면 전 직원이 전체 메뉴에 자동 접근하게 되므로, 저장 시 강한 확인 경고를 노출한다. |

## 8. 공용 조직도 선택기 스펙

메뉴 관리의 '직접 지정'과 권한 관리의 '조직도에서 추가' 양쪽에서 동일한 컴포넌트를 재사용한다. 기존 프로토타입에는 단일 선택용(개인 1명) 인라인 조직도 트리 패턴이 있으나, 이번 기능은 부서·개인 복수 선택이 필요해 별도 선택기로 확장 설계한다(D8). 기존 패턴과 마찬가지로 별도 오버레이를 쌓지 않고, 페이지 레벨의 단일 모달 원칙을 유지한다.

조직도 데이터는 전 화면 공용 단일 출처를 바라본다. 메뉴·권한·사용자 관리의 조직도 선택기는 경영방침 점검자, 의견청취 위원, 인력평가 평가자 선택 등 다른 화면의 조직도와 동일한 조직도(부서·인원)를 사용하며, 어느 한 화면에서 조직 구성이 바뀌면 모든 화면에 동일하게 반영된다.

### 레이아웃

```
+-- 조직도에서 선택 (단일 모달) -----------------------------------------+
| 검색 [                                    ]                            |
| ---- 트리 ------------------------+ ---- 선택됨(바스켓) --------------- |
| [ ] 재난안전과                     | 재난안전과 중대재해팀        [제거]  |
|   [x] 중대재해팀   (하위 부서 포함)| 박안전(재난안전과)           [제거]  |
| [ ] 보건소                         |                                    |
|   [ ] 이보건                       |                                    |
| ...                                |                                    |
+-------------------------------------------------------------------------+
|                                                 [취소]   [선택 적용(2건)]|
+---------------------------------------------------------------------------+
```

### 요소 정의

| 요소 | 설명 |
|---|---|
| 검색 | 부서명·개인명·직책 통합 검색. 검색 시 매칭 항목의 상위 조직이 자동으로 펼쳐지고 매칭 텍스트가 강조 표시되며, 매칭이 없는 가지는 숨긴다. 검색어를 지우면 기본 상태로 복귀한다. |
| 좌 트리 | 실·국 > 과 > 팀 계층의 부서 체크 + 부서별 개인 펼침. 기본 상태에서 모든 부서명이 보이도록 국 단계까지 펼쳐 제공하고, 검색줄 옆에 [전체 펼치기]/[전체 접기]를 둔다. 군수·부군수 직위 노드는 부서 단위 선택 대상이 아니며 소속 개인만 선택할 수 있다. |
| 하위 부서 포함 체크 | 하위 조직이 있는 부서에만 노출되며, **그 부서를 체크한 경우에만 활성화**된다(기본 비활성). 부서 체크를 해제하면 함께 해제·비활성된다. 켜면 하위 조직 전체를 동적으로 포함하고 포함 인원수를 실시간 표기하며, 하위의 부서·개인 체크박스는 자동 체크·비활성 처리되어 중복 선택을 막는다(P-6). |
| 우 바스켓 | 현재까지 선택한 부서·개인 목록과 개별 [제거] |
| 이미 등록된 항목 | 호출한 화면(메뉴 관리 지정 목록 또는 등급 구성원)에 이미 있는 대상은 체크 표시 + 비활성 + '등록됨' 표기로 중복 선택을 막는다. |
| 하단 버튼 | [취소], [선택 적용(N건)] — N은 체크 수에 따라 실시간 갱신되고 0건이면 적용이 차단된다. |

### 동작 원칙

- 이 선택기가 열려 있는 동안 호출 화면(메뉴 관리 우측 패널, 권한 관리 우측 패널)은 모달이 아니라 인라인 패널이므로, 선택기가 뜨는 순간에도 화면에는 모달이 정확히 1개만 존재한다(CLAUDE.md 단일 모달 규칙).
- 선택기 제목에는 호출 컨텍스트를 표기한다 — 권한 관리에서는 "구성원 추가 — {등급명}", 메뉴 관리에서는 "직접 지정 — {메뉴명}".
- [선택 적용]을 누르면 선택된 부서·개인이 호출 화면의 통합 목록 또는 구성원 목록에 지정 행 / 구성원 항목으로 각각 추가된다. 부서와 그 소속 개인이 함께 지정되는 등 경로가 겹치면 중복은 제외하고 안내한다(실효 권한은 합집합).
- 부서 선택은 §5-3의 '부서 지정 = 동적 포함' 규칙을 따른다. 즉 선택 시점의 인원이 아니라 그 부서 소속 전체를 가리키는 참조로 저장된다(P-5).

## 9. 사용자 관리 화면 변경사항

기존 `admin-users.html`('사용자·권한')은 '사용자 관리'로 명칭을 바꾸고 GNB 위치는 시스템 관리 하위 1번째를 유지한다. SSO 연동 배너와 조직도 동기화 동작은 변경하지 않는다.

### 컬럼 변경

| 컬럼 | 기존 | 변경 후 |
|---|---|---|
| 권한 | 4역할 중 1개를 고정 표기 | 대표 등급 배지의 산출값 — 소속 등급 중 최상위 등급을 표기하되, 일반 공무원(자동) 외 소속 등급 없이 개별 지정만 있으면 "메뉴별 지정"으로 표기한다. 이 화면에서 직접 편집하지 않으며, 전체 소속 등급과 근거는 [접근 메뉴] 역조회에서 확인한다. |
| 담당 대메뉴 | 텍스트로 고정 표기 | 개별 지정으로 부여받은 메뉴 목록, 또는 등급 경유 접근 범위의 요약("전체", "전 업무 메뉴" 등)을 표기하는 산출값. 어느 쪽에도 해당 없으면 "-"로 표기한다. |
| (조작) | [권한] 버튼 — 편집 placeholder | [접근 메뉴] 버튼 — 사용자별 실효 권한 역조회를 여는 버튼으로 재정의 |

### 사용자별 실효 권한 역조회

행의 [접근 메뉴]를 누르면 사용자별 실효 권한 역조회를 단일 모달로 연다. 사용자 관리 화면은 페이지이므로 이 모달이 화면의 유일한 오버레이가 되어 단일 모달 규칙(한 시점 모달 1개, 모달 위 모달 금지)에 부합한다. 모달 내용은 다음과 같다.

- 이 사용자가 속한 등급 전체(겸직이면 전부 나열)
- 등급 경유로 접근 가능한 메뉴 요약과 개별 지정으로 추가된 메뉴 목록
- 최종 실효 권한을 대메뉴별로 묶어 "보기 N개 · 수정 N개" 형태로 표기하고, 중메뉴 단위 상세 목록을 펼쳐 볼 수 있게 한다

### 하단 구성 변경

기존 화면 하단의 '권한 등급' 표(4등급 고정 목록)는 새 6등급 체계의 요약 표(등급 · 구성원 · 적용 메뉴 수 · 설명의 산출값, 읽기 전용)로 갱신하고, 카드 상단에 "권한 관리로 이동" 링크를 둔다. 등급의 정의·구성원 관리는 권한 관리에서만 이루어진다(D7).

### 기존 데이터 재해석 예시

`admin-users.html`의 기존 6명 데이터는 새 체계에서 다음과 같이 재해석된다. 개별 지정은 부서 단위와 개인 단위 두 유형을 함께 시연한다 — 안전보건교육은 보건소(부서) 지정이라 부서 인원 변동이 자동 반영되고, 유해·위험요인 관리와 도급관리는 정환경·최회계(개인) 지정이라 인사이동 시 사용자 관리 화면에서 개인 지정을 정리하는 운용 흐름(§5-3)을 따른다.

|---|---|---|---|
| 박안전 | 재난안전과 중대재해팀 | 시스템 관리자 | 시스템 관리자(전체 권한) |

## 10. 초기 세팅 프리셋 (부록)

## 12. 후속 과제

- **화면정의서 2건 작성** — SCR-ADMIN-005(메뉴 관리)·SCR-ADMIN-006(권한 관리)을 화면정의서 템플릿(v6) 양식으로 정식 작성한다. 본 PRD §6·§7을 골격으로 확장하되, 진입/이탈 경로와 사용자 시나리오 등 템플릿 고유 섹션을 채운다.
- **데이터 스코프 설계(Phase 2)** — PERM-002·D6에 따라 보류된 "본인/부서/전체" 조회 범위를 지정 행에 추가하는 확장을 별도 기획한다.
- **권한 변경 이력 상세 설계** — P-7에서 진입점만 정의한 변경 이력의 데이터 모델과 전용 조회 화면(감사 요건 대응)을 설계한다.
- **실제 접근 강제** — 로그인 연동, 메뉴·버튼 단위 게이팅, 데이터 필터링 등 실제 차단 로직 구현은 본개발 범위다. 본 PRD는 설정 데이터와 화면 UX까지만 다룬다.
- **`_공통_권한정의.md` v2 개정** — §3 역할 정의를 신 6등급 체계로 교체하고, §5 매트릭스를 초기 세팅 프리셋(§10)으로 재규정하며, §7에서 PERM-001·003·007·008을 해소 처리하고 PERM-002·004·005·006의 상태를 갱신한다.
- **`00_화면목록.md` 갱신** — SCR-ADMIN-005·006 행 추가, SCR-ADMIN-001의 화면명 표기를 '사용자·권한'에서 '사용자 관리'로 갱신한다(본 PRD는 파일을 직접 수정하지 않으며 별도 작업으로 진행한다).
- **기존 화면정의서 8종 §2 재검토** — SCR-SBM-001 등 기존 화면정의서의 4역할 표기를 6등급 체계로 재매핑할지 여부를 화면별로 확인한다.
- **PERM-006 정책 확정** — 열람자(외부 점검·감사용 읽기 전용 계정)의 운용 여부와 방식을 확정한다. 새 체계에서는 신규 등급 생성(구성원 자동 없이 개별 지정)으로 유사하게 구성할 수 있으나 운용 정책 자체는 미확정이다.
- **전체 권한 · 구성원 자동 플래그의 신규 등급 허용 범위 재검토** — §5-3·§7 N-2·N-3에서 도입한 일반화 규칙(신규 등급에도 두 플래그 허용, 위험 조합 경고)은 이번 PRD에서 새로 제안한 내용이므로, 실제 운용 시 문구·가드 강도를 다시 확인한다.

## 13. 성공 기준 · 검수 항목

1. 아무 중메뉴나 선택했을 때, 등급 불러오기와 직접 지정(부서·사용자)이 동시에 통합 목록에 누적되고 라디오 전환으로 기존 결과가 초기화되지 않는다.
2. 동일 대상에 보기와 수정이 경로별로 중복 지정되면 항상 더 높은 권한(수정)이 실효값으로 표시된다.
3. 수정을 체크하면 보기가 자동으로 함께 체크되며, 보기만 남기고 수정을 해제하는 것은 가능하지만 보기를 해제하려면 수정도 함께 해제해야 한다.
4. 전체 권한 플래그가 켜진 등급은 메뉴 관리의 지정 대상 목록에 노출되지 않고 "전체 권한 등급 N개는 자동 적용됩니다" 안내만 노출된다.
5. 지정 행이 하나도 없는 메뉴는 트리에 '미설정' 배지가 표기되고, 전체 권한 등급을 제외한 모든 사용자의 GNB에서 숨겨진다. 데이터 일괄등록 메뉴는 초기 상태에서 이 상태로 시작한다.
6. 대메뉴 노드를 선택하면 하위 중메뉴 설정 현황 요약과 [하위 일괄 적용] 진입점이 노출되며, 대메뉴 자체에는 지정 행이 생기지 않는다.
7. 미저장 상태에서 트리의 다른 노드를 클릭하면 이탈 확인 다이얼로그가 노출된다.
8. 등급 삭제를 시도하면 적용 중인 메뉴 수가 경고 문구에 표기되고, 확인 후에만 삭제되며 삭제 시 관련 메뉴의 지정 행도 함께 제거된다.
9. 전체 권한 플래그가 켜진 등급은 삭제 버튼이 비활성화되고, 구성원이 1명일 때 그 구성원의 제거도 차단된다.
10. 공용 조직도 선택기는 어느 화면에서 열든 항상 모달 1개로만 존재하며, 복수 부서·개인 선택 · 하위 부서 포함 옵션 · 선택 바스켓이 정상 동작한다.
11. 초기 세팅 프리셋을 적용한 결과가 §10 부록 표의 9개 대메뉴 × 31개 중메뉴 값과 정확히 일치한다.
