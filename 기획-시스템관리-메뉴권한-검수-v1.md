## 5. 알려진 사항 · 후속 과제

1. **00_화면목록 총계 드리프트(선행 이슈)** — 빌드 소스(`_inventory_rows.json`)는 의견청취·인력평가 3분할이 반영된 구조인데 `00_화면목록.md` §3 표는 그 이전 구조라, 뷰어 총계(62)와 문서 프로즈(56)가 이번 작업 이전부터 불일치. 이번 범위와 무관해 손대지 않음 — 별도 후속 작업으로 분리됨.
2. **화면정의서 2건 미작성** — SCR-ADMIN-005(메뉴 관리)·SCR-ADMIN-006(권한 관리). 정의서 Phase 2 작업 시 작성 (BGT01·02와 동일 상태).
3. **PERM-006(열람자/감사용 계정 운용)** — 유일하게 미해소로 남은 권한 미결사항. 발주처 확인 필요 (PRD §12에 기록됨).
4. **프로토타입 표현 한계(의도된 범위)** — 로그인·실제 접근 차단 없음. 권한 저장·사용 여부 토글은 두 관리 화면과 역조회 안에서만 표현되고 실제 GNB 노출에는 영향 없음(시연 보호 — 화면 상단 안내 문구로 명시).
5. 커밋은 발주자 로컬 확인 후 진행.

## 9. 추가 검수 — 유효성 전수 점검 및 이슈 5건 수정 (v1.3)

발주 지시("하위 포함 기본 비활성 + 두 화면 유효성 전반 점검")에 따라 브라우저 실측과 로직 정독으로 전수 점검을 수행했다.

**견고함 확인(수정 불요)** — 등급명 trim·중복 검증 / 피커 적용 시 부서·개인 겹침 안내(합집합) / 등급 삭제 시 메뉴 지정 연쇄 제거 / 보호 등급 마지막 1명 가드 / 시스템 관리 대메뉴의 일괄 적용 차단·안내 / 빈 목록 저장 → 미설정 전환·기본 차단 안내 / 전체 권한 등급의 메뉴 등급 select 자동 제외 / dirty guard·리셋·일괄 적용 confirm / 사용 여부 토글(즉시 적용 + 프로토타입 한계 문구) / 피커 취소·재오픈 시 선택 초기화.

**수정 5건 (실측 재검증 완료)**
| # | 이슈 | 수정·실측 결과 |
|---|---|---|
| 1 | '하위 포함'이 부서 미체크 상태에서 토글 가능, 해제 후 체크 잔존 | 기본 비활성 → 부서 체크 시에만 활성 → 해제 시 해제 상태로 리셋+비활성 (실측 통과) |
| 2 | 하위 포함 조상(국) 체크 후에도 하위 부서 개별 선택 가능(중복) | 체크+하위 포함 노드의 하위 부서·개인·하위 포함 전부 자동 체크+비활성, 해제 시 원복, 선택 목록 1건 유지 (실측 통과). 기추가 부서의 하위도 동일 차단 |
| 3 | '전체 권한'+'구성원 자동' 동시 체크 무경고 (PRD N-3 미구현) | 동시 체크 시 인라인 경고("전 직원에게 모든 메뉴의 보기·수정 권한이 부여되는 조합입니다…") 노출, 한쪽 해제 시 제거 (실측 통과) |
| 4 | 보기·수정 모두 해제된 무권한 지정 행이 저장·카운트됨 | 해제 시 '권한 없음 — 저장 시 제거' 배지 → 저장 시 자동 제거 + 건수 toast, 행 수·요약·트리 배지 갱신 (실측: 6→5건) |
| 5 | 미설정 메뉴를 복사·일괄 적용 소스로 선택 시 의도치 않은 전체 제거 | 소스 목록에 "(미설정)" 접미 표기 + 실행 confirm에 기본 차단 경고 명시 (실측: "데이터 일괄등록 (미설정)") |

**부수 발견·정정** — ① 기존 v1.2 검수의 "부서 체크 시 소속 개인 자동 비활성 양호" 판정은 이미 추가된 인물로 테스트한 착시였음(해당 규칙 부재). 이번 수정 2에서 통일 규칙으로 정식 구현됨. ② PRD의 '하위 일괄 적용' 서술이 합집합 추가 방식이었으나 구현·확인 문구는 대체 방식 — 대체 방식 기준으로 PRD N-2·사용자 시나리오를 정정하고, §8 선택기 스펙에 하위 포함 활성 조건·검색·기본 펼침·직위 노드 규칙·제목 컨텍스트를 명문화. ③ 검증 후 데모 데이터는 시드로 복원함(안전보건교육 지정 6건 확인).

## 8. 추가 검수 — 조직도 피커 UX 개선 (v1.2)

발주 지시("메뉴·권한 관리 피커를 직접 검증하고 기획해 개선")에 따라 사용성 실측 → 개선 기획 확정 → 구현 → 재검수를 수행했다.

**실측으로 확인했던 기존 문제** — ① 검색 무력화(접힌 조상 안의 매칭 미노출, '과장' 검색 결과 0건) ② 트리 영역 191px 과소 ③ 기본 15/16 접힘(개인 지정 시 펼침 클릭 과다) ④ 군수·부군수 노드에 '하위 포함' 노출(전 조직 선택 함정) ⑤ 적용 건수·대상 컨텍스트 미표기.

**개선 구현·재검수 결과 (브라우저 실측)** — 전부 통과:
| 항목 | 실측값 |
|---|---|
| 모달 크기 | 본문 712×700px(70vh) — 트리 영역 기존 191px 대비 대폭 확대, 2단 독립 스크롤 |
| 기본 상태 | 부서 11개 전부 노출(국 레벨 펼침) + [전체 펼치기]/[전체 접기] |
| 검색(치명 수정) | '과장' 검색 → 재난안전과장 홍길동·회계과장 조회계·환경과장 차환경·건설과장 이건설 4건 하이라이트 노출, 무관 부서(문화체육과 등) 가지 숨김, 해제 시 복귀. 입력 디바운스 약 300ms |
| 노드 체크 규칙 | 군수·부군수(post) 노드 부서 체크·하위 포함 제거(개인 체크만). 하위 포함 토글 시 인원수 실시간 갱신(재난안전과 2명↔5명) |
| 피드백 | 제목 컨텍스트("구성원 추가 — 부군수·국장" / "직접 지정 — {메뉴명}"), 적용 버튼 "적용 (N건)"·0건 비활성, 부서 단위 기추가 노드 '추가됨' 비활성 |
| 회귀 | 사용자 관리 산출값 4종 불변, 시드·프리셋 무변경, 콘솔 에러 0건, 이모지 신규 0건, 단일 모달 유지 |
